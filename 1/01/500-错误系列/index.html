502 Bad Gateway

在打开 WP 网站时提示这个错误。

然后修改 index.php, phpinfo();exit();能执行，说明可能是脚本错误。

但是打开 php-fpm.log 错误日志又看不到信息。

再打开 nginx 的错误日志 error.log 发现upstream sent too big header while reading response header from upstream

2018/09/25 09:22:08 [error] 31059#0: *745 upstream sent too big header while reading response header from upstream, client: 127.0.0.1, server: www.sample.org, request: “GET / HTTP/1.1”, upstream: “fastcgi://127.0.0.1:9000”, host: “www.sample.org”

按这里 https://stackoverflow.com/questions/23844761/upstream-sent-too-big-header-while-reading-response-header-from-upstream

修改了 nginx 的几个配置

fastcgi_buffer_size 128k;
fastcgi_buffers 16 256k;
fastcgi_busy_buffers_size 256k;

加大到256k，重启 nginx 还是报错，并且 nginx 错误日志和之前一样的内容

查看 php-pfm 的错误日志（这个文件好像记录有延迟，要过几分钟才显示刚502的信息），看大

[pool www] ‘user’ directive is ignored when FPM is not running as root

查看https://segmentfault.com/q/1010000004110091/a-1020000004110130

需要修改 php-fpm 的运行用户 /usr/local/etc/php/7.1/php-fpm.d/www.conf?

想起同一个服务器，另外一个虚拟主机没问题啊。

对比他们的 nginx vhost 里 php 的配置，也没有什么异常啊？

再仔细看 php-fpm 错误日志

ERROR: unable to bind listening socket for address ‘127.0.0.1:9000’: Address already in use (48)

原来是端口占用的问题

sudo lsof -i:9000

结束进程

brew services stop php71
sudo brew services stop php71

无效?

后来想起是升级到 4.9 后出现的问题（但是刚开始没报错，是过了一个周末来看才报错的）， 恢复到原来的文件版本就正常了？

然后继续添加下面的proxy内容就好了。 奇怪？ 这不是代理有关的吗，我的vhost 是直接绑定 ip 域名的，没有用它啊。

proxy_buffer_size  512k;
proxy_buffers   32 512k;
proxy_busy_buffers_size 512k;
proxy_temp_file_write_size 512K;

补充

过了几天又出现问题了，还是一样的错误。upstream send too big…..

并且怎么修改参数加大到 1M 都不行，重装 nginx ，配置文件到默认。 还是这个站点出问题，其他的php站点非常流畅一点问题都没有。

突然想起不记得在哪里看到，cookie 的问题。 于是进入 chrome 的开发工具 Appllication -> Cookies 清空

重新刷新，居然好了！！怀疑是不是什么插件的 cookie 导致？ 待继续观察
