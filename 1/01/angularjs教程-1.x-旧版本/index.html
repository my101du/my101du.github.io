Angular是Google出品的一个MV*前端框架.

Angular简介，以及与jQuery，Backbone等的区别

首先看看我们熟悉的jQuery

它是一个前端库，一般用于对DOM的定位、操作等，例如下面一个例子，找到页面里id=test的元素，然后把它内部的html改为“hello world”，最后添加一个样式。 在这个过程中，html和jquery的代码都是独立的。

<p id="test"></p>

$("#test").html("hello world").addClass("newClass");

Angular的特性

用于简化和规范javascript程序的开发过程，只要给DOM添加一些angular定义的metadata属性，就能够实现HTML和Javascript的“紧密关联”。特别是，它相对比较独立，你可以把它和jQuery等前端库混在一起使用。 你现在可能不太理解，不用着急，后面会有很多例子。只要知道Angular好像一个网络交换机，HTML和javascript都连在上面，通过它来很好地互相传输数据。

Backbone（还有EmberJS)

也是类似的前端框架，但要求开发者继承他们的一些javascript对象，“污染”了开发者自己的代码，也有更高的记忆成本。

#试试开发一个最简单的应用

首先写一个js文件main.js，在里面里创建模块“myApp”

var app = angular.module('myApp', []);

然后写一个index.html，调用了上面的js文件

<!doctype html>
 <html ng-app="myApp">
   <head>
     <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.0.7/angular.min.js"></script>
     <script src="js/main.js"></script>
   </head>
   <body>
   </body>
 </html>

注意html标签里的 ng-app="myApp",这个指令实现了：

  • 实例化myApp模块
  • 告诉Angular，这个模块拥有该标签，以及标签内所有DOM的结构(因为这个例子写在根元素上，所以myApp模块拥有整个页面的DOM）

如果你的应用不需要拥有对整个页面的控制权，那可以把 ng-app 写在其他任意的html元素上（例如一个div）

指令属性

我们知道DOM元素有很多“属性”，例如 <a> 就有 href、src、target 等属性。

而Angular里的“指令属性”，实际上看上去也和“属性”类似（上面例子里的 <html ng-app="myApp">），但它更像一种绑定在DOM上的“函数”，因为它能够调用方法、定义行为、绑定$controller及$scope对象，操作DOM，以及更多。

启动浏览器的时候，DOM元素上“指令属性”和普通的属性一起被解析。

Angular应用的动态性和响应能力，都是通过指令属性来实现的。

例如

<input ng-model="name" name="Name" placeholder="Enter your name"/>
<h4>Your name: {{ name }}</h4>

这段代码里的 ng-model 指令属性，用来将DOM文本框输入的值，与controller里的$scope model 绑定起来。（具体实现是在这个值上绑定一个$watch函数，这个函数运行在Angular的$digest事件循环里，让Angular能对DOM进行更新）

常见的指令属性

  • {{ 表达式 }} 这个双大括号指令属性使用$watch函数，给括号内的表达式注册一个监听器。 关于表达式的更多说明在下一节

  • ng-init

表达式

先看一个例子

{{ 8 + 1 }}9
{{ person }}{"name":"Ari Lerner"}
{{ 10 * 3.3 | currency }}$33.00

$scope

##什么是$scope

$scope是一个javascript对象，存储数据和函数，在controller和view之间传递信息，用传统的mvc来帮助理解：

MVC结构 view <——-> (model) <——-> controller 对应着 DOM <——-> (scope)

##顶级的$rootScopte

每一个Angular应用都有一个顶级的$rootScope，对应着含有np-app指令属性的那个DOM。例如上一节的main.js的代码中，添加一段

app.run(function($rootScope) {
  $rootScope.name = "Ari Lerner";
});

html文件里的{{ name }}，就会被替换成"Ari Lerner”

###自己创建一个$scope对象

为了给指定的DOM元素创建一个$scope，需要用到 ng-controller 指令属性

<div ng-controller="MyController">
  {{ person.name }}
</div>

现在，可以在这个创建的$scope对象里新建一个person对象。

app.controller('MyController', function($scope) {
  $scope.person = {
    name: "Ari Lerner"
  };
});

这样，拥有ng-controller="MyController"属性的DOM元素的任意子元素里访问这个person对象了，即{{ person.name }}

##原型继承与独立的scope

所有的scope遵循原型继承（prototypal inheritance）,能从它们的父级scope里寻找和访问自己找不到的属性和方法，一直到$rootScope；同时子scope也能修改父scope的属性和访问。如下例

app.controller('ParentController', function($scope) {
  $scope.person = {greeted: false};
});

app.controller('ChildController', function($scope) {
  $scope.sayHello = function() {
    $scope.person.greeted = true;
  }
});

<div ng-controller="ParentController">
  <div ng-controller="ChildController">
    <input type="text" ng-model="person.name" placeholder="Name"></input>
    <a ng-click="sayHello()">Say hello</a>
  </div>
  {{ person }}
</div>

input元素使用 ng-model 进行了数据的双向绑定，修改里面的值时，同时会修改它所属scope的对应属性值。 a元素添加了一个 ng-click 指令属性，表示当点击时，ChildController 的 sayHello 方法会将 ParentController 的 person.greeted 属性从false改成true。（因为子scope可以修改父scope的属性和方法）

有些指令属性可以选择性地创建一个独立的scope，不继承父级

#通过controller创建DOM元素

前面我们所有的DOM元素都是通过手写html代码来实现的。 那么有没有办法直接通过controller来创建DOM呢？特别是这在需要动态生成一些DOM元素的时候特别有用。

例如下例，生成一个audio元素（需要html5支持）,并加载一个音频。默认是不会自动播放的，我们需要在html里加入两个按钮来控制播放和暂停，并且需要处理它们的点击事件。

<div ng-controller="PlayerController">
  <button ng-click="play()" class="button" ng-show="!playing">Play</button>
  <button ng-click="stop()" class="button alert" ng-show="playing">Stop</button>
  Playing audio: <b>{{ playing }}</b>
</div>

var app = angular.module('myApp', []);

app.controller('PlayerController', ['$scope', function($scope) {
  $scope.playing = false;
  $scope.audio = document.createElement('audio');
  $scope.audio.src =media/npr.mp4';

  $scope.play = function() {
      $scope.audio.play();
      $scope.playing = true;
  };

  $scope.stop = function() {
    $scope.audio.pause();
    $scope.playing = false;
  };

  $scope.audio.addEventListener('ended', function() {
    $scope.$apply(function() {
      $scope.stop()
    });
  });

}]);

app.controller('RelatedController', ['$scope', function($scope) {
}]);

#数据绑定

##双向绑定 首先看下面的例子，input里添加有 ng-model 属性，下面有一个{{yourName}}，当在输入框里填写不同的内容时，下面<h1>里的名字也随之改变，这就是 view到model的绑定。

<input type="text" ng-model="yourName" placeholder="Enter a name here">
<h1>Hello, {{ yourName }}</h1>

那么怎么实现在model里修改值，从而在前端html里体现出来改变呢？看下一个例子，使用一个定时器不断改变clock变量的值，然后在html文件的{{clock}}里显示最新的值。

app.controller('MyController', function($scope) {  
    $scope.person = { name: "Ari Lerner" };  

    var updateClock = function() {    
        $scope.clock = new Date();  
    };  

    var timer = setInterval(function() {    
        $scope.$apply(updateClock);  
    }, 1000);  

    updateClock();
});

<div ng-controller="MyController">
  <h5>{{ clock }}</h5>
</div>

##调用函数 除了可以通过绑定来调用数据，也可以调用 $scope 中的函数，前面有例子提到的 ng-click 指令就可以将DOM元素的点击事件绑定到一个方法上。
