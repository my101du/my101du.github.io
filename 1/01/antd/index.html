import Button from 'antd/lib/button';

Grid 栅格

24 等分

通过row在水平方向建立一组column（简写col）

你的内容应当放置于col内，并且，只有col可以作为row的直接元素

Row的gutter除了设置为固定的数值，还可以设置为{ xs: 8, sm: 16, md: 24, lg: 32 }实现响应式 Col可设置 span 和 offset

<Row gutter={16}>
  <Col span={4} />
  <Col span={4} />
  <Col span={4} />
  <Col span={4} />
  <Col span={4} />
  <Col span={4}  offset={8} />
</Row>

栅格系统中的列是指1到24的值来表示其跨越的范围。例如，三个等宽的列可以使用.col-8来创建

如果一个row中的col总和超过 24，那么多余的col会作为一个整体另起一行排列

push 和 pull改变默认顺序

<Row>

  <Col span={6} pull={18}>col-6 col-pull-18</Col>
</Row>

import { HashRouter as Router, Route, Switch, Link, withRouter } from 'react-router-dom';

const breadcrumbNameMap = {
  '/apps': 'Application List',
  '/apps/1': 'Application1',
  '/apps/2': 'Application2',
  '/apps/1/detail': 'Detail',
  '/apps/2/detail': 'Detail',
};

const Apps = () => (
  <ul className="app-list">
    <li>
      <Link to="/apps/1">Application1</Link>：<Link to="/apps/1/detail">Detail</Link>
    </li>
    <li>
      <Link to="/apps/2">Application2</Link>：<Link to="/apps/2/detail">Detail</Link>
    </li>
  </ul>
);

const Home = withRouter((props) => {
  const { location } = props;
  const pathSnippets = location.pathname.split('/').filter(i => i);
  const extraBreadcrumbItems = pathSnippets.map((_, index) => {
    const url = `/${pathSnippets.slice(0, index + 1).join('/')}`;
    return (
      <Breadcrumb.Item key={url}>
        <Link to={url}>
          {breadcrumbNameMap[url]}
        </Link>
      </Breadcrumb.Item>
    );
  });
  const breadcrumbItems = [(
    <Breadcrumb.Item key="home">
      <Link to="/">Home</Link>
    </Breadcrumb.Item>
  )].concat(extraBreadcrumbItems);
  return (
    <div className="demo">
      <div className="demo-nav">
        <Link to="/">Home</Link>
        <Link to="/apps">Application List</Link>
      </div>
      <Switch>
        <Route path="/apps" component={Apps} />
        <Route render={() => <span>Home Page</span>} />
      </Switch>
      <Alert style={{ margin: '16px 0' }} message="Click the navigation above to switch:" />
      <Breadcrumb>
        {breadcrumbItems}
      </Breadcrumb>
    </div>
  );
});

报错

Inline JavaScript is not enabled.

需要修改前面的 dev 和 production 两个 webpack 配置文件，把 loader less-loader 这段，添加一个 options 改成如下

{
  loader: require.resolve('less-loader'),
  options: {
    javascriptEnabled: true
  }
}

重新启动 yarn start （让 webpack 配置文件生效），自定义主题成功

简介

Antd 包括了很多的组件，但随着项目的增大，为了在不同的团队之间封装多个组件并共享，ant design pro 提供了一个脚手架和很多“模板”，你可以直接使用它来创建一个中后台项目，而不需要慢慢搭建。

可以认为 React+Dva+antd+ES2015+g2(图形可视化) ，带了一大堆工具（路由、图表库、测试、mock数据等等）, 用 dva 简化 redux …

也可以认为是官方替代 antd-admin 的方案。

对于 IE 系列浏览器，需要提供 es5-shim 和 es6-shim 等 Polyfills 的支持。

如果你使用了 babel，强烈推荐使用 babel-polyfill 和 babel-plugin-transform-runtime 来替代以上两个 shim。

避免同时使用 babel 和 shim 两种兼容方法，以规避 #6512 中所遇问题

如果在 IE 浏览器中遇到 startsWith 的问题，请引入 es6-shim 或 babel-polyfill。

npm install --save babel-preset-es2015 babel-polyfill babel-plugin-transform-runtime

在.babelrc文件添加

{ "presets": [ "es2015", "react" ],

  "plugins": [
    ["transform-runtime", {
      "helpers": false, // defaults to true
      "polyfill": false, // defaults to true
      "regenerator": false, // defaults to true
      "moduleName": "babel-runtime" // defaults to "babel-runtime"
    }]
    ]
}

然后修改 webpack.config.js, 在 entry添加'babel-polyfill', 重新 build 就可以在IE打开了

entry: ['babel-polyfill', require.resolve('./polyfills'), paths.appIndexJs],

我在 build 的时候报错了，还没时间去看， 说是 helper 无效， 因此删除了{}。
