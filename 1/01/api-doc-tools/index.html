REST API 测试

Postman

postman 处理 session/cookie

因为 postman chrome app 版/或者 native desktop 版，都无法使用 cookie/session. （restlet 倒是可以直接用 cookie 但是很卡，常打不开而且，对结果无法格式化成 json 显示）

  1. 先安装 chrome store 里的 postman interceptor 插件
  2. 点击图标，启用它
  3. 回到 postman app
  4. 点击顶上的相同的图标，打通两个之间的传输通道
  5. 现在 postman app 可以用 chrome 保存的 session/cookie 了

出现Expected ‘:’ instead of ‘'

如果服务器端返回 json 数据较大，而“response”又设置了显示 json 的“pretty”方式，会有这个问题。 改成 raw 模式显示返回数据即可

Swagger

https://github.com/swagger-api/swagger-ui

下载后，把 dist 拷贝到项目目录下， 例如 /public/swagger

默认指向到这个文档 https://petstore.swagger.io/v2/swagger.json

要改成我们自己的文档，修改 swagger-ui/dist/index.html 文件，将

url: "http://petstore.swagger.io/v2/swagger.json",

//改成
url: "openapi.yml",

引用 & 组件

如果所有 api 信息都写在一个文件，会导致文件很长，不方便维护。我们可以把它拆分出来

/productList:
    $ref: 'auction.yml#paths/productList'

新建 auction.yml 文件，内容如下

paths:
  productList:
    get:
      tags:
      - Auction - Product List API
      security: []
      summary: get products list and child categories of category (like categoryTree + categoryLeaf)
      description: 
      operationId: getProductList
      parameters:
        - name: category
          in: query
          description: 
          required: true
          schema:
            type: int
        - name: page
          in: query
          description: 
          required: false
          schema:
            type: int
        - name: sort
          in: query
          description: (s1) feature end bidorbuy bids cbids score2
          required: false
          schema:
            type: string
        - name: order
          in: query
          description: (o1) a d
          required: flase
          schema:
            type: string
      responses:
        '200':
          description: Success
          content:
            application/json:
              schema:
                type: object
                properties:
                  ResultSet:
                    properties:
                      Result:
                        type: object
                        properties:
                          Category:
                            type: object
                            $ref: 'components/definitions.yml#components/schemas/Category'
                          Auctions:
                            type: array
                            items:
                              $ref: 'components/definitions.yml#components/schemas/Auction'
                        
                      attributes:
                        type: object
                        properties:
                          pagination:
                            $ref: 'components/definitions.yml#components/schemas/Pagination'

可以看到在这个被引用的文件里，又引用了外部文件components/definitions.yml, 还有个类似“锚点”的，定位到某个 schemas 位置

Category:
    type: object
    $ref: 'components/definitions.yml#components/schemas/Category'

文件components/definitions.yml内容如下，注意里面 Category 还能引用本文件内的另外一个 scheme ChildCategory

openapi: 3.0.0

components:
  schemas:
    Category:
      type: object
      title: Category model
      properties:
        CategoryId:
          type: integer
          format: int32
        CategoryName:
          type: string
        ChildCategory:
          type: array
          items:
            $ref: '#components/schemas/ChildCategory'
      example:
        CategoryId: 0,
        CategoryName: オークション,
        ChildCategory: 
          items:
            $ref: '#components/schemas/ChildCategory/example'
    ChildCategory:
      type: object
      title: Category model
      properties:
        CategoryId:
          type: integer
          format: int32
        CategoryName:
          type: string
        NumOfAuctions:
          type: integer
          format: int32
      example:
        CategoryId: 23336,
        CategoryName: コンピュータ,
        IsLeafToLink: false

注意到，这里还有个 example 的技巧，就是直接应用另外一个 scheme 的 exmaple

 ChildCategory: 
  items:
    $ref: '#components/schemas/ChildCategory/example'
