数据可视化技能进阶路线 SQL+BI+Echarts

记录自 数据可视化技能的进阶路线：SQL+BI+Echarts

开始

数据可视化，并不就等于各种图表

它是“使用技术手段表达数据结果的一种方式”. 需要 理解业务需求、数据结构、数据交互原理、数据表达逻辑以及考察审美的复杂工作

在整个数据分析链路的位置

是整个数据分析链路的最后一个环节. 之前，我们需要对原始数据进行大量的整理和清洗处理，这一环节首选数据库SQL，数据处理之后才是分析和可视化

两个方向

  • 商业智能BI(商业价值挖掘)
  • 动态交互图表(表达数据内容)

分析一个电商销售数据的实操案例. 如何用SQL、BI商业智和Echarts制作数据可视化

使用 SQL 技能来导入、处理数据(如 PostgreSQL)

如果从外部导入数据，如果数据小，可以直接在 Excel 里用一些函数、宏来处理； 否则要先放入数据库来提高效率

为了方便后面的处理，一般要进行如下步骤

  1. 创建对应的表
  2. 删除导入数据里多余的值、缺失的值
  3. 根据需要修改列名称，一目了然看到数据的作用

传统 BI

例如 sql server的Bi三件套，也就是SSIS，SSAS，SSRS。分别对应Etl、分析服务、报表服务。这其实也是整个数据分析的流程，从数据库取出数据，然后进行清洗转换，建立数据模型，进而生成报表。 这也是传统BI的一套流程。

有人的见解： 传统BI 比现在这些网络上的敏捷BI 还是更强大些的

敏捷BI

商业智能BI是目前各大公司常用的数据工作和数据可视化平台。它可以将数据进行快速准确的整合，计算，并提供出结果图表及决策依据，辅助决策

用户研究分析师：需要了解客户画像、用户组成、用户习惯。 市场数据分析师：需要分析产品销量、地区排名、供需关系。 财务数据分析师：关注企业资产负债率、现金流动负债率、应收款项周转率、应收款项周转率。

总之，可以表达你的研究成果，并且用数据、图表、表格来展示。

主流的工具平台有Tableau、PowerBI、FineBI, datafocus(后起之秀，据说没有前面几个处理时缓慢的问题)

以 FindBI 的使用来举例。

  • 理解数据和关系，列出公式(例如销售额=价格x销量)
  • 制作图表
  • 可视化仪表盘结果呈现(美化)

当然，如果有自己搭建的开源系统，能更好保护自己的一些敏感数据, 也能学习是如何基于数据的分析来产生决策参考的背后算法。

对比： https://www.zhihu.com/question/57677856

https://zhuanlan.zhihu.com/p/46904493: 数据建模、加工、可视化 价值高, 可据此列出数据分析需要学习的清单

Echarts（比 BI 自带的图表更自由和全面）

BI可以解决大部分可视化分析需求，但它始终是既定好的图表，只能优化组合，无法自由创作

复杂交互图表是一种蕴含更大体量数据及更复杂数据结构的可视化，可以使读者和可视化图表之间有更好的互动。其形式包括地理空间可视化、层次关系网络可视化、基于时间的多图表可视化等等。

工作流

  1. 使用SQL将数据处理成合适的格式（JSON）
  2. 选择合适的图表(模板)，读懂echarts图表中的JS参数 修改配置对应参数
  3. 加载JSON数据
  4. 实现可视化效果
