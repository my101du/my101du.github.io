Chrome 是由 google 公司开发的一款高速的、现代化的浏览器

修改被记忆的“地区”

例如曾经使用过地址在新加坡、日本的代理服务器，访问 Google 的时候，会自动跳转到 www.google.com.sg （新加坡）或 www.google.jp （日本）。

即使之后停止使用代理服务器，google 也不会自动识别到你的真实地理位置。 其实只要在浏览器地址栏里给 google 指定一个 “ncr”的参数即可，如下：

www.google.com/ncr

删除网址自动完成中的某一项

当你浏览过一些“特别”的网站，chrome 会自动帮你记录，下次当在地址栏里输入一个字母时，会自动补全一个列表，其中就含有这个网址。有时候在公共场合演示时，被别人看见很尴尬。那么如何删除呢？

在出现这个列表的时候， fn+shift+delete(Mac) 或者 shift+delete(Windows)

无法打开某些页面

在一次把 production 上的 wordpress 在本地还原后。一直无法打开文章编辑页面，在 console 里看到报错

Failed to load resource: net::ERR_INCOMPLETE_CHUNKED_ENCODING

一直以为是还原后环境有问题。网上查到有的说要改 nginx 的所属用户，但是我对比了 Safari 却功能正常。

后来发现原来是 Do Not Track功能导致的

打开 chrome 的设置，找到高级设置，Privacy，把"Send a Do Not Track"选项取消勾选就可以了（估计是我以前为了看一些特殊的网站，把这项启用了）

如果还是无法解决，后来参看

http://www.qcyoung.com/2015/09/11/%E5%85%B3%E4%BA%8Echrome%E6%8A%A5%E9%94%99netERR_INCOMPLETE_CHUNKED_ENCODING%E7%9A%84%E8%A7%A3%E5%86%B3%E6%96%B9%E6%A1%88/

把nginx的 gzip on 打开就可以了。。。。

背景

今天遇到一个 bug ，这个 bug 很神奇，在 Chrome 下刷新项目页时会报错 net::ERR_INCOMPLETE_CHUNKED_ENCODING 而显示不出网页，在 Safari 和 Firefox 下正常。查了一下这个错误的网上的解决方案。众说纷纭比较多，由于这个 bug 着实查着改了很久..这里进行一个总结，来帮助可能被坑的后来人。

BUG 描述及解决办法

在某项目页中，后台传给我了一串比较长的 json 数据，我在处理这些数据，刷新页面时 Chrome 意外报错 net::ERR_INCOMPLETE_CHUNKED_ENCODING。页面加载不了 DOM 结构。在 Firefox 及 Safari 上加载正常。

在调试后发现，报错原因在于 json 数据在 Chrome 接收的过程中会被随机截断，不能加载完全的 json 数据。导致 json 结构报错。

报错提示

这里参考我的尝试步骤以及网上的部分可行的办法来推荐尝试：

1.关闭杀毒软件

2.关闭 Chrome 高级设置里的 “预提取资源，以便更快速的加载网页”

3.关闭 Chrome 扩展程序或停用代理类扩展

4.设置 header 报表头 content-length

5.nginx fastcgi buffer 的设置

6.nginx 打开 gzip

对某个 tab 静音，以及禁止自动播放声音

有时候打开一个页面，突然会播放声音。必须右键 tab 选择 mute 才可以关闭

  1. 快速显示一个小图标，点一下就能关闭，简化右键两步操作

The second way is also simple. Go to Chrome address bar and type “chrome://flags/#enable-tab-audio-muting”, and you will see the highlighted option with ocher color “Tab audio muting UI control ” and click on “Enable”.

  1. 安装插件，像 Really Mute Inactive Tabs LT 这类，关闭自动播放声音

方式：

  1. 点小锁
  2. 把证书图标拖动到某个文件夹
  3. 打开钥匙串，切到 “证书”
  4. 添加证书
  5. 双击证书，展开 “trust”, 选‘always trust”
