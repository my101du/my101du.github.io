Swift类型工程使用最新百度地图SDK.md

  百度地图  评论 字数统计: 2722字 阅读时长: 6分

在 APP 中，如果我们要基于地图开发一些功能，可以使用 Apple 官方自带的 MapKit， 或者百度地图的 SDK。因为百度地图提供了多个平台（Android、JavaScript）的文档，学习一次后在开发其他平台上的应用时也会有帮助，一般中国的开发者都选择它。 因为目前最新版本 2.9.1 仍然是用 Objective-C 编写的，因此如果我们的工程是 Swift 的话，在引入 SDK 时，会有一些要注意的地方，踩了不少坑后，我把一些要注意的关键地方记录了下来供大家参考。

参考如下文章

http://developer.baidu.com/map/index.php?title=iossdk/guide/buildproject

http://developer.baidu.com/map/index.php?title=iossdk/guide/swift

编辑项目属性文件 info.plist

由于iOS9改用更安全的https，为了能够在iOS9中正常使用地图SDK，需要修改 info.plist 右键点击 info.plist 文件，选择“Open As……Source Code”，添加如下几行内容

<key>NSAppTransportSecurity</key>
<dict>
   <key>NSAllowsArbitraryLoads</key>
   <true/>
</dict>

然后添加以下两个 Boolean 值，都为“YES”，可以右键，“Add Row”

NSLocationWhenInUseUsageDescription
NSLocationAlwaysUsageDescription

另外 info.plist 中必须添加 Bundle display name，否则运行时会报错 manager start failed!，填写 $(PRODUCT_NAME)即可。

导入SDK文件到项目中

最新版的 SDK 不再提供 .a 类型的库，而都采用了 framework 形式的库 按照 http://developer.baidu.com/map/index.php?title=iossdk/guide/buildproject 的说明，可以通过两种方式引入 framework 文件。

第一种方法：使用 CocoaPod

$vi Podfile

#添加如下一行后保存退出
pod 'BaiduMapKit'

$pod install

出现

Installing BaiduMapKit (2.9.1) Generating Pods project

这样会在 Pods/BaiduMapKit 目录下保存所有的 framework 文件和一个 Resource 目录下的资源文件 bundle

第二种方法：手动复制

将需要的 framework 文件复制到 Resource/BaiduMapSDK 目录下，其中 BaiduMapAPI_Base.framework 是必需的，其他的并非必需，仔细看文档上有详细说明。

对于第二种方法，还要手动引入一个资源文件包（这里面有一些图片什么的）

右键项目名，“Add Files To……”，把 BaiduMapAPI_Map.framework/Resources/mapapi.bundle 这个文件添加进来，如果提示是否要复制，选是。

添加“Linked Frameworks and Libraries”(或“Link Binary With Libraries”)

进入项目设置的 Target->Build Phases 选项卡，找到“Link Binary With Libraries”，点“+”号，选 Add Other，把刚刚导入的 framework 文件添加进来（可以多选）

  • BaiduMapAPI_Base.framework
  • BaiduMapAPI_Cloud.framework
  • BaiduMapAPI_Location.framework
  • BaiduMapAPI_Map.framework
  • BaiduMapAPI_Radar.framework
  • BaiduMapAPI_Search.framework
  • BaiduMapAPI_Utils.framework

然后继续添加其他的系统库，共 9 个

  • CoreLocation.framework
  • QuartzCore.framework
  • OpenGLES.framework
  • SystemConfiguration.framework
  • CoreGraphics.framework
  • Security.framework
  • libsqlite3.0.tbd
  • CoreTelephony.framework
  • libstdc++.6.0.9.tbd

切换到 Target->General 的 Linked Frameworks and Libraries 下也能看到这些link了的库，如果没有，则添加一下。

桥接头文件

这一步特别重要，以后我们在 Swfit 项目中，引入 Objective-C 编写的第三方库，或者与 Objective-C 代码混编，都需要这一步操作。 这里假设你的项目叫“TestProject”。

  1. 首先新建一个空的 .m 文件(Objective-C)，假设命名为 tmp.m。
  2. 有人说会自动提示是否建立一个桥接头文件，但是我在 Xcode7 上没发现。
  3. 于是只好手动建立一个文件，命名为 TestProject-Bridging-Header.h
  4. 进入项目设置“Target->Build Settings”，找到“Swift Compiler-Code Generation”,编辑 Objective-C Bridging Header，输入刚创建的 .h 文件的路径，注意包含项目名，例如TestProject/TestProject-Bridging-Header.h
  5. 然后再找到 “Linking”，编辑Other Link Flags，添加一个类型为 -objC
  6. 最后选中tmp.m文件，在最右侧的文件属性编辑器里，把它的后缀名改成 .mm，看到它下面的 Type会自动变成 Default - Objective-C++ Source（这个什么编译类型的东西解释起来比较复杂，我猜测是百度地图的SDK用的什么C++编译环境？反正你只要知道 Apple 在兼容 C，C++，Objective-C，Objective-C++，Swift 方面很牛逼就是了）

import 头文件以及初始化百度地图

编辑那个桥接头文件 TestProject-Bridging-Header.h，输入如下内容

#import <BaiduMapAPI_Base/BMKBaseComponent.h>//引入base相关所有的头文件
#import <BaiduMapAPI_Map/BMKMapComponent.h>//引入地图功能所有的头文件

因为百度地图的文档还是用 Objective-C 做案例代码写的，所以你还是要看得懂这些代码，然后转成 Swift 语法写一次。

编辑 AppDelegate.swift，看看是否报错，如果没有报错的话，说明百度地图的库已经被正常加载进来了。

var mapManager: BMKMapManager?

func application(application: UIApplication, didFinishLaunchingWithOptions launchOptions: [NSObject: AnyObject]?) -> Bool {
        mapManager = BMKMapManager()
        return true
    }

Spring 动画、自定义控件外观 SwiftyJSON 解析 JSON 格式的数据 TPKeyboardAvoiding 让输入框一直在屏幕中间，避免被键盘挡住 Alamofire 网络库
