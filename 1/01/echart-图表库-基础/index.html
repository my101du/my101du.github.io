title: EChart 图表库 permalink: echarts-library tags:

  • 前端
  • echarts

如果你手中有很多的数据，需要把它们制作成直观的图表在网页、手机上显示出来，那么使用各种Javascript 图表库可以帮助你快速解决问题。现在流行的图表库有D3.js、Highcharts、ECharts等。

其中ECharts是由百度的前端开发团队出品，兼容 PC 端和移动端以及多种浏览器，体积小巧，性能强大，能够提供非常直观，生动，可交互，可高度个性化定制的数据可视化图表。特别是有详细的中文文档，对本土的开发者十分友好。

本套教程就来通过几个实际的案例，帮助你学会如何使用Echarts来制作专业和强大的图表。

快速开始

npm/webpack 方式略（SPA 等应用中）

这里直接传统的web页面里引入 echarts.min.js, 以及一个主题文件

<script src="{{ asset('js/echarts.min.js') }}" defer></script>
<script src="{{ asset('js/themes/shine.js') }}" defer></script>

创建一个 dom 容器显示图表

<div id="sales-data" style="width:800px; height:600px;"></div>

然后在 js 代码中依次

  1. 初始化一个 echart 对象
  2. 设置 options (echart 对象的属性：包括数据、定制参数等)
  3. 调用setOption向这个对象赋值
jQuery( document ).ready(function($) {
    // step 1
    var chartSalesData = echarts.init(document.getElementById('sales-data'), 'dark');
    
    // step 2
    var optionSalesData = {
        title: {
            text: 'Sales data(Monthly/BU)'
        },
        tooltip: {},
        //图例？
        legend: { 
            data:['Power', "Network", "Auto", "Wireless", "Comsumer"]
        },
        xAxis: {
        		type: 'category',
                data: ["Jan", "Feb", "Mar", "Apr", "May", "Jun", "Jul", "Aug", "Sep", "Oct", "Nov", "Dec"]
        },
        yAxis: {
        	type: 'value'
        },
        series: [{
            name: 'Power',
            type: 'bar',
            stack: 'total',
            data: [20,32,42,34,23,55,67,11,44,43]
        },
        {
            name: 'Network',
            type: 'bar',
            stack: 'total',
            data: [23,55,67,11,44,43,20,32,42,34]
        },
        //...
    ]
    };
    
    // step 3
    chartSalesData.setOption(optionSalesData); 
});

异步加载数据, 以及动态更新

除了在初始化后setOption中直接填入, 还可以通过 jQuery 等工具异步获取数据后通过 setOption 填入数据和配置项

先加载 json 数据，然后在 callback 里调用 setOption

$.get('data.json').done(function (data) {
    myChart.setOption({
        title: {
            text: '异步数据加载示例'
        },
        tooltip: {},
        legend: {
            data:['销量']
        },
        xAxis: {
            data: ["衬衫","羊毛衫","雪纺衫","裤子","高跟鞋","袜子"]
        },
        yAxis: {},
        series: [{
            name: '销量',
            type: 'bar',
            data: [5, 20, 36, 10, 10, 20]
        }]
    });
});

先 setOption 设置一个空的直角坐标轴，然后获取数据后填入(比上面一开始空白要好)

// 显示标题，图例和空的坐标轴
myChart.setOption({
    title: {
        text: '异步数据加载示例'
    },
    tooltip: {},
    legend: {
        data:['销量']
    },
    xAxis: {
        data: []
    },
    yAxis: {},
    series: [{
        name: '销量',
        type: 'bar',
        data: []
    }]
});

// 异步加载数据
$.get('data.json').done(function (data) {
    // 填入数据
    myChart.setOption({
        xAxis: {
            data: data.categories
        },
        series: [{
            // 根据名字对应到相应的系列
            name: '销量',
            data: data.data
        }]
    });
});

    //拿到数据后隐藏
    myChart.hideLoading();
    myChart.setOption(...);

定时获取数据，setOption 填入数据，而不用考虑数据到底产生了那些变化，ECharts 会找到两组数据之间的差异然后通过合适的动画去表现数据的变化

dataset 管理数据

好处，对比在 series 里的 data

用于单独的数据集声明，从而数据可以单独管理，被多个组件复用

ECharts 4 以前，数据只能声明在各个“系列 series”中, 类似这样子

{
    xAxis: {
        type: 'category'
        data: ['Matcha Latte', 'Milk Tea', 'Cheese Cocoa', 'Walnut Brownie']
    },
    yAxis: {},
    series: [{
        type: 'bar',
        name: '2015',
        data: [89.3, 92.1, 94.4, 85.4]
    }, {
        type: 'bar',
        name: '2016',
        data: [95.8, 89.4, 91.2, 76.9]
    }, {
        type: 'bar',
        name: '2017',
        data: [97.7, 83.1, 92.5, 78.1]
    }]
}

缺点是，为匹配这种数据输入形式，常需要有数据处理的过程，把数据分割设置到各个系列（和类目轴）中.

不利于多个系列共享一份数据，也不利于基于原始数据进行图表类型、系列的映射安排

而 dataset

  • 能够贴近这样的数据可视化常见思维方式：基于数据（dataset 组件来提供数据），指定数据到视觉的映射（由 encode 属性来指定映射），形成图表。
  • 数据和其他配置可以被分离开来，使用者相对便于进行单独管理，也省去了一些数据处理的步骤。
  • 数据可以被多个系列或者组件复用，对于大数据，不必为每个系列创建一份。
  • 支持更多的数据的常用格式，例如二维数组、对象数组等，一定程度上避免使用者为了数据格式而进行转换。

定义 dataset(形式 / 数量)

二维数组形式

例如前面的案例可以简化成 source里的二维数组（就像一个表格的表头、多个行。 表头是维度）

x轴是category，对应第一列，也就是说 x 轴会依次把 Matcha Latte, Milk Tea, … 列出来

然后 series 只需要声明 type就可以了，自动对应 dataset 每一列。

那么 Matcha Latte 会看起来像一组数据，有三个bar,分别显示 2015 2016 2017 的数据(下面有简称)

option = {
    legend: {},
    tooltip: {},
    dataset: {
        // 提供一份数据。
        source: [
            ['product', '2015', '2016', '2017'],
            ['Matcha Latte', 43.3, 85.8, 93.7],
            ['Milk Tea', 83.1, 73.4, 55.1],
            ['Cheese Cocoa', 86.4, 65.2, 82.5],
            ['Walnut Brownie', 72.4, 53.9, 39.1]
        ]
    },
    // 声明一个 X 轴，类目轴（category）。默认情况下，类目轴对应到 dataset 第一列。
    xAxis: {type: 'category'},
    // 声明一个 Y 轴，数值轴。
    yAxis: {},
    // 声明多个 bar 系列，默认情况下，每个系列会自动对应到 dataset 的每一列。
    series: [
        {type: 'bar'},
        {type: 'bar'},
        {type: 'bar'}
    ]
}

key-value 形式

或者用更清晰的对象数据 (key-value) 的形式，把 维度 和数据分开来定义

//...
    dataset: {
        // 这里指定了维度名的顺序，从而可以利用默认的维度到坐标轴的映射。
        // 如果不指定 dimensions，也可以通过指定 series.encode 完成映射，参见后文。
        dimensions: ['product', '2015', '2016', '2017'],
        
        // 按行的 key-value 形式（对象数组），这是个比较常见的格式。
        source: [
            {product: 'Matcha Latte', '2015': 43.3, '2016': 85.8, '2017': 93.7},
            {product: 'Milk Tea', '2015': 83.1, '2016': 73.4, '2017': 55.1},
            {product: 'Cheese Cocoa', '2015': 86.4, '2016': 65.2, '2017': 82.5},
            {product: 'Walnut Brownie', '2015': 72.4, '2016': 53.9, '2017': 39.1}
        ]
    }
 //...

而 key-value 的形式又分两种，分别是

  • 按 每一行 的 key-value 形式 （这时的 key 是多个 列名: 数据 中的）
  • 按 列名: [全部数据] 的 key-value 形式 (这时的 key 是一个列名)
dataset: [{
    // 按行的 key-value 形式（对象数组），这是个比较常见的格式。
    source: [
        {product: 'Matcha Latte', count: 823, score: 95.8},
        {product: 'Milk Tea', count: 235, score: 81.4},
        {product: 'Cheese Cocoa', count: 1042, score: 91.2},
        {product: 'Walnut Brownie', count: 988, score: 76.9}
    ]
}, {
    // 按列的 key-value 形式。
    source: {
        'product': ['Matcha Latte', 'Milk Tea', 'Cheese Cocoa', 'Walnut Brownie'],
        'count': [823, 235, 1042, 988],
        'score': [95.8, 81.4, 91.2, 76.9]
    }
}]

同时定义多个 dataset。系列可以通过 series.datasetIndex 来指定引用哪个 dataset

var option = {
    dataset: [{
        // 序号为 0 的 dataset。
        source: [...],
    }, {
        // 序号为 1 的 dataset。
        source: [...]
    }, {
        // 序号为 2 的 dataset。
        source: [...]
    }],
    series: [{
        // 使用序号为 2 的 dataset。
        datasetIndex: 2
    }, {
        // 使用序号为 1 的 dataset。
        datasetIndex: 1
    }]
};

数据到图形的映射

指定 column/row 映射 series

使用 series.seriesLayoutBy 属性来配置, 默认是 column

注意，dataset 只有二维数组格式可以用， {key: '', value: ''}这种对象数组目前并不支持 seriesLayoutBy

改造前面的例子，对比默认用 column 和改成 row作为 layoutBy

dataset: {
    source: [
        ['product', '2012', '2013', '2014', '2015'],
        ['Matcha Latte', 41.1, 30.4, 65.1, 53.3],
        ['Milk Tea', 86.5, 92.1, 85.7, 83.1],
        ['Cheese Cocoa', 24.1, 67.2, 79.5, 86.4]
    ]
},
xAxis: [
    {type: 'category', gridIndex: 0},
    {type: 'category', gridIndex: 1}
],
yAxis: [
    {gridIndex: 0},
    {gridIndex: 1}
],
grid: [
    {bottom: '55%'},
    {top: '55%'}
],
series: [
    // 这几个系列会在第一个直角坐标系中，每个系列对应到 dataset 的每一行。
    {type: 'bar', seriesLayoutBy: 'row'},
    {type: 'bar', seriesLayoutBy: 'row'},
    {type: 'bar', seriesLayoutBy: 'row'},
    // 这几个系列会在第二个直角坐标系中，每个系列对应到 dataset 的每一列。
    {type: 'bar', xAxisIndex: 1, yAxisIndex: 1},
    {type: 'bar', xAxisIndex: 1, yAxisIndex: 1},
    {type: 'bar', xAxisIndex: 1, yAxisIndex: 1},
    {type: 'bar', xAxisIndex: 1, yAxisIndex: 1}
]

思考理顺：

  1. grid 有两个元素，表示这个图表“组”里有两个独立的图表
  2. 声明一个 X 轴为类目轴（category）：默认情况下，类目轴对应到 dataset 第一列（几个产品的名称），因此其他列就是类目外的"数据”
  3. xAxis 和 yAxis 分别指定了 grid 里两个元素的 index
  4. 最重要的 series中，前面3个指定 layoutBy row, 什么意思呢？

假设默认是 column，就是以“列”为“组”（产品名），“行”为组内的元素（年份）

那么除了第一列（类目轴），剩下的数据中，以第二列为例，从上到下

2012
41.1
86.5
24.1

就是： 2012年，三个产品 A B C 的数据。

在图表中就是展示：“产品为组名，年份为元素，再显示某产品某年的数据”

      |             |                           |
  |   |     |       |   |     |       |   |     |
2012 2013 2014    2012 2013 2014    2012 2013 2014
      A                 B                 C

那如果是 row, 就是以“行”为“组”（年份），“列”为组内的元素（产品）

剩下的数据中，第二行是

'Matcha Latte', 41.1, 30.4, 65.1, 53.3

就是： Matcha Latte 这个产品，2012 2013 2014 三年的数据

图表中. 年份 为组名，然后组的元素是 产品

      |             |                           |
  |   |     |       |   |     |       |   |     |
  A   B     C       A   B     C       A   B     C    
    2012               2013                 2014

指定 dimension 映射 x/y（以及 label 等）

从 dataset 的维度（一个“维度”的意思是一行/列）映射到坐标轴（如 X、Y 轴）、提示框（tooltip）、标签（label）、图形元素大小颜色等（visualMap）

使用 series.encode 属性，以及 visualMap 组件来配置

默认映射 X 坐标轴声明为类目轴，默认情况下会自动对应到 dataset.source 中的第一列； 三个柱图系列，一一对应到 dataset.source 中后面每一列。

“维度” dimention 的概念

常用图表所描述的数据大部分是“二维表”结构

当我们把系列（series）对应到“列”的时候，那么每一列就称为一个“维度（dimension）”，而每一行称为数据项（item）。 反之，如果我们把系列（series）对应到表行，那么每一行就是“维度（dimension）”，每一列就是数据项（item）。

维度名（dimension name）

可以在定义在 dataset 的第一行（或者第一列）

//...
    dataset: {
        // 这里指定了维度名的顺序，从而可以利用默认的维度到坐标轴的映射。
        // 如果不指定 dimensions，也可以通过指定 series.encode 完成映射，参见后文。
        dimensions: ['product', '2015', '2016', '2017'],
        source: [
            {product: 'Matcha Latte', '2015': 43.3, '2016': 85.8, '2017': 93.7},
            {product: 'Milk Tea', '2015': 83.1, '2016': 73.4, '2017': 55.1},
            {product: 'Cheese Cocoa', '2015': 86.4, '2016': 65.2, '2017': 82.5},
            {product: 'Walnut Brownie', '2015': 72.4, '2016': 53.9, '2017': 39.1}
        ]
    }
 //...

上面的例子"对象数组形式"中，‘score’、‘amount’、‘product’ 就是维度名,从第二行开始，才是正式的数据.

dataset.source 中第一行（列）到底包含不包含维度名，ECharts 默认会自动探测。(dataset.sourceHeader: true 显示声明第一行（列）就是维度, 否则第一行（列）开始就直接是数据)

维度可以使用单独的 dataset.dimensions 或者 series.dimensions 来定义. 可以同时指定维度名，和维度的类型（dimension type）：

维度的类型（dimension type）

一般不需要设置维度的类型，如果因为数据为空之类原因导致判断不足够准确时，可以手动设置维度类型

var option1 = {
    dataset: {
        dimensions: [
            {name: 'score'},
            // 可以简写为 string，表示维度名。
            'amount',
            // 可以在 type 中指定维度类型。
            {name: 'product', type: 'ordinal'}
        ],
        source: [...]
    },
    ...
};

var option2 = {
    dataset: {
        source: [...]
    },
    series: {
        type: 'line',
        // 在系列中设置的 dimensions 会更优先采纳。
        dimensions: [
            null, // 可以设置为 null 表示不想设置维度名
            'amount',
            {name: 'product', type: 'ordinal'}
        ]
    },
    ...
};

可以取这些值：

  • ‘number’: 默认，表示普通数据。
  • ‘ordinal’: 对于类目、文本这些 string 类型的数据，如果需要能在数轴上使用，须是 ‘ordinal’ 类型。ECharts * 默认会自动判断这个类型。但是自动判断也是不可能很完备的，所以使用者也可以手动强制指定。*
  • ‘time’: 表示时间数据。设置成 ‘time’ 则能支持自动解析数据成时间戳（timestamp），比如* 该维度的数据是 * ‘2017-05-10’，会自动被解析。如果这个维度被用在时间数轴（axis.type 为 ‘time’）上，* 那么会被自动设置* 为 ‘time’ * 类型。时间类型的支持参见 data。*
  • ‘float’: 如果设置成 ‘float’，在存储时候会使用 TypedArray，对性能优化有好处。
  • ‘int’: 如果设置成 ‘int’，在存储时候会使用 TypedArray，对性能优化有好处。

数据到图形的映射（encode）

下例，指定了 dataset，第一行是三个维度的名称(‘score’, ‘amount’, ‘product’)，就好比数据表的“field”, 下面每行都是一组数据，分别是有关维度的数值

var option = {
    dataset: {
        source: [
            ['score', 'amount', 'product'],
            [89.3, 58212, 'Matcha Latte'],
            [57.1, 78254, 'Milk Tea'],
            [74.4, 41032, 'Cheese Cocoa'],
            [50.1, 12755, 'Cheese Brownie'],
            [89.7, 20145, 'Matcha Cocoa'],
            [68.1, 79146, 'Tea'],
            [19.6, 91852, 'Orange Juice'],
            [10.6, 101852, 'Lemon Juice'],
            [32.7, 20112, 'Walnut Brownie']
        ]
    },
    xAxis: {},
    yAxis: {type: 'category'},
    series: [
        {
            type: 'bar',
            encode: {
                // 将 "amount" 列映射到 X 轴。
                x: 'amount',
                // 将 "product" 列映射到 Y 轴。
                y: 'product'
            }
        }
    ]
};

在 series中，用 encode指定 x y 轴分别映射哪个维度。

  • x 轴显示“amount”
  • y 轴显示 “product”

那么当 x(amout)=20112 时，表示最后一行数据，它的 y(product)就是’Walnut Brownie’ 依次类推

encode格式：

冒号左边是坐标系、标签等特定名称，如 ‘x’, ‘y’, ‘tooltip’ 等，冒号右边是数据中的维度名（string 格式）或者维度的序号（number 格式，从 0 开始计数），可以指定一个或多个维度（使用数组）。通常情况下，下面各种信息不需要所有的都写，按需写即可。

完整的参数

以及一个复杂的示例（认真学习下） https://www.echartsjs.com/gallery/editor.html?c=dataset-encode1

// 在任何坐标系和系列中，都支持：
encode: {
    // 使用 “名为 product 的维度” 和 “名为 score 的维度” 的值在 tooltip 中显示
    tooltip: ['product', 'score']
    // 使用 “维度 1” 和 “维度 3” 的维度名连起来作为系列名。（有时候名字比较长，这可以避免在 series.name 重复输入这些名字）
    seriesName: [1, 3],
    // 表示使用 “维度2” 中的值作为 id。这在使用 setOption 动态更新数据时有用处，可以使新老数据用 id 对应起来，从而能够产生合适的数据更新动画。
    itemId: 2,
    // 指定数据项的名称使用 “维度3” 在饼图等图表中有用，可以使这个名字显示在图例（legend）中。
    itemName: 3
}

// 直角坐标系（grid/cartesian）特有的属性：
encode: {
    // 把 “维度1”、“维度5”、“名为 score 的维度” 映射到 X 轴：
    x: [1, 5, 'score'],
    // 把“维度0”映射到 Y 轴。
    y: 0
}

// 单轴（singleAxis）特有的属性：
encode: {
    single: 3
}

// 极坐标系（polar）特有的属性：
encode: {
    radius: 3,
    angle: 2
}

// 地理坐标系（geo）特有的属性：
encode: {
    lng: 3,
    lat: 2
}

// 对于一些没有坐标系的图表，例如饼图、漏斗图等，可以是：
encode: {
    value: 3
}

例如，要实现

  1. 第三列设置为 X 轴，第五列设置为 Y 轴
series: {
    // 注意维度序号（dimensionIndex）从 0 开始计数，第三列是 dimensions[2]。
    encode: {x: 2, y: 4},
    ...
}
  1. 第三行设置为 X 轴，第五行设置为 Y 轴
series: {
    encode: {x: 2, y: 4},
    seriesLayoutBy: 'row',
    ...
}
  1. 第二列设置为标签
series: {

默认映射的规则(不能满足采用 encode)

针对最常见直角坐标系中的图表（折线图、柱状图、散点图、K线图等）、饼图、漏斗图，给出了简单的默认的映射，从而不需要配置 encode 也可以出现图表（一旦给出了 encode，那么就不会采用默认映射）。

默认的映射规则不易做得复杂，基本规则大体是

在坐标系中（如直角坐标系、极坐标系等）

  • 如果有类目轴（axis.type 为 ‘category’），则将第一列（行）映射到这个轴上，后续每一列（行）对应一个系列。
  • 如果没有类目轴，假如坐标系有两个轴（例如直角坐标系的 X Y 轴），则每两列对应一个系列，这两列分别映射到这两个轴上。

如果没有坐标系（如饼图）

  • 取第一列（行）为名字，第二列（行）为数值（如果只有一列，则取第一列为数值）。

默认的规则不能满足要求时，就可以自己来配置 encode

dataZoom

『概览数据整体，按需关注数据细节』是数据可视化的基本交互需求。dataZoom 组件能够在直角坐标系（grid）、极坐标系（polar）中实现这一功能。

  • 对 数轴（axis） 进行『数据窗口缩放』『数据窗口平移』操作。
  • 可同时存在多个，起到共同控制的作用 会自动联动
  • 运行原理是通过『数据过滤』来达到『数据窗口缩放』的效果(可设置过滤模式)
  • 数据窗口范围的设置（百分比形式 绝对值形式）
