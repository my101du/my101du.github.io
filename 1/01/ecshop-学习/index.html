ECShop 是一个常用的商城系统，虽然比较古老，代码也不是很优雅，但是各种二次开发和第三方插件都很多

问题

ECSHOP安装数据库失败date_default_timezone_get()问题

Warning: date_default_timezone_get(): It is not safe to rely on the system’s timezone settings. You are required to use the date.timezone setting or the date_default_timezone_set() function. In case you used any of those methods and you are still getting this warning, you most likely misspelled the timezone identifier. We selected ‘UTC’ for ‘8.0/no DST’ instead in D:\wwwroot\shop\upload\install\includes\lib_installer.php on line 223

修改方法

打开 php.ini，找到 date.timezone = 去掉前面的注释，然后改成 date.timezone =Asia/Shanghai，保存配置文件，重启服务器。

高于 php5.3 的兼容问题

主要参考了本文 http://www.cnblogs.com/Gile/p/4333887.html

preg_replace 需要替换的问题，报错类似如下**

Deprecated: preg_replace(): The /e modifier is deprecated, use preg_replace_callback instead in \ecshop\includes\cls_template.php on line 300

出现以上问题是 preg_replace() 函数中用到的修饰符 /e 在 PHP5.5.x 中已经被弃用了。在PHP 5.5以上的版本用 preg_replace_callback 函数替换了 preg_replace函数。

解决方法，以 includes/cls_template.php 为例

//第300行 
//return preg_replace("/{([^\}\{\n]*)}/e", "\$this->select('\\1');", $source); 
return preg_replace_callback("/{([^\}\{\n]*)}/", function($r) { return $this->select($r[1]); }, $source);

//第494行 
//$out = "<?php \n" . '$k = ' . preg_replace("/(\'\\$[^,]+)/e" , "stripslashes(trim('\\1','\''));", var_export($t, true)) . ";\n";
$out = "<?php \n" . '$k = ' . preg_replace_callback("/(\'\\$[^,]+)/", function($r) { return stripslashes(trim($r[1],'\'')); }, var_export($t, true)) . ";\n";

//553行
//$val = preg_replace("/\[([^\[\]]*)\]/eis", "'.'.str_replace('$','\$','\\1')", $val);
$val = preg_replace_callback("/\[([^\[\]]*)\]/is", function($r){ return str_replace('$','\$',$r[1]); }, $val);

在php5.4中array_shift只能为变量，不能是函数返回值

报错类似

Strict Standards: Only variables should be passed by reference in ……\includes\cls_template.php on line 418

//422行 把下面的代码拆成两部分
$tag_array = explode(' ', $tag); $tag_sel = array_shift($tag_array);

PHP5.3+废除了”=&”符号，对象复制用”=”

报错类似

Deprecated: Assigning the return value of new by reference is deprecated in…

搜索所有PHP文件，将”=&”替换为”=”

mktime 方法不带参数被调用

报错类似

Strict Standards: mktime(): You should be using the time() function instead in ……\admin\shop_config.php on line 32

将mktime()替换成time()方法

construct 和同名函数作为构造函数的前后顺序问题

报错类似

Strict Standards: Redefining already defined constructor for class cls_sql_dump ……

PHP早期版本是使用跟类同名的函数作为构造函数，后来又出现了使用 __construct()作为构造函数，这俩个函数可以同时存在。到了PHP5.4版本，对这俩个函数在代码中的先后顺序有了严格的要求。在PHP5.4版本下，必须__construct() 在前，同名函数在后，否则就会出现上面的的错误提示。把__construct()函数放在，同名函数上面就行了。

gd_version() 问题

报错类似

Strict Standards: Non-static method cls_image::gd_version() should not be called statically in /Users/zzy/zzy_work/upupw/htdocs/temp/ecshop/includes/lib_base.php on line 346

因为 cls_image.php 中 gd_version() 不是 static 函数所以在lib_base.php 与 lib_installer.php 中调用时才会出现以上问题。

解决方法: 在 cls_image.php 文件中，用 Shift+F 去搜索 gd_version 函数。然后在gd_version 方法前加 static 修饰符，是此函数变成静态函数。

或者在lib_base.php 与 lib_installer.php 函数中找到 cls_image::gd_version() 部分, 然后分别创建cls_image 实例，之后用创建的实例再去调用 gd_version() 函数。

$cls_gile = new cls_image();
return $cls_gile->gd_version();

vbb继承了integrate类并且重写了 set_cookie() 函数，但vbb重写set_cookie函数的参数(好吧我复制的)

报错类似

Strict Standards: Declaration of vbb::set_cookie() should be compatible with integrate::set_cookie($username = ‘', $remember = NULL)

vbb继承了integrate类并且重写了 set_cookie() 函数，但vbb重写set_cookie函数的参数 与 其父类set_cookie 的参数不符所以出现以上问题。

解决方法: function set_cookie ($username="") 改为 function set_cookie ($username="", $remember = NULL)

注意：以上修改文件后，都需要进入后台更新缓存
