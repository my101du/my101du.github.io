Electron 跨平台 GUI 应用开发

  frontend  评论 字数统计: 3811字 阅读时长: 8分

快速开始

# 克隆示例项目的仓库
$ git clone https://github.com/electron/electron-quick-start

# 进入这个仓库
$ cd electron-quick-start

# 安装依赖并运行
$ npm install && npm start

文件说明

package.json 里关键的有几个

"main": "main.js" 在该文件里定义 electronic 应用的入口文件、窗口、启动事件等。比如，在 react 开发阶段，加载 mainWindow.loadURL('http://localhost:3000/');

build 完毕后，加载 html 文件

mainWindow.loadURL(url.format({
     pathname: path.join(__dirname, './build/index.html'),
     protocol: 'file:',
     slashes: true
   }))

在“scripts” 里定义通过 npm 启动 electron "electron-start": "electron .",

在已有的项目里添加 electron 支持

实际上就是把前面两步有关的文件复制过来，以及修改 package.json 增加运行命令

  1. clone 示例项目到另外一个文件夹（我们需要 main.js）
  2. 复制main.js到当前项目根目录, 修改package.json,增加electron 应用入口文件定义"main": "main.js"
  3. 运行npm i --save-dev electron 安装依赖, 如果提示“npm WARN ajv-keywords@3.2.0 requires a peer of ajv@^6.0.0 but none is installed”，就手动安装下这个缺失的包npm install ajv@^6.0.0(据说是 npm 的bug)
  4. 编辑package.json文件，在scripts中增加一个命令"start-electron": "electron ." (react 项目里已经有 “start"了)
  5. 复制index.html和renderer.js文件到项目目录（第二个文件里面是空的）
  6. 修改 main.js文件里mainWindow.loadFile('index.html')这段的参数，如果开发阶段，就改成localhost:3000,如果已经编译了，就指向到正确的 dist 或 build 目录
  7. 全局安装npm -g install electron
  8. 运行npm run start-electron

基础

主进程 & 渲染进程

lectron 运行 package.json 的 main 脚本的进程被称为主进程，总是有且只有一个主进程

每个 Electron 中的 web 页面运行在它自己的渲染进程中

主进程使用 BrowserWindow 实例创建页面。 每个 BrowserWindow 实例都在自己的渲染进程里运行页面。 当一个 BrowserWindow 实例被销毁后，相应的渲染进程也会被终止。

主进程管理所有的web页面和它们对应的渲染进程。 每个渲染进程都是独立的，它只关心它所运行的 web 页面

在页面中调用与 GUI 相关的原生 API 是不被允许的，因为在 web 页面里操作原生的 GUI 资源是非常危险的，而且容易造成资源泄露。 如果你想在 web 页面里使用 GUI 操作，其对应的渲染进程必须与主进程进行通讯，请求主进程进行相关的 GUI 操作

进程间通讯

Electron为主进程（ main process）和渲染器进程（renderer processes）通信提供了多种实现方式，如可以使用ipcRenderer 和 ipcMain模块发送消息，使用 remote模块进行RPC方式通信

web页面间如何共享数据？ https://electronjs.org/docs/faq#how-to-share-data-between-web-pages

ipc 的方法传递大量数据时，会导致应用卡住 remote 方法实际上是简化 ipc

还有渲染进程之间的通信

web 页面之间共享数据

用 Storage API， localStorage，sessionStorage 或者 IndexedDB

或者用 Electron 内的 IPC 机制实现。将数据存在主进程的某个全局变量中，然后在多个渲染进程中使用 remote 模块来访问它

// 在主进程中
global.sharedObject = {
  someProperty: 'default value'
}

// 在第一个页面中
require('electron').remote.getGlobal('sharedObject').someProperty = 'new value'

// 在第二个页面中
console.log(require('electron').remote.getGlobal('sharedObject').someProperty)

使用Electron的API

Electron通过remote模块暴露一些通常只能在主进程中获取到的API。

例如在渲染进程中创建一个BrowserWindow的实例（而不是通常 main.js 主进程）

//这样写在渲染进程中时行得通的，但是在主进程中是'未定义'
const { remote } = require('electron')
const { BrowserWindow } = remote

const win = new BrowserWindow()

node.js 的支持

可以在主进程和渲染进程都能使用。 因为集成了 Nodejs，渲染进程也有了操作系统底层 API 的能力，Nodejs 中常用的 Path、fs、Crypto 等模块在 Electron 可以直接使用

"scripts": {
    
     "dev": "concurrently \"cross-env NODE_ENV=development webpack-dev-server --config webpack/dev.config.js\" \"cross-env NODE_ENV=development electron ./app/src/main/app.js\"",
    "prod": "cross-env NODE_ENV=production webpack --progress --color --config webpack/prod.config.js",
    
    "start": "npm run prod && electron app/build/app.js",
    
    "dist": "npm run prod && electron-builder",

// A simple test to verify a visible window is opened with a title
var Application = require('spectron').Application
var assert = require('assert')

var app = new Application({
  path: '/Applications/MyApp.app/Contents/MacOS/MyApp'
})

app.start().then(function () {
  // Check if the window is visible
  return app.browserWindow.isVisible()
}).then(function (isVisible) {
  // Verify the window is visible
  assert.equal(isVisible, true)
}).then(function () {
  // Get the window's title
  return app.client.getTitle()
}).then(function (title) {
  // Verify the window's title
  assert.equal(title, 'My App')
}).then(function () {
  // Stop the application
  return app.stop()
}).catch(function (error) {
  // Log any failures
  console.error('Test failed', error.message)
})

Spectron, Mocha, Chai 的关系

  • Spectron exports an Application class that can start and stop an Electron app.
  • Mocha is a test framework
  • Chai is an expectation library（chai-as-promised extends Chai）
