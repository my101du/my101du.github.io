创建项目

手动创建项目文件

$mkdir myapp
$cd myapp

# 初始化一个node项目，在这个过程中会提示你输入一些信息。
# 如项目名，作者等，并创建一个 package.json 文件
$npm init

# 安装 express （在当前项目）
$npm install express --save

创建入口文件

$vi index.js

文件内容如下：

var express = require('express')
var app = express()

app.get('/', function (req, res) {
  res.send('Hello World!')
})

app.listen(3000, function () {
  console.log('Example app listening on port 3000!')
})

然后运行入口文件

$node index.js

然后在浏览器里打开localhost:3000即可看到运行结果。

使用express-generator自动创建 Express 项目骨架文件

手动创建文件不仅麻烦，而且结构不一定合理，我们可以使用一个工具来自动生成 Express 项目相关的文件

$npm -g install express-generator

# 安装完毕后，就可以使用 `express`这个命令来创建 Express 项目了
#  -h 帮助，看到可以使用很多自定义参数，例如指定模板引擎类型（默认是 jade）， 

# 一般情况下，如果不想学 Jade 这个模板的特殊语法，使用 ejs 就可以了，和PHP/ASP 的 html 和 标签混合的格式非常相似
$express --view=ejs another_app

# 看看自动生成的文件和项目结构
$tree ./another_app

# 进入目录后，根据自动生成的 package.json 文件里的内容（一个列表），下载所有依赖文件
$cd another_app
$npm install

npm start 命令执行的背后

分析一下，为什么运行 npm start 命令会有这个结果呢？这个node ./bin/www 是怎么来的？

实际上npm start 命令，会调用package.json文件里面的参数（你可以认为这是整个项目的配置文件）。打开它发现有这么一段内容

注意这个package.json文件里scripts下面的start，就对应npm start 里的那个start了

//其他内容
"scripts": {
 	"start": "node ./bin/www"
 },

也就是说这里运行npm start，实际上等于运行node ./bin/www。你可以试试把启动命令改成下面的，返回结果是一模一样的

$DEBUG=pulse-production-line:* node ./bin/www

another-app:server Listening on port 3000 +0ms

我们看看./bin/www文件是干什么的（别看它长得像个二进制文件，其实里面是文本）

$vi ./bin/www

其中最需要关注的就是这行，它引入了我们的最终入口文件 app.js并创建了一个 HTTP 服务。你也可以在里面修改默认的端口。（默认是3000）

var app = require('../app');

知道了这样的一个流程后，这样我们就可以把注意力放到这个 app.js了。

app.js 详解

使用 PM2 来管理 express 应用程序

注意启动的是 bin/www 作为入口文件

npm i -g pm2

cd ~/your-express-proj

# 注意是 bin/www
pm2 start bin/www

然后在 nginx 里配置个反向代理就可以了

location / {
            proxy_pass http://localhost:3000;
            root   html;
            index  index.html index.htm;
}

反向代理后，静态资源 404 的问题（例如默认的 stylesheets/style.css 文件）

express 可以添加并使用 static file 的相关 middleware 来处理。

但更好的办法是向 nginx 的配置文件中指定静态文件的路径。

location ~ ^/(images/|img/|javascript/|js/|css/|stylesheets/|flash/|media/|static/|robots.txt|humans.txt|favicon.ico) {
          root /home/ubuntu/expressapp/public;
          access_log off;
          expires max;
        }

路由

匹配动作 get, post, all， 或者使用app.route()对一个路径绑定多个动作

app.get('/about', function(req, res){
	res.send('match router');
})

app.all('/secret', function (req, res, next) {
  console.log('Accessing the secret section ...')
  next() // pass control to the next handler
})

app.route('/book')
  .get(function (req, res) {
    res.send('Get a random book')
  })
  .post(function (req, res) {
    res.send('Add a book')
  })
  .put(function (req, res) {
    res.send('Update the book')
  })

可以使用字符串匹配的形式

app.get('/ab+f', function(req, res){})
app.get('/a(bc)?f', function(req, res){})
//更多

或者正则表达式匹配

app.get(/.*fly$/, function (req, res) {
  res.send('/.*fly$/')
})

参数

Route path: /users/:userId/books/:bookId Request URL: http://localhost:3000/users/34/books/8989 req.params: { “userId”: “34”, “bookId”: “8989” }

app.get('/users/:userId/books/:bookId', function (req, res) {
  res.send(req.params)
})

善用”-“和”.“符号可以让url更语义

Route path: /flights/:from-:to
Request URL: http://localhost:3000/flights/LAX-SFO
req.params: { "from": "LAX", "to": "SFO" }

捕获路由。可以使用多个callback，注意定义next对象

var cb0 = function (req, res, next) {
  console.log('CB0')
  next()
}

var cb1 = function (req, res, next) {
  console.log('CB1')
  next()
}

var cb2 = function (req, res) {
  res.send('Hello from C!')
}

app.get('/example/c', [cb0, cb1, cb2])

Response 的方法

res 对象会发送一个 response 到客户端，如果在路由 handler 里没有进行响应，客户端会一直等待。

res.download() Prompt a file to be downloaded. res.end() End the response process. res.json() Send a JSON response. res.jsonp() Send a JSON response with JSONP support. res.redirect() Redirect a request. res.render() Render a view template. res.send() Send a response of various types. res.sendFile() Send a file as an octet stream. res.sendStatus()

使用express.Router类来创建一个模块化、可安装的 handler

router.use()创建一个路由作为 module，加载了一个 middleware function

var express = require('express')
var router = express.Router()

// middleware that is specific to this router
router.use(function timeLog (req, res, next) {
  console.log('Time: ', Date.now())
  next()
})
// define the home page route
router.get('/', function (req, res) {
  res.send('Birds home page')
})
// define the about route
router.get('/about', function (req, res) {
  res.send('About birds')
})

module.exports = router

然后就可以在 app 里使用它了

var birds = require('./birds')
// ...
app.use('/birds', birds)

中间件 middleware

是一些 functions，在 application 的 request-response 周期中，能够访问 request object/response object/next function

middleware functions 做的

  • Execute any code.
  • Make changes to the request and the response objects.
  • End the request-response cycle.
  • Call the next middleware in the stack.

例如 cookies，等

如果当前 middleware function 没有结束 request-response cycle, 它必须调用 next() 把控制权交给下一个 middleware function，否则 request 会一直挂起。

创建一个

每次 application 得到一个请求，就会执行 myLogger

var myLogger = function (req, res, next) {
  console.log('LOGGED')
  next()
}

app.use(myLogger)

app.get('/', function (req, res) {
  res.send('Hello World!')
})

注意，middleware 如果被优先加载，那么也会优先执行。 因此上面例子中，放在 route 之前，否则因为res.send结束了 request-response 周期而不会执行它。

在 middleware function 中向 request object 添加属性

首先创建

var requestTime = function (req, res, next) {
  req.requestTime = Date.now()
  next()
}

接着可以从 request object 中把这个属性req.requestTime读取出来

app.use(requestTime)

app.get('/', function (req, res) {
  var responseText = 'Hello World!<br>'
  responseText += '<small>Requested at: ' + req.requestTime + '</small>'
  res.send(responseText)
})

可配置的 middleware

可以export 一个可以接受 options object 参数的 function

module.exports = function(options) {
  return function(req, res, next) {
    // Implement the middleware function based on the options object
    next()
  }
}

使用

var mw = require('./my-middleware.js')

app.use(mw({ option1: '1', option2: '2' }))
