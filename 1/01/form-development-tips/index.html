表单元素

设置为 disabled 的输入元素，其值不会提交到后台

如果需要让一些表单元素不可编辑，但又能获取到其提交的值。 使用 disabled 属性虽然能够让它不能编辑，但值也不会被提交过去。

例如

{!! Form::open(['url'=>route('portal.expediteCS'), 'method'=>'post']) !!}
{!! Form::bsText('cs','test', ['disabled'=>true]) !!}
{!! Form::bsText('type', 'TestType', ['disabled'=>true]) !!}
{!! Form::close() !!}

提交后把 request 打印出来，发现都没有

array:2 [▼
  "_token" => "NRqwwem6yscscRjZ8A62MrqBaQL4ine9Lm8QKYUW"
  "Commet" => "Hi, this is the Commet, please do it now, tks"
]

比较好的办法是用 hidden 类型的 input，然后在旁边直接显示数据。

提交多组结构类似的数据

例如一个表单里，可以添加一个用户，包括姓名、年龄、性别等

<form>
    <p>
        <input type="text" name="name" />
        <input type="text" name="age" />
        <input type="text" name="sex" />
    </p>
    <button>添加用户</button>
</form>

用户还可以点击“添加”按钮，复制生成一组一模一样的输入框。

前端可以用 jQuery 的 clone() 来实现复制表单组件。

为了能够方便地在后端处理，首先把这些表单元素改名成“数组”形式，如name[]

<p>
    <input type="text" name="name[]" />
    <input type="text" name="age[]" />
    <input type="text" name="sex[]" />
</p>

当提交多组相同结构的表单后，后端可以得到这样的一个数组

[
    'name'=>['Sally', 'Bill', 'Jack'],
    'age'=>['20', '32', '11'],
    'sex'=>['male', 'female', 'male'],
]

注意不要使用user[]['name'],会得到一个非预期的结果(多个只含有一个元素的 user)

'user' => [
    ['name' => 'Sally'],
    ['age' => '20'],
    ['sex' => 'male'],
    ['name' => 'Bill'],
    ['age' => '32'],
    ['sex' => 'female'],
],
