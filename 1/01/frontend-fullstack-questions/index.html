如何防止异步回调层级太深的问题？

回调地狱

setTimeout(function(){
  left(function(){
    setTimeout(function(){
       left(function(){
         setTimeout(function(){
           left();
         },2000);
       });
    }, 2000);
  });
}, 2000);

答案：Promise （或在 ES7 中的 async/await，更佳！ ）

var p = new Promise((resolve,reject)=>{
    setTimeout( resolve , 2000 )
})
.then( ()=>setTimeout( null, 2000 ) )
.then( ()=>setTimeout(function(){
    console.log(11)
},2000) )

聊聊 webpack 前端工作流。 （覆盖点越多越好）

  • 项目目录结构 （src, dist, js, css 等子目录，npm webpack 配置文件）
  • gulp autoprefixer （自动为 css 规则添加浏览器前缀）
  • gulp eslint （代码错误自动检查）
  • gulp watch （检测文件变动）
  • gulp connect 和 livereload （自动刷新）
  • gulp mock server （使用模拟数据）
  • gulp test （自动化测试）
  • 其他…(例如文件合并、精简、按需加载等等）
