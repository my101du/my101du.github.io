github 的 pages 服务

https://pages.github.com/

个人主页，或组织的主页

新建一个 repository 如 ironman.github.io

切换到 settings

往下滚动, 找到 Github Pages 选项, 将 Source 改为 master branch, 最后点击 Save 按钮

后面就是在本地修改、推送

git clone https://github.com/username/username.github.io

git push

或者用 hexo 来自动更新成 index.html 等文件推送了。

自定义域名

项目的主页

可以选主题

点击项目 setting 来到 GitHub Pages section Choose a theme.

然后编辑内容 例如 README.md（可以直接在 online editor 里进行） 然后 commit

测试： http://username.github.io/repository

或者从一个 scratch 开始

新建一个 repository, 添加文件 index.html （当然可以用 hexo 或其他工具来写 landing page 的 html。）

commit

settings: 在 source 里选 master branch

测试访问。

其他

注意， github pages 月流量限制 1GB 还是10G? 还有请求数量限制。 所以套一个 cloudflare 。

github pages 使用自带的 jenklly 生成站点。

之前用 hexo,

(1)经常在发布的时候报错，一大堆 js 错误信息也看不到是哪篇文章的 markdown 格式出错

(2)修改完后，还要等 hexo build 完成

(3)生成的站点是另外一个仓库（html文件），和文章本身是分割的。

而用 GitHub 自带的，体验更流畅，更重要的是发布的时候，pages 服务能告诉你哪篇文章有问题。

对于双引号， 用 {% raw %} 和 {% endraw %} 包裹起来

jekyll 本地安装，解决无详细报错的错误 Your site is having problems building: Page build failed

sudo gem install jekyll bundler

jekyll new myblog

# 如果已经有目录，强制： jekyll new myblog --force

cd myblod

# 4000 端口
bundle exec jekyll serve

不修改文件，--allow-empty 直接推送（如果 build timeout）

git commit -m 'rebuild pages' --allow-empty
git push origin master
