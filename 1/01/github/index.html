Repository

基本信息

创建 topics 便于归类

Issues

创建 issue

Assignees 分配给谁

如果团队项目，就可以给别人，否则私有项目给自己

Labels 标签，可以给一个 issue 多个标签

例如默认自带了 bug 等常见的标签. 也可以点"edit” 来添加

  • bug：无法工作的
  • duplicate: 这个 issue 或者 pull request 已经存在
  • enhancement: 新特性~
  • good first issue: 适合新手 ？
  • help wanted: 需要帮助
  • invalid: 看起来和预期的不对
  • question: 需要更多信息

Milestones 里程碑

Projects 关联到 project

拿到 issue 并完成任务后，关闭它

  • 领到任务的开发者 pull /rebase 代码
  • 本地新建一个 issue 分支 issue/xxxxxx
  • 修改代码
  • 提交 issue 分支
  • 切换到 master 分支
  • 合并 issue 分支到 master 分支
  • 删除 issue 分支

其中“关闭 issue” 最关键的就是两步：

  1. 在 issue 分支的 commit message 里用 [commit message] #[issue-id] 关联这个 issue id.
  2. 如果确认这个 issue 已经被处理，管理者 merge 到 master。 自动关闭这个 issue

如果某个提交修复了一个Issue，当提交到 master 分支时，提交信息里可以使用 fix/fixes/fixed , close/closes/closed 或者 resolve/resolves/resolved 等关键词，后面再跟上Issue号，这样就会关闭这个Issue。

$ git commit -m "Fix screwup, fixes #12"

这将会关闭Issue #12 并且在Issue讨论列表里关联引用这次提交。

如果关联了 Project, 还会自动移动到 Done

Projects

New Feature~

https://help.github.com/en/articles/about-project-boards (以及底部有几篇关联的深入文章)

和 Repository 专注于代码的管理不同，一个 Project 专注于用跟踪项目的进度、Issue 等方方面面， 并且可以和一个或几个 Repository 关联起来集中管理。

我的个人理解是：

Github 做的 Trello

只不过 trello 的 list 这里叫 Column

创建 project

  • 从 Repository 里创建基于这它的 Project
  • 单独新建一个 Project, 然后选择是否关联 Repositories (能够关联多个，对于跨 repositories 的合作很有用)

关联协作者

特性

【从 Repo 里创建项目】打开一个 Repo, 找到 Project 这个 Tab 标签, 发现一行大字，以及下面几个图标, 表明了 Project 是干什么的

Organize your issues with project boards

  • Sort tasks: 任务管理 将 issues 和 pull requests 放到 board 里面，分配优先级、备注、以及制作 task lists(任务列表)
  • Plan your project 项目规划 将前面得到的 task list 按照状态(todo/down/in process)放到不同的 column 里去
  • Automate your workflow 自动化工作流程 设置触发事件，系统自动会将 tasks 放入合适的 column(例如完成了一个 issue fix)
  • Track progress 跟踪进度 跟踪项目最近的所有变化~
  • Share status 分享与沟通 每个 card 有独立的 url, 可以把它分享出去和团队沟通
  • Wrap up 总结 完成一个项目后，关闭它并挪出当前进行的项目列表

手动在 Project 里添加 note

普通的 note 就类似一条简单文本. 在 column 里显示全部文本。 （而 issue 只显示标题。）

不过你可以 convert to issue

card 绑定 issue (issue 类型的 card 可以点击, 右侧直接编辑和分配任务。)

在 repository 里增加一个 issue, 可以在这里直接绑定 project.

也可先不绑定，回到 project, 点击 add card , 搜索 issue 绑定进来

Automatically： 自动根据 Pull Request, Issue 的 Open 等状态来纳入不同的 board/list

默认界面

OK，现在给关联的 Repo 添加一个 Issue 看看

https://help.github.com/en/articles/adding-issues-and-pull-requests-to-a-project-board

在 Repository 的界面里添加完毕后（前提是已经关联了这个 Project）

就能在 Todo Column 里看到这个新的 issue 了
