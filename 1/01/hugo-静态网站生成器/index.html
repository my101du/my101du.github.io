生成速度、兼容问题，迁移到 hugo

因为 Hexo 在处理 {{}}有问题，导致很多内容无法解析，而且想换个东西折腾。。。

Hugo 是 go 开发的

安装与初始化站点

# 安装
brew install hugo

# 创建站点
hugo new site /path/to/site

生成的目录结构

archetypes/
content/
layouts/
static/
data/
themes/
config.toml

站点配置文件

根目录下 config.toml, 配置站点的标题、主题等.

baseURL = "https://blog.4guangnian.com"
languageCode = "en-us"
title = "this is a sample site"
theme = "ananke"

完整的配置参数 https://gohugo.io/getting-started/configuration/

不得不说，实在是太多了

创建内容

创建页面

下面的命令会在目录/content/里自动生成文件about.md

hugo new about.md

文件格式如下。内容是 Markdown 格式的，+++ 之间的内容是 TOML 格式的，根据你的喜好，你可以换成 YAML 格式（使用 — 标记）或者 JSON 格式。

---
title: "About"
date: 2018-08-07T16:11:36+08:00
draft: true
---

正文内容

创建文章

和创建页面类似，但这里放到 /content/post 子目录里面，方便之后生成聚合页面。

hugo new post/first.md

可以这样组织内容，注意，目录结构和 url 的层级结构相关 /post/happy/ness

.
└── content
    └── about
    |   └── _index.md  // <- https://example.com/about/
    ├── post
    |   ├── firstpost.md   // <- https://example.com/post/firstpost/
    |   ├── happy
    |   |   └── ness.md  // <- https://example.com/post/happy/ness/
    |   └── secondpost.md  // <- https://example.com/post/secondpost/
    └── quote
        ├── first.md       // <- https://example.com/quote/first/
        └── second.md      // <- https://example.com/quote/second/

Front Matter Formats 内容元数据格式

TOML identified by opening and closing +++.

YAML identified by opening and closing ---.

支持变量 (标题、布局、别名等等)

https://gohugo.io/content-management/front-matter/

theme

下载主题

在主题中心 https://themes.gohugo.io/ 找到一个合适的

$ cd themes
$ git clone https://github.com/spf13/hyde.git

例如 universal 适合企业/产品主页。

按提示 clone 到站点的 themes 目录下，然后修改 config.toml 文件， 设置 theme

theme="universal"

主题的个性化配置

每个主题都有很多的个性化配置，例如边栏、布局、统计代码等。这些在主题的说明页面里有。

以 universal 为例

运行与部署

运行

使用 –watch 参数可以在修改文章内容时让浏览器自动刷新。

theme和 后面的baseUrl等参数，可以直接在 config.toml 文件里指定，这样运行命令时就不需要手动指定了

--watch是默认的，可以不指定？

hugo server --theme=hyde --buildDrafts --watch

hugo server -D

浏览器里打开： http://localhost:1313

居然还有自动监听修改！！！！

Draft, Future, and Expired Content

Hugo will not publish:

Content with a future publishdate value Content with draft: true status Content with a past expirydate value

部署

假设你需要部署在 GitHub Pages 上，首先在GitHub上创建一个Repository，命名为：coderzh.github.io （coderzh替换为你的github用户名）。

在站点根目录执行

hugo -D"

如果一切顺利，所有静态页面都会生成到 public 目录

然后将pubilc目录里所有文件 push 到刚创建的Repository的 master 分支。

$ cd public
$ git init
$ git remote add origin https://github.com/coderzh/coderzh.github.io.git
$ git add -A
$ git commit -m "first commit"
$ git push -u origin master
