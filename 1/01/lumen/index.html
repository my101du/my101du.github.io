配置 Dingo 的参数

有两种配置方法。

一种是直接修改 .env文件，往里面添加如下的配置。这种方式把所有配置参数都放在一起，便于查询，但有时候往往一眼看上去难以理解它们的用途。

API_STANDARDS_TREE=vnd
API_VENDOR=myproject
API_PREFIX=api
#API_DOMAIN=api.test.com
API_VERSION=v1
API_NAME=api.myproject.com
API_CONDITIONAL_REQUEST=true
API_STRICT=false
API_DEBUG=true

另一种办法是让 Lumen 在启动时查找并调用 /config 目录里的一个配置文件（这个文件在 dingo 的目录里有模板，复制过来即可），因为来自于开发者提供的模板，所以每个参数都有详细的解释，但缺点就是查找、修改参数时比较麻烦。

mkdir config

cp vendor/dingo/api/config/api.php 

# 编辑这个文件，把里面的几个参数修改成和上面 .env 文件里一样的
vi config/api.php

# 编辑 bootstrap/app.php, 添加代码，让应用启动时加载这个配置文件 /config/api.php
// $app->withFacades();
// $app->withEloquent();
$app->configure('api');

# 并且注册服务提供者， 对应文件 vendor/dingo/api/src/Provider/LumenServiceProvider.php
$app->register(Dingo\Api\Provider\LumenServiceProvider::class);

当你以后熟悉这个包的配置参数有哪些，以及各自的用途后，建议使用第一种方法提高效率。

seeJson() 和 seeJsonEqual

seeJson表示只要目标 json 字符串中有“部分匹配被测试的 json 片段”在任意位置就表示测试通过. 例如 ['title'=>'Pulse-001'] 在 ['title'=>'Pulse-001'],['title'=>'Pulse-002']里是一个片段，测试通过。

而 seeJsonEquals则要求目标 json 和被测试的 完全匹配，否则不通过。

出现报错的时候，使用 JSON 返回，而不是 HTML 的 HttpNotFound 报错

首先通过跟踪和分析 app::run 对 Exception的处理知道整个流程。

编辑 app/Exceptions/Handler.php的render方法。 其中$request->wantsJson()用于判断请求中是否有Accept json

use Symfony\Component\HttpFoundation\Response;
//略

public function render($request, Exception $e) {
		if ($request->wantsJson()) {
			$response = [
				'message' => (string)$e->getMessage(),
				'status'  => 400,
			];

			if ($e instanceof HttpException) {
				$response['message'] = Response::$statusTexts[$e->getStatusCode()];
				$response['status']  = $e->getStatusCode();
			}

			if ($this->isDebugMode()) {
				$response['debug'] = [
					'exception' => get_class($e),
					'trace'     => $e->getTrace(),
				];
			}

			return response()->json(['error' => $response], $response['status']);
		}
		return parent::render($request, $e);
	}

    public function isDebugMode(){
        return (Boolean) env('APP_DEBUG');
    }

然后你可以用curl -H "Accept: application/json" http://product.pulse.com/products/bbb 和curl -I http://product.pulse.com/products/bbb来对比一下返回的结果。（记得 APP_DEBUG 设为 false ，否则会有多余的信息出来）
