2k 与 4k 的缩放

https://www.zhihu.com/question/35300978

总结：p2415q （4k），开成1920*1080p非常好用，我凑近来看不见锯齿。完全不是2k可比拟。 大家就别折腾了。没钱4k，有钱5k。图个舒服完事儿

但是我的 P2418D 开 1080P 感觉字太小，并且有锯齿。。。（想想也是 2560 缩到 1920 并不是 2x，不会很精细的）

2k显示器实现 1600 x 900 去掉黑边（以及自定义 HiDPI 最终版）

SwtichResX (安装完毕甚至可以不需要这个，RDM 就好)

1920x1080 略模糊 (其实这里有个最关键的一步：插上电源 使用镜像，合上笔记本盖子。 ???) 1440 x 900 两边黑边

希望 1600 x 900, 可是列表里没有，添加自定义的 3200 x 1800(1600x900), 一直是“未激活”。

网上查了一些资料，添加自定义分辨率去黑边，要么都比较老，不适合 10.13 ，要么对两个关键文件的说明不明确。

经测试 1440 x 900 更瑞丽

  1. 首先，打开 SwtichResX, 找到对应外接显示器的这两个参数, 记下来，注意区分

Vendor: 厂商 例如这里的 10AC Product: 设备型号 例如这里的 D0C1

如果没有装这个软件，也可以运行命令，查看显示配置文件代码。 （注意有两个显示器，一般第二个是外接显示器的型号，第一个是mac自带的）

ioreg -lw0 | grep IODisplayPrefsKey

# 内置显示器
#    | |   | |         "IODisplayPrefsKey" = "IOService:/AppleACPIPlatformExpert/PCI0@0/AppleACPIPCI/IGPU@2/AppleIntelFramebuffer@0/display0/AppleBacklightDisplay-610-a02a"

# Dell外置显示器 （对应 10ac-d0c1）
#    | |   | |         "IODisplayPrefsKey" = "IOService:/AppleACPIPlatformExpert/PCI0@0/AppleACPIPCI/IGPU@2/AppleIntelFramebuffer@1/display0/AppleDisplay-10ac-d0c1"
  1. 新建“自定义分辨率”的定义 plist 文件

例如在 ~/Download 目录新建这个自定义分辨率文件

新建一个文件夹，叫 DisplayVendorID-10ac 注意是VendorID，值 10ac 在这个文件夹里新建一个文件叫 DisplayProductID-d0c1 注意是 “DisplayProductID”，值 d0c1

正好对应上一步找到的两个值

  1. 编辑 DisplayProductID-d0c1 文件内容

先把下面的内容粘贴进去,然后修改几个值

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
        <key>DisplayProductID</key>
        <integer>53441</integer>
        <key>DisplayVendorID</key>
        <integer>4268</integer>
        <key>scale-resolutions</key>
        <array>
                <data>
                AAAGQAAAA4QAAAABACAAAA==
                </data>
                <data>
                AAAMgAAABwgAAAABACAAAA==
                </data>
                <data>
                AAAFoAAAAyoAAAABACAAAA==
                </data>
                <data>
                AAALQAAABlQAAAABACAAAA==
                </data>
        </array>
</dict>
</plist>

把 DisplayProductID 改成十进制的数字，怎么算呢？ 可以用系统自带的计算器。。。 例如我这里ProductID d0c1 就是 53441 vendorID 10ac 就是4268

下面那一堆 scale-resolutions 里的 AABBxxxxx 怎么算？

首先要得到我们自定义分辨率的 16进制值（完整长度）

比如我想使用1600*900这个HiDPI,那么我就需要生成两个分辨率,其中一个是1600*900,一个是其双倍3200*1800.

1600,900两个值的16进制是00000640 00000384 ; 3200,1800两个值的16进制是00000C80 00000708;

后面补上 00000001 00200000

于是会得到下面的数字

00000640 00000384 00000001 00200000 00000C80 00000708 00000001 00200000

然后使用 PlistEdit Pro 这个软件，把值输入进去，让它自动转换值

  1. 最后把这个自定义分辨率的配置文件复制到系统目录里面，让 MacOS 能识别到它
cp -r ~/Download/DisplayProductID-d0c1 /System/Library/Displays/Contents/Resources/Overrides/DisplayVendorID-10ac/
  1. 注销系统，重新登录，就可以在 switchResX 里看到这个 1600x900 的 HiDPI 分辨率了，终于没黑边啦！

百度云同步盘提示“网络错误”

同步状态出错，一个红色叉叉。

重装了几次也不行。删除 ~/.baidu 和 同步目录下 .BDYunpan 都无效，重装后选择同步目录居然提示网络错误，无法列出来网路目录！

这是因为之前记录的登录信息被删除了，又开启了手机验证造成的。

退出账号，重新登录，让本地用户信息重新保存即可。

校色文件影响显示性能？

网上有说法，切换显示器的颜色模式为 sRGB IEC61966-2.1 可以改善系统显示性能和续航， 不知道有没有什么根据。。。
