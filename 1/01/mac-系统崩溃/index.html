Mac 系统崩溃

 评论 字数统计: 2982字 阅读时长: 6分

Mac 系统崩溃

前情

操作系统使用情况：

  • 最开始在一台 rmbp 2014 年 early 版上运行 10.10，一路持续跟着升级到 10.13，很正常;
  • 去年使用“迁移助手”，把系统从 rmbp 2014 上迁移到一台 rmbp 2015 新机器上，运行都正常
  • 在整个使用周期中，多次安装、卸载各种软件、开发环境。包括一些危险行为，比如关闭 SIP ，允许任何开发者的应用，等等，都没有出现过问题。
  • 多次使用 onyx, cleanmymac 等等工具对系统进行调整，多次使用一些命令行来调整系统配置（例如增加 HiDPI 分辨率配置文件），也没有出现问题。
  • 使用强度较大，平均每天都是 8:30AM-17:30PM 持续开机写代码，偶尔周末加班。
  • 极少“关机”，基本上合盖带走

近期系统变化

大概10天前升级了 10.14.1 （Mojave）,运行一切正常，感觉响应速度什么的也比 10.13 好很多，暂时没发现网友反馈的一些 bug。

昨天公司分配的域账号密码到期（系统管理员设置了3个月必须更换），修改密码后，重新登录 Outlook, 公司 wifi, Skype For Business 等等。 其他都正常，只发现 Skype 有一直无法登陆的问题，多次重启无效。

现象

早上打开盖子，像往常一样输入账号的密码，短暂黑屏，鼠标在屏幕左上角停留几秒后，回到登录界面。多次尝试，都是无法登陆系统。

？？？

排查

是否密码输入错误

  1. 我的密码很简单，没有大小写、数字、特殊字符，不可能出错。重复尝试了 N 次都不行。
  2. 仔细检查大小写切换按键，没有亮灯

检查“Guest”账号

点击 Guest 账号，发现和普通账号的情况一样，黑屏，鼠标在左上角停顿一下，自动跳回登陆界面

使用 iCloud 重置密码

点击密码输入框后面的图标，尝试使用 iCloud 重置密码。

输入新密码后，重置密码窗口自动消失，然后使用新密码、旧密码均无法登陆

使用“主密码（Master Password）”重置

我不记得这个有没有设置过了。。。放弃。 按上面步骤的操作，估计也是老样子。

恢复模式(后来才找到这个资料，未尝试)

重启 Cmd + R, 打开终端工具，输入

resetpassword

进入单用户模式(root)

重启，按住 Cmd + S，进入单用户模式（命令行界面）

# 默认是只读，打开读写
mount -uaw /

#检查磁盘
/sbin/fsck -fy

删除多用户模式用户数据库,让系统重启后自动重新设置密码

rm /var/db/.AppleSetupDone

按正常情况，操作完成后，运行reboot重启，进入到系统安装的初始界面，新建一个管理员账户即可进入系统。

但是，我的情况是，无限重启，连登录界面都进不去了。。。。

文件备份

事已至此。。系统进不去，重置密码这条路也走不通。 看来只能重装系统了。怀疑是这几年折腾系统，一直都是“升级”，没有做过“抹盘安装”操作，一些系统文件出问题了。

日常备份很重要！！！

我有做过 TimeMachine 全盘备份，但是，那是1个月以前了。。。。这个月很多的文件、代码都还没有备份的。

还有上周写的代码，也只是在本地 commit，还没 push 到远程服务器的 Git 仓库的。。。（更严重的是，ignore 了很多配置参数，就算 pull 下来，忘了这些参数，项目都跑不起来了）

由于备份的缺失，此时我心里不寒而栗。为啥？

  1. rmbp 不像台式机，可以随意拆下硬盘，装到别的机器把文件拷贝出来
  2. 我现在进不了系统，没办法“先备份，后重装”
  3. macOS 没有 win 平台下那种强大的“WinPE"启动盘，可以进去在图形界面直接复制原来磁盘的文件。

尝试在 单用户模式 加载移动硬盘，直接复制

首先把移动硬盘格式化成 FAT32 格式。

进入单用户模式 Cmd+S

# 设置可读写模式，检查磁盘
mount -uaw /
/sbin/fsck -fy

# 查看当前磁盘列表, disk0  disk1s1 .... 一般 disk 后数字最大的就是外部移动磁盘，例如 disk2s1
ls /dev/disk*

# 检查下准备复制出去的文件在不在
ls /Volumes/Macintosh HD/Users/eric

#....再看看其他目录文件

# 创建挂载点
mkdir /Volumes/usb

# 挂载移动硬盘, 如果磁盘是 ExFat NTFS 等，命令改成 mount_exfat mount_ntfs
mount_msdos -w -t /dev/disk2s1 /Volumes/usb/

如果能成功，就可以直接用 cp来复制了

结果一直挂载不了，提示什么 Kext 签名错误…

ERROR: invalid signature for ‘/System/Library/Extensions/msdosfs.kext’, will not load /System/Library/Extensions/AppleKextExcludeList.kext has invalid signature; Trust cache is disabled. Untrusted kexts are not allowed.

惨了。。。 无法挂载磁盘。那怎么把里面的文件复制出来呢？

macOS 的恢复模式里, 用磁盘工具从文件夹创建镜像

macOS 没有像 WinPE 这么强大的工具可以直接操作磁盘。

而现在我们唯一能让 rmbp 的磁盘和移动硬盘关联起来的另外一条路就是“恢复模式”下的磁盘工具。（因为前面单用户模式，无法挂载移动硬盘，这条路断了）

[笔记本硬盘]/Volumes/Macintosh HD/Users/eric —> [移动硬盘] /backup/eric.dmg

重启按住 CMD + R， 进入恢复模式

打开磁盘工具， 在顶部菜单中，选文件，新镜像，从文件夹

然后定位到 macOS 里的文件夹，作为“源”

然后选择移动硬盘的某个目录，作为“目的”

可以选择是否压缩，或者镜像的格式（DVD 什么的。。。）

开始。

试了N次，如果备份的目录里小文件很多的话（像我就有大量的 node_modules 目录），极慢，又容易中途报错

结合上面两步，先在单用户模式里压缩，然后再用磁盘工具创建镜像

既然这个磁盘工具里的“从文件夹创建镜像”有缺陷，处理“文件数量少”的情况还行，那就先压缩打包吧。

  1. 先重启 CMD+S 进入单用户模式
  2. 把那些不需要的海量小文件给删除（例如 node_modules 这种可以重新通过 npm install 生成的）
  3. 然后运行 tar czvf /Volumes/Macintosh HD/Users/eric/Download/documents.tar.gz /Volumes/Macintosh HD/Users/eric/Documents/*, 将 Documents 目录整个打包成一个 document.tar.gz.
  4. 依次打包多个压缩文件。例如 music.tar.gz work.tar.gz ….
  5. 再重启 CMD+R 进入恢复模式，使用磁盘工具，把这些压缩包全部再转存到 dmg 文件，放到移动硬盘里

为保险起见，全部做完后，再找一台 mac 电脑，把移动硬盘插上去打开这个 dmg 文件检查下是否已经备份成功了

重装系统

文件备份完了，可以重装系统了

在恢复模式（CMD+R）在线重装系统

进入恢复模式后，选第二项“Install OS”…

但是这个操作我一直失败，有人说和网络，以及与 apple 服务器的连接状况都有很大的关系。（这点在下一步也证实了）

放弃

制作启动盘

找到另外一台 macbook (真幸运，公司还有别的机器。如果没有，你只能先弄个黑苹果，或者在虚拟机里装 macos 再走后面的步骤了)

在 App Store 里下载 Install MacOS Mojave.app (失败)

重试了 N 次，每次都是下到25M左右的一个文件，就自动完成。（实际上应该有差不多 6G 的大小）

弹出安装界面，走到同意协议那一步，就报错。

要么是公司网络有问题，要么是这台 mac 有问题（重点怀疑前者）

直接下载安装镜像

由于 10.13 后，apple 就不再提供 dmg 安装镜像下载了。 网络上流出的都是网友自己先下载 Install MacOS Mojave.app 后，再打包成 dmg 提供的。

这东西不好找。。。没看到不限速就能下载的资源，基本上都是百度网盘的。 为了下这个玩意儿，还特地买了个百度云超级会员，30块一个月。。。。（不然的话 80k 的速度，6G 得下载到什么时候？）

下载完成后，加载这个 dmg， 把里面的 app 拖动到 /Application 目录去

准备移动硬盘，格式化

准备一个 8G 的移动硬盘，用磁盘工具格式化成 macos 扩展日志式，命名为 Mojave (其实随便了，这个名称主要是方便下一步直接复制启动盘制作命令)

把他插入 macbook, 会出现卷标

/Volumes/Mojave

制作启动盘

Mojave 安装文件和空的U盘都准备好了，可以制作启动盘了

打开终端，复制下面的命令

sudo /Applications/Install\ macOS\ Mojave.app/Contents/Resources/createinstallmedia --volume /Volumes/Mojave /Applications/Install\ macOS\ Mojave.app --nointeraction

慢慢等。。。 10% 20% 。。。。 100%…..

开始安装系统

启动盘做完后，插入故障的笔记本。

重启，按住 Option 键，进入启动磁盘选择模式。

会看到这个 U盘做的启动盘了，点击它。

抱着一线希望，是否可以不用抹盘，保留之前的文件？

直接点击安装系统。

希望能够像在恢复模式一样，能够只安装系统，不修改用户目录下的文件

假设这样能成功，重装完成后之前的用户数据全部保留，能够通过输入密码进入系统，这样还能把之前的用户文件全部找回来。

结论 失败

等待了半个小时，装完后，点击用户名，还是和之前一样，黑屏，然后无限登录。

抹盘安装

重新回到安装界面，磁盘工具，把整个 Macintosh HD 抹掉。

安装系统。

这次终于一切正常了

恢复文件

把移动硬盘里的 backup.dmg 复制进来，依次解压里面的 tar.gz 文件。 呼呼，幸好都在。。。。。

重新安装工作环境

漫长又乏味的 brew install， 配置参数。。。。
