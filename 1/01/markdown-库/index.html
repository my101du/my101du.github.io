list(json) to tree

因为并没有“唯一key”,而是有 h1 h2 h3 又恢复到 h1 h2 所以会有找“前一个同标签”的现象

只能逐个匹配"type": "heading", 记录“当前 depth”

如果发现一个新 heading 节点

  • depth 缩小，例如从 1 -> 2, 相当于在当前容器内 push 一个新容器。
[
    // head 1 -> H1(1)
    {
        type: heading,
        depth: 1,
        children: { //...}
    },
    // 第一个 head level 2 -> H2(1), 属于 H1(1)
    {
        type: heading,
        depth: 2,
        children: { //...}
    },
    // 后面内容都属于 H2(1), 也属于 H1(1)
    {
        type: paragraph,
        children: { [{type: text}, {type: inlineCode}, {type: text}] }
    },
    // 第二个 head level 2 -> H2(2), 属于 H1(1)
    {
        type: heading,
        depth: 2,
        children: { //...}
    },
    // 后面内容都属于 H2(2), 也属于 H1(1)
    
    // head 1 -> H1(2) 第二个一级标题
    {
        type: heading,
        depth: 1,
        children: { //...}
    },
    // 后面内容都属于 H1(2)

]

更好的处理方式：

  1. 导入 md, 简单给每个 heading 一个 uuid
  2. 独立出来一个“关联表”，标注它们之间的关系（父子、平级、以及前后顺序、打tag）
  3. 不建议在导入一批 md 后并生成关系网后， 重新又去导入生成（但能保存在云端，或智能匹配）

编辑模式，可以把整块（H1 H2 级别）进行关联、拖拽到网络中

数据 -> md 实体文件： 就像 hexo 一样读数据，然后逐个生成 md （页/文章/主题）

编辑 / 预览

react-ace + react-markdown 有问题

左侧 React-ACE 编辑器，右侧 React-markdown 渲染，通过 state 更新输入数据， 因为两边的样式不一致，无法同时滚动到相同的位置

react-syntax-highlighter ：让 react-markdown 渲染出的内容中代码高亮（或者用 highlight.js,以renderer的属性）

最新的一个 react-markdown代码高亮在这里实现：（它说 highlight.js 有问题，换另外一个） https://medium.com/young-developer/react-markdown-code-and-syntax-highlighting-632d2f9b4ada

code mirror 比 react-ace 更强大的“编辑器”，注重代码在线编辑时快捷键什么的

能在编辑模式就预览的 （不需要拆分一个渲染窗口就能看到最终效果）

需要编辑器功能： 语法高亮 head方法效果 文本加粗删除效果 表格

ACE, 能部分高亮，但不能支持快捷键。标题只能变色，不能改变大小(修改样式方式不太方便)
