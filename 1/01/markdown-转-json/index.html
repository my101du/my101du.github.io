markdown 转 json

事情是这样的。领导要求把之前一个 html 页面里的类似下面结构里，每个省里增加一条记录

<h2>省份1</h2>
<h3>记录标题</h3>
<p>第一行</p>
<p>第一行</p>
<p>第一行</p>
<h3>记录标题</h3>
<p>第一行</p>
<p>第一行</p>
<p>第一行</p>

<h2>省份2</h2>
<h3>记录标题</h3>
<p>第一行</p>
<p>第一行</p>
<p>第一行</p>
....

50个省！ 一个个复制会崩溃的。就算这次手动弄完了，下次又增加怎么办？所以想到

  1. 先把当前内容转成 json
  2. 增加一个变量，保存这个新增的记录。（每个省的这条数据是一样的）
  3. 在页面里插入一段 js, 在读取 json 数据的时候，循环每个省结束时，插入这个变量。

首先用 sublime 的正则替换，把它换成下面的简单点的 markdown 格式 （注意标题前后有空行，以及 # 后面有个空格，否则后面解析失败 只有个 raw 字段）

# 省份1

## 记录标题

第一行
第一行
第一行

## 记录标题

第一行
第一行
第一行


# 省份2

## 记录标题

第一行
第一行
第一行

[略]

然后用把 MarkDown 转 json

https://stackoverflow.com/questions/40442058/convert-markdown-to-json-object

看文档还能转换 hugo 博客系统里的文章格式，解析里面的 metadata

https://www.npmjs.com/package/markdown-to-json

npm install -g markdown-to-json

m2j -h


m2j lottery.md >> lottery.json

转出来不是我想要的。。。

换这个就好了

https://www.npmjs.com/package/md-2-json

npm install md-2-json

由于无法直接import .md 文件，需要 fs 模块将文件引入，然后解析成字符串才行

var md2json = require('md-2-json');
var fs = require('fs');

function readModuleFile(path, callback) {
    try {
        var filename = require.resolve(path);
        fs.readFile(filename, 'utf8', callback);
    } catch (e) {
        callback(e);
    }
}

readModuleFile('./distributors_usa.md', function (err, words) {
    console.log(words);

    console.log(md2json.parse(words));
});
