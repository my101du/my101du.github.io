安装Optware IPKG

安装Git

如果是用的 Entware-ng 0.97， 下面的所有ipkg都改成opkg

ipkg update
ipkg install git

初始化git服务器端仓库。你的git仓库务必存放在非系统自带的目录下(也就是说新建一个目录吧)，否则系统重启之后数据会被抹掉（我尝试了是这样的）。假定服务器端git仓库目录为：/opt/repos，项目目录为hets.git。

为NAS添加名称为git的用户和用户群，用于所有的git仓库访问。这里最好通过NAS自带的WEB界面创建用户和用户群，且git用户无需其他目录的权限。创建之后，通过WEB界面使用git用户进行登陆，这样NAS系统会自动设置好git用户的默认目录（这个目录是/share/homes/git）。

更改git仓库目录的所有者为git用户。运行如下命令。

chown -R git:git /opt/repos

由于QNAP自带的SSH server不允许除admin账号以外的其他账号使用SSH的方式连接系统，所以我们需要手动安装OpenSSH。通过admin账户登陆Web管理页面，修改系统自带的SSH端口号22为22000或其他不冲突的值。这样可以让OpenSSH正常使用22端口来运行。

安装OpenSSH并启动服务。同样需要通过系统原有的SSH server登陆admin账号（通过参数 –P22000 来指定为修改后的端口号）。

ipkg install openssh openssh-sftp-server
/opt/etc/init.d/S40sshd start

再客户端机器上通过git GUI工具生产SSH密钥，并将公钥（id_rsa.pub）添加到服务器端的认证文件里。在客户端机器的git bash中运行如下命令。

#将公钥复制到NAS
scp /c/Users/baib/.ssh/id_rsa.pub git@www.xxxxxx.net:/share/homes/git/id_rsa.pub

#通过SSH登陆admin账户后运行如下命令。
su git
cd /share/homes/git
mkdir .ssh
cat ./id_rsa.pub >> /home/git/.ssh/authorized_keys
rm ./id_rsa.pub

设置认证文件及其所在各级目录的权限。由于SSH要求密钥文件及其目录的读写权限只能开放给当前用户，否则无法通过密钥进行认证，这样每次登陆时还是需要输入密码。

chown -R git:git /share/homes/git
chmod 700 /share/homes/git
chmod 700 /share/homes/git/.ssh
chmod 600 /share/homes/git/.ssh/authorized_keys

这时，git客户端应该已经可以访问服务器端的仓库了。在客户端机器的git bash中运行如下命令。

git clone git@www.xxxxxx.net:/opt/repos/hets.git

为了安全起见，禁止通过SSH来登陆git账户。修改/etc/passwd文件。找到行git❌502:502:Linux User,,,:/home/git:/bin/sh修改为git❌502:502:Linux User,,,:/home/git:/opt/bin/git-shell。如果不知道git-shell的具体路径，可以使用which git-shell命令来获得。 使用vim时如果报这个错误“E437: terminal capability “cm” required”，则先执行以下如下命令再使用vim即可。 export TERM=xterm

我没有新建 git 用户，直接用 admin 的。(反正就在自己家里自己一个人用。。。不用和别人共享。 如果你是公司里的 NAS，就要创建若干用户了)

在 nas 上建好裸版本库

cd /repo/test.git
git init --bare

本地 push 和 clone 都报错，提示git-receive-pack和git-upload-pack命令不存在

cd ~/my_projects
git remote add origin-nas admin@192.168.93.71:/share/Repos/my_projects.git

git push origin-nas master

git clone admin@192.168.93.71:/share/Repos/my_projects.git
sh: git-upload-pack: command not found

登录 nas，找到路径。在 /opt/bin 下

which git-upload-pack
/opt/bin/git-upload-pack

修改 .bashrc 和 .bash_profile 添加路径。都无效,还是报错

vi .bashrc
#...
source .bashrc

只好在 nas 上直接复制这两个文件到 /usr/bin/ 目录了

[admin@ZZY-NAS Repos]# cp /opt/bin/git-upload-pack /usr/bin/
[admin@ZZY-NAS Repos]# cp /opt/bin/git-receive-pack /usr/bin/

在客户端重试，搞定。

开启自启动

实现开机自启动，例如和 frp 连接，在外部可访问

挂载 (过时)

根据

https://wiki.qnap.com/wiki/Running_Your_Own_Application_at_Startup

重点一句：

虽然 QNAP 的系统基于 linux，但是重启后会自动重置“开机启动”丢失配置信息

但是通过编辑 autorun.sh 这个脚本，可以让 NAS 在每次重启时自动执行它，变相实现“开启启动”

根据不同型号运行 mount 命令, 例如

mount -t ext2 /dev/mtdblock5 /tmp/config

创建文件 autorun.sh ，然后增加执行权限

vi /tmp/config/autorun.sh
chmod +x /tmp/config/autorun.sh

编辑文件。在里面增加你的命令（例如启动应用程序等。。。）

编辑内容后记得 unmount

umount /tmp/config

【测试失败】或者通过修改 QPKG 的配置文件来实现

补充：

这个方法试过，插件是已经出现在 App Center 了，但是无法启动（甚至根据某个其他正常插件复制配置文件、修改参数，也是一样）

http://techlightup.blogspot.com/2013/08/qnap-automatically-run-script-at-startup.html

http://www.lycom.net/blogs/qnapturbonasstartupscript

Edit QPKG config file:

# vi /etc/config/qpkg.conf

里面可以看到很多软件的定义参数

Declare a new dummy package by adding something like that in this file: 定义一个“假”的包，名称是 autorun

Shell is the interesting variable: at boot-time, QNAP OS will launch each QPKG’s Shell variable content.

[autorun]
Name = autorun
Version = 0.1
Author = neomilium
Date = 2013-05-06
Shell = /share/MD0_DATA/.qpkg/autorun/autorun.sh
Install_Path = /share/MD0_DATA/.qpkg/autorun
Enable = TRUE

Note: if your NAS doesn’t have /share/MD0_DATA (i.e. is a one-drive NAS), put the right directory into the Shell and Install_Path variables and adapt the following commands to your needs.

Create the dummy package directory:

# mkdir /share/MD0_DATA/.qpkg/autorun

Then I created autorun.sh in /share/CACHEDEV1_DATA/.qpkg/autorun

# vi /share/MD0_DATA/.qpkg/autorun/autorun.sh

内容如下。（Note: don’t forget “#!/bin/sh” at the beginning of script.）

CONF 和 QPKG_NAME 都不要忘了

#!/bin/sh
CONF=/etc/config/qpkg.conf
QPKG_NAME="autorun"
# autorun script for Turbo NAS
# start BIND DNS server
/opt/etc/init.d/S09named start
exit 0

Set the execute bit:

# chmod +x /share/MD0_DATA/.qpkg/autorun/autorun.sh

Reboot and enjoy!

最终

修改别的包的 sh 文件。

例如我安装 IFTTT 后，编辑/share/CACHE1_asdfasf/.qpkg/IFTTT/IFTTT.sh

在最后添加脚本的路径。重启，OK

（可以试试停用插件、再启用），

这里有个问题，注意脚本放的位置，如果像 IFTTT 有复杂的分支，分别是“启动”、“关闭”，如果把 setsid ….. & 放最后会，会产生两个进程。。因此可以只放到 start 里面

然后通过 QNAPCloud 进入后，启动 IFTTT 即可

或者通过判断进程是否存在来实现, processString改成你的进程名

#!/bin/sh
ps -fe|grep processString |grep -v grep
if [ $? -ne 0 ]
then
echo "start process....."
else
echo "runing....."
fi
