  • nodejs

Node.js是一个Javascript运行环境(runtime)。实际上它是对Google V8引擎进行了封装。V8引 擎执行Javascript的速度非常快，性能非常好。Node.js对一些特殊用例进行了优化，提供了替代的API，使得V8在非浏览器环境下运行得更好。 [1] Node.js是一个基于Chrome JavaScript运行时建立的平台， 用于方便地搭建响应速度快、易于扩展的网络应用。Node.js 使用事件驱动， 非阻塞I/O 模型而得以轻量和高效，非常适合在分布式设备上运行数据密集型的实时应用。

安装 node 和 npm

Node 有很多种安装方法，最简单的是下载各种OS对应的安装包。以及以下方法：

通过包管理器

https://nodejs.org/en/download/package-manager/

在 Ubuntu 下安装

$ sudo apt-get install -y build-essential

# 下面是装的 6.x 如果需要 4.x ，修改数字即可
# 现在最新版到 12 了！  不过这篇文章的方法还是一样的
$ curl -sL https://deb.nodesource.com/setup_6.x | sudo -E bash -
$ sudo apt-get install -y nodejs

由于我公司网络有问题（全部强制代理），导致一直安装的都是老版本，无法使用新的 apt source 地址。 可以用 snapd 来安装

sudo apt install snapd

sudo snap install node --channel=12/stable --classic

# 注意要把 /snap/bin 放到环境变量
vi ~/.bashrc  # 或者 .zshrc

export PATH=/snap/bin:$PATH

在 CentOS 下使用 yum 安装

以下是安装 6.x 版本，如果要 7.x 或者 0.10.x ，需要修改对应的版本号

注意 CentOS6.8 32 bit只能支持到 4.x 以下

可以在这里看到所有的版本支持https://github.com/nodesource/distributions#enterprise-linux-based-distributions

# 安装必备的库
yum install gcc-c++ make

# 或者直接安装开发者工具包: 
yum groupinstall 'Development Tools'

#下载自动解决rpm依赖的脚本，并安装 NodeJS (默认安装 yum -y install nodejs， 提示仓库中没有这个包)
curl --silent --location https://rpm.nodesource.com/setup_10.x | bash -
yum -y install nodejs

npm install -g yarn


# 测试一下是否安装成功
node -v
npm -v

# 如果失败，可以用下面的命令来安装
yum install nodejs npm --enablerepo=epel

NPM 包管理器

全局安装

在项目中安装一个包

列出全局安装的包

npm list -g --depth 0

检查全局包与升级

#更新全局包命令 npm install -g <package>:
npm install -g jshint

#查找过期包
npm outdated -g --depth=0

#更新所有全局包
npm update -g

删除包

删除掉项目里的包

带 –save 或 –save-dev 参数表示同时从 package.json 里删掉引用的那行

npm uninstall --save lodash

修改默认的源为淘宝 NPM 镜像

因为运行 npm 命令后，要从国外服务器上拉文件，这个速度非常慢，而且可能中断，因此修改源为国内的。

淘宝 NPM 镜像说明

我没有使用 cnpm 别名的方法，而是修改了.npmrc文件

# 最快的办法
npm config set registry https://registry.npm.taobao.org

# 相当于新建一个文件
$ sudo vi ~/.npmrc
# 添加
registry = https://registry.npm.taobao.org

#配置后可通过下面方式来验证是否成功
npm config get registry
#或
npm info express

问题

解决办法

npm config set strict-ssl false

yarn config set "strict-ssl" false

快速入门

HTTP服务器

var http = require('http');
http.createServer((req, res) => {
	res.writeHead(200, {'Content-Type': 'text/plain'});
	res.end('Hello world');
}).listen(3000, '127.0.0.1', ()=>{
	console.log('running server');
});

当调用 createServer 的时候， 我们传了一个匿名函数作为参数。 此函数绑定在新创建服务器的事件监听器上进行 request 事件处理 我们称这类方法为回调（ callback）。 因为每当一个事件发生时， 我们将回调监听此事件的所有函数

关于 node 的版本

注意，除非必要，请不要随意在生产环境升级大版本。

例如一次，之前在 Node8 上正常 npm install的 sharp 库（依赖 node-libcurl 和 node-pyg-?不记得了）。 在升级到 Node 10 后，突然系统无法启动了，报错说缺少什么。。。版本不对， 于是运行 npm install想更新重新安装 node_modules 里的库文件，却提示 node-libcurl 的下载文件 404 ,怎么都不行。

只能恢复到 Node8 再 npm install 就好了

部署：Nginx 代理 node 应用程序

假设 node 程序运行在 3000 端口（通过 node 命令，pm2 等方法启动）

修改 nginx 配置，添加以下内容，重启 nginx 即可。

upstream nodejs{
      server 127.0.0.1:3000;
      keepalive 64;
  }
  
server {
         listen 80;
         server_name www.example.org example.org;
         location / {
              proxy_set_header X-Real-IP $remote_addr;
              proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
              proxy_set_header Host  $http_host;
              proxy_set_header X-Nginx-Proxy true;
              proxy_set_header Connection "";
              proxy_pass      http://nodejs;
         }
  }
