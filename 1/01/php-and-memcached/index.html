PHP 与 Memcached 缓存

  php  评论 字数统计: 1485字 阅读时长: 3分

安装 Memcached 系统服务与启动

mac

$brew install memcached

$which memcached
/usr/local/bin/memcached

# 手动启动服务
$/usr/local/bin/memcached -d

# 开机启动

Linux

安装 PHP 扩展

首先运行 phpinfo()。

如果没有搜到memcache关键词的信息，说明未安装 memcached 扩展

extension_dir

在phpinfo()的结果中找到extension_dir参数对应的目录，可能是

/usr/lib/php/extensions/no-debug-non-zts-20131226/

列出里面的文件看看，默认是没有 memcache.so 的

首先安装依赖库

这个库据说能够让 php 能更高效地使用 memcached 服务。

$brew install libmemcached

下载扩展包

在这个页面里： http://pecl.php.net/packages.php

搜索“memcached”

注意我们需要的是第一个 memcached PHP extension for interfacing with memcached via libmemcached library，就是和前面的依赖库“libmemcached”结合一起才能使用的。

第二个没有“d”，应该是可以独立使用，不需要“libmemcached”

在打开的列表里点击 memcahce-[version].tar.gz 下载。 要注意版本号，3.x以上需要 PHP7 才能使用，我的是 PHP5.6，下载 2.2.0 就好。

点开“Version”的名称后看到一句话，说明了 memcached 的用途

Description This extension uses libmemcached library to provide API for communicating with memcached servers.

安装扩展

我们这里使用phpize命令来安装。这个命令是 PHP 运行环境安装后就自动包含了的。让我们可以对 PHP 动态增加新的扩展，而不需要重新编译 PHP。

一般搭配php-config命令一起使用。

# 看看这个命令在哪里
$which phpize
/usr/local/bin/phpize

# 还有 php-config 命令
$which php-config
/usr/local/bin/php-config

# 解压并进入扩展包目录
$tar vxzf memcached-2.2.0.tgz  
$cd memcache-2.2.0  

# 运行 phpize，大概是进行扫描和预处理，然后看到如下的结果
$phpize  
Configuring for:
PHP Api Version:         20131106
Zend Module Api No:      20131226
Zend Extension Api No:   220131226

# 可以编译了，会为 PHP 增加 memcache 扩展，注意 --with-php-config 的路径根据你自己的情况进行修改（查看前面）
$./configure --enable-memcached --with-php-config=/usr/local/bin/php-config
$make
$make install

成功后会提示如下，记下这个路径，等下就要使用了

Installing shared extensions: /usr/local/Cellar/php56/5.6.30_6/lib/php/extensions/no-debug-non-zts-20131226/

打开这个目录，会看到里面有个文件memcached.so

注意：可能需要把生成的 memcached.so 复制到正确的目录下

如上所示，我在编译后，告诉我这个文件生成后

/usr/local/Cellar/php56/5.6.30_6/lib/php/extensions/no-debug-non-zts-20131226/

目录下。

你还记得最开始我们通过phpinfo()命令看到，extension_dir目录是

/usr/lib/php/extensions/no-debug-non-zts-20131226/

两个目录不一致。所以需要把这个memcached.so文件，手动复制到 phpinfo() 显示的 extension_dir 目录里。这样后面启用扩展的时候，就不用指定很长的路径了。

当然这一步可以忽略，只不过下一步中，你需要手动把完整路径补上。

启用扩展

目前我们的 PHP 还不知道去哪里加载 memcache 扩展，需要修改 php.ini文件，在最后一行添加

extension = memcached.so

注意，php.ini 也需要在 phpinfo()里找，有时候系统里会有多个 php.ini 文件（特别是你用 brew 和 编译方法多次安装、卸载 php 的话）。当修改后重启php-fpm无效，一般是改错了文件。

如果你遇到

cp: /usr/lib/php/extensions/no-debug-non-zts-20131226/memcached.so: Operation not permitted

问题，可以指定完整路径。

重启php-fpm

$brew services restart php56
$sudo killall php-fpm
$sudo php-fpm

重新看看phpinfo()，已经出现了扩展的信息

测试

写一个 test-memcached.php 文件

$memcache = new Memcached();
$memcache->addServer('localhost', 11211);

$value = $memcache->get('key');

if(!$value){
  $memcache->set('key', 'this is the value in memcache');
  echo 'no value in memcahe';
}else{
  echo $value;
}

在这里，我遇到一个很有意思，也可能会有很多人遇到的问题。 因为我没有把 memached 服务放到开机启动，有一次重启后，我发现这段 php 无论如何也没办法输出、写入数据到 memcached 缓存。

MySQL 的mysql_connect()函数可以跟一个die("MySQL server connect error") 来报错，而 memcached 是没有类似的返回的。

即使没有 memcached 服务可以连，照样会正常运行，只不过无法处理数据而已。

我翻了翻 http://php.net/manual/en/book.memcached.php 提供的函数，测试 getServerList(), 发现根本不受 memcached 服务没启动的影响。

所以一定要注意，在代码中，做好向下兼容（如发现数据为空，改用文件缓存）或者手动写报错处理来检查 memcached 没有正常运行。

一些可能问题

运行任何php命令的时候，提示“Module ‘XXX’ already loaded in Unknown on line 0”

例如

PHP Warning: Module ‘memcached’ already loaded in Unknown on line 0

运行php --ini 提示了 php.ini路径

Loaded Configuration File: /usr/local/etc/php/5.6/php.ini

但是由于我们还是用了 /etc/php.ini，造成了混乱，所以编辑/usr/local/etc/php/5.6/php.ini文件，去掉和 /etc/php.ini 里重复的 extension 配置即可
