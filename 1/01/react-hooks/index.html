关于 Hook

React 16.7.0-alpha 版本推出的新特性

React Hooks 要解决的问题是状态共享，是继 render-props 和 higher-order components 之后的第三种状态共享方案，不会产生 JSX 嵌套地狱问题。

状态共享可能描述的不恰当，称为状态逻辑复用会更恰当，因为只共享数据处理逻辑，不会共享数据本身。

Hooks are a new addition in React 16.8. They let you use state and other React features without writing a class.

A Hook is a special function that lets you “hook into” React features. (如 setState,is a Hook that lets you add React state to function components.)

为何要用 hook?

因为之前写了很多 function component,(也成为 stateless component) 如果需要添加 state, 只能把他们转换成 class。 现在有了 Hook, 就不需要了~ 可直接在 function component 内部使用 Hook 管理 state

之前状态管理的一些方案 redux mobx 等

就是要解决相邻组件通信问题.

如果用“状态提升”

  1. 子组件的更新，会触发负责下发状态的父组件的整体更新(Context 也有这个问题),产生很多 PureComponent shouldComponentUpdate 等
  2. 逻辑较多

所以"状态管理"会更好。

使用mobx而不再使用setState原因有如下几个（个人理解）

1 setSate 不是同步操作，当我们通过setstate修改state数据后，后面的逻辑立即取值是旧值，state的新值在下一次render时获取到。 2 如果将所有的数据都保存在state中，那么例如定时器，异步请求返回等于UI渲染实际无关的数据也同样会导致组件执行不必要的重新渲染。 3 setState 不适合管理全局的状态。

为什么要用状态管理？ 把「与服务器交互」和「操作数据」的逻辑，从组件中提取出去，组件只用接收处理好的数据即可。这就相当于分离了个 Service 层出去，很好的开发模式。

什么样的状态管理

模块：推荐按照路由入口还划分。 每个状态管理的模块 - model. 与路由入口组件绑定起来，每个入口组件及其子组件，整体对应一个 model。

下面是一个使用了 useState 的组件

// import {useState}
import React, { useState } from 'react';

function Example() {
  // Declare a new state variable, which we'll call "count"
  // 用到了 js 的 数组解构？
  const [count, setCount] = useState(0);

  return (
    <div>
        <!-- 直接用 {count}  相当于class 里的 {this.state.count} -->
      <p>You clicked {count} times</p>

      <!-- 相当于class 里的 {() => this.setState({ count: this.state.count + 1 })} -->
      <button onClick={() => setCount(count + 1)}>

      </button>
    </div>
  );
};

hooks 只能在 function component 内使用， 无法用在 class 内

const Demo = (props)=> {
    // 可以
    const [count, setCount] = setState(0);
    return <div />;
};


function Demo(props) {
    //可以
    const [count, setCount] = setState(0);
    return <div />
}

class Demo extends React.Component {
    // 不可以,因为是 class
    const [count, setCount] = setState(0);

    render() {
        return <div />
    }
}

with clearup

大部分时候不需要清理，但是部分情况需要。 例如订阅外部数据源，使用 effect 可能会引起内存泄露

在一个 class 的组件中 componentDidMount 中订阅， componentWillUnmount 中清理掉

class FriendStatus extends React.Component {
  constructor(props) {
    super(props);

    // 好友是否上线
    this.state = { isOnline: null };

    // 处理变化
    this.handleStatusChange = this.handleStatusChange.bind(this);
  }


//组件加载完，调用ChatAPI.subscribeToFriendStatus， 应该会把持续请求得到的好友在线状态数据，不停地返回给 this.handleStatusChange 来设置 state
  componentDidMount() {
    // 这里调用一次 document.title
    document.title = `You clicked ${this.state.count} times`;

    ChatAPI.subscribeToFriendStatus(
      this.props.friend.id,
      this.handleStatusChange
    );
  }


  componentDidUpdate() {
    // 这里也调用了， 因为是 update
    document.title = `You clicked ${this.state.count} times`;
  }

// 组件 umount 的时候，把这个持续的订阅给取消掉。
  componentWillUnmount() {
    ChatAPI.unsubscribeFromFriendStatus(
      this.props.friend.id,
      this.handleStatusChange
    );
  }

//从订阅拿到的数据，(好友状态)，放到这个函数来处理  
  handleStatusChange(status) {
    this.setState({
      isOnline: status.isOnline
    });
  }

  render() {
    if (this.state.isOnline === null) {
      return 'Loading...';
    }
    return this.state.isOnline ? 'Online' : 'Offline';
  }
}

来看看 effect hook 的版本，简单多了 (出现了两次 useEffect())

import React, { useState, useEffect } from 'react';

function FriendStatus(props) {

    // 因为 count 这个 state 不需要clear up, 所以没有 return . (也就是说 document.title  = xxx 代码不出现在 componentWillUnmount 生命周期)
    const [count, setCount] = useState(0);
  useEffect(() => {
    document.title = `You clicked ${count} times`;
  });


  const [isOnline, setIsOnline] = useState(null);

  useEffect(() => {
    function handleStatusChange(status) {
      setIsOnline(status.isOnline);
    }
    ChatAPI.subscribeToFriendStatus(props.friend.id, handleStatusChange);
    
    // Specify how to clean up after this effect: 就是在 umount 里的代码
    return function cleanup() {
      ChatAPI.unsubscribeFromFriendStatus(props.friend.id, handleStatusChange);
    };
  });

  if (isOnline === null) {
    return 'Loading...';
  }
  return isOnline ? 'Online' : 'Offline';
}

由于 effect 每次render 都会运行， 所以 React会在下次运行 effect 之前清除之前渲染的 effect。

详解前面提到的 每次清理 和 内存泄露问题, 以及为什么每次 update 都会运行 effect

首先上面的例子里 componentDidMount（运行一次） componentWillUnmount（运行一次） 中分别订阅/取消订阅, 传入参数 props.friend.id

代码略

如果 props.friend.id 发生变化, 会发生什么事情？

例如一开始是 100， 然后变成了 200

  • 首先组件会一直显示 100 这个用户的数据
  • 然后 componentWillUnmount 会把 200 传进去取消订阅，导致之前的 100 这个用户的数据一直无法注销，产生内存泄露

为了解决这个问题，在 class 类型 component 的 componentDidUpdate 里，要手动先取消订阅旧的 friend.id 然后重新订阅新的 friend.id

这就是我们经常会在 React 应用中，因为忘记 componentDidUpdate 更新数据而产生的常见 bug

componentDidUpdate(prevProps) {
    // Unsubscribe from the previous friend.id
    ChatAPI.unsubscribeFromFriendStatus(
      prevProps.friend.id,
      this.handleStatusChange
    );
    // Subscribe to the next friend.id
    ChatAPI.subscribeToFriendStatus(
      this.props.friend.id,
      this.handleStatusChange
    );
}

而使用 effect hook, 这个操作是自动的。useEffect 会在应用 next effect 之前，自动清理之前的 effect,

// Mount with { friend: { id: 100 } } props
ChatAPI.subscribeToFriendStatus(100, handleStatusChange);     // Run first effect

// Update with { friend: { id: 200 } } props
ChatAPI.unsubscribeFromFriendStatus(100, handleStatusChange); // Clean up previous effect
ChatAPI.subscribeToFriendStatus(200, handleStatusChange);     // Run next effect

// Update with { friend: { id: 300 } } props
ChatAPI.unsubscribeFromFriendStatus(200, handleStatusChange); // Clean up previous effect
ChatAPI.subscribeToFriendStatus(300, handleStatusChange);     // Run next effect

// Unmount
ChatAPI.unsubscribeFromFriendStatus(300, handleStatusChange); // Clean up last effect

在默认情况下确保了一致性，并防止了由于缺少更新逻辑而在类组件中常见的bug

因为 effect 会自动 clear, 产生了性能问题，如何优化？

例如，上面的计数器的例子， 如果每次得到的新的 state.count 的值都和之前一致，就没有必要重新 render 组件！

componentDidUpdate(prevProps, prevState) {
  if (prevState.count !== this.state.count) {
    document.title = `You clicked ${this.state.count} times`;
  }
}

effect hook 版本就是给 useEffect 传入第二个参数，表示只有当 count 被 change 的时候才会 re-run 这个 effect.

useEffect(() => {
  document.title = `You clicked ${count} times`;
}, [count]); // Only re-run the effect if count changes

当然，对于需要 clear up 的场景同样有效

useEffect(() => {
  function handleStatusChange(status) {
    setIsOnline(status.isOnline);
  }

  ChatAPI.subscribeToFriendStatus(props.friend.id, handleStatusChange);
  return () => {
    ChatAPI.unsubscribeFromFriendStatus(props.friend.id, handleStatusChange);
  };
}, [props.friend.id]); // Only re-subscribe if props.friend.id changes

rules

仅仅在 top level 调用 hook, Don’t call Hooks inside loops, conditions, or nested functions.

只在 function component/custom hooks 上调用 hooks（这个前面提到了）

举个例子，解释为何不能在 if loop 等里面使用 hook

function Form() {
  // 1. Use the name state variable
  const [name, setName] = useState('Mary');

  // 2. Use an effect for persisting the form
  useEffect(function persistForm() {
    localStorage.setItem('formData', name);
  });

  // 3. Use the surname state variable
  const [surname, setSurname] = useState('Poppins');

  // 4. Use an effect for updating the title
  useEffect(function updateTitle() {
    document.title = name + ' ' + surname;
  });

  // ...
}

发现了两个 useState() ，表示调用两次 hook , 那 React 怎么知道哪个 state 对应哪个 useState() ?
答案就是根据 调用的顺序

// ------------
// First render
// ------------
useState('Mary')           // 1. Initialize the **name** state variable with 'Mary'
useEffect(persistForm)     // 2. Add an effect for persisting the form
useState('Poppins')        // 3. Initialize the **surname** state variable with 'Poppins'
useEffect(updateTitle)     // 4. Add an effect for updating the title

// -------------
// Second render
// -------------
useState('Mary')           // 1. **Read** the name state variable (argument is ignored)
useEffect(persistForm)     // 2. Replace the effect for persisting the form (每次都会 replace, 因为没有传入第二个参数)
useState('Poppins')        // 3. Read the surname state variable (argument is ignored)
useEffect(updateTitle)     // 4. Replace the effect for updating the title

// ...

只要钩子调用的顺序在多次渲染之间是相同的，React就可以将一些本地状态与它们每个关联起来。

但是，如果我们将一个钩子调用(例如，persistForm效果)放在一个条件中，会发生什么呢?

// 🔴 We're breaking the first rule by using a Hook in a condition
  if (name !== '') {
    useEffect(function persistForm() {
      localStorage.setItem('formData', name);
    });
  }

当第一次 render 的时候， useEffect(persistForm) 这个 hook 会被运行。(name !=== '’ 等于 true)

但是第二次 render 的时候，如果用户清空了 form ! 导致 name 变成空值。 这次渲染就会跳过这个 hook, 顺序就变了

useState('Mary')           // 1. Read the name state variable (argument is ignored)
// useEffect(persistForm)  // 🔴 This Hook was skipped!
useState('Poppins')        // 🔴 2 (but was 3). Fail to read the surname state variable
useEffect(updateTitle)     // 🔴 3 (but was 4). Fail to replace the effect

如果一定要有条件地运行 effect, 必须放到 hook 里面. 保证定义和运行hook 的顺序一致

useEffect(function persistForm() {
    // 👍 We're not breaking the first rule anymore
    if (name !== '') {
      localStorage.setItem('formData', name);
    }
  });

使用 use-global-hook 包

https://juejin.im/post/5d783fca6fb9a06af50ff577

最小案例

import React from 'react';

//1 导入
import globalHook from 'use-global-hook';

//2 初始化 state
const initialState = {
  counter: 0,
};

// 3 actions. 注意 store.state 就是“全局”的 state
const actions = {
  // 方法传入 store 和 真正的参数
  addToCounter: (store, amount) => {
    const newCounterValue = store.state.counter + amount;
    // 修改后重新保存回来
    store.setState({ counter: newCounterValue });
  },
};

// 4 定义 useGlobal，调用 globalHook, 传入 initialState 和 actions
const useGlobal = globalHook(React, initialState, actions);

const App = () => {

  // 调用 useGlobal() 得到 globalState 和 globalActions
  const [globalState, globalActions] = useGlobal();

  return (
    <div>
      <p>
        counter:
        // globalState 读取 state
        {globalState.counter}
      </p>

      // globalActions 调用 action
      <button type="button" onClick={() => globalActions.addToCounter(1)}>
        +1 to global
      </button>
    </div>
  );
};

export default App;

使用全局状态之前映射它的一个子集。只有更新了子集，组件才会重新呈现。

在 components/CounterA.js 中，只读取/设置全局 state 的一部分。 点 button ，给 A +1 复制1000个 CounterB 后，任意一个 B 上点按钮， 全部 B 都 +1（他们都用的全局state里的 counterB ）

index.js

import React, { useState } from "react";
import ReactDOM from "react-dom";

import "./styles.css";
import { renderTime } from "./utils";
import CounterA from "./components/CounterA";
import CounterB from "./components/CounterB";

function App() {

  const [children, setChilds] = useState([]);

  const addCounterA = () => {
    const counter = <CounterA />;
    const newChildren = [...children, counter];
    setChilds(newChildren);
  };

  const addCounterB = () => {
    const counter = <CounterB />;
    const newChildren = [...children, counter];
    setChilds(newChildren);
  };

  const addManyCounterB = count => {
    const newChildren = [...children];
    for (let index = 0; index < count; index++) {
      newChildren.push(<CounterB />);
    }
    setChilds(newChildren);
  };
  return (
    <div className="App">
      <h1>I am the parent component</h1>
      <h3>{renderTime()}</h3>
      <button onClick={addCounterA}>Add CounterA component</button>
      <button onClick={addCounterB}>Add CounterB component</button>
      <button onClick={() => addManyCounterB(10000)}>
        Add 10 000 CounterB components
      </button>
      <div className="container">{children}</div>
    </div>
  );
}

const rootElement = document.getElementById("root");
ReactDOM.render(<App />, rootElement);
