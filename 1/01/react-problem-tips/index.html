在 React 里读取配置参数

使用 axios 等网络请求时，需要一个 “root url”， 这个地址是变化的，本地开发和线上环境可能是两个不同的地址

那么如何把它做成外部配置参数，在 react 代码里引用呢?

根据 https://stackoverflow.com/questions/30568796/how-to-store-configuration-file-and-read-it-using-react

  1. webpack 里可以用 external

  2. 如果使用 create-react-app 创建的应用，可以直接使用 .env 文件

create a file named .env in the root of project.

define permanent environment variables.

REACT_APP_SECRET_CODE=abcdef

You must create custom environment variables beginning with REACT_APP_

What other .env files can be used?

.env files should be checked into source control (with the exclusion of .env*.local).

  • .env: Default.
  • .env.local: Local overrides. This file is loaded for all environments except test.
  • .env.development, .env.test, .env.production: Environment-specific settings.
  • .env.development.local, .env.test.local, .env.production.local: Local overrides of environment-specific settings.

Files on the left have more priority than files on the right:

  • npm start: .env.development.local, .env.development, .env.local, .env
  • npm run build: .env.production.local, .env.production, .env.local, .env
  • npm test: .env.test.local, .env.test, .env (note .env.local is missing)

you can access it from the component with

process.env.REACT_APP_SECRET_CODE

问题， .env.production 没有覆盖 .env.local 的配置

向子组件传入 props， 以及在组件的某个方法中再传入 props (以及 Upload 组件处理文件是否全部上传完毕)

标题有点绕，事情是这样子的：

使用 antd 开发项目时，有一个 Steps 组件，每完成一步，就加载一个对应的 Content 组件(最重要的是一个上传文件组件)。一开始使用了几个 Functional Component, 没有传入任何 props。

const steps = [
  {title: 'Ready', content: <ReadyComponent />},
  {title: 'Upload .s2p files', content: <UploadComponent />},
  {title: 'Beautify format of files', content: <BeautifiedComponent />},
  {title: 'Merge to single file', content: <MergedComponent />}
];

注意那个{...uploadProps}, 这个对象主要是设置 Dragger 组件的属性的。

const uploadProps = (props) => {
  return {
    name: 'file',
    multiple: true,
    action: '/api/hub/tools/uploadS2P?code=' + code,
    onChange(info) {
      const status = info.file.status;
      if (status !== 'uploading') {
        console.log(info.file, info.fileList);
      }
      if (status === 'done') {
        message.success(`${info.file.name} file uploaded successfully.`);

        uploadedFiles.push(info.file.name);
      } else if (status === 'error') {
        message.error(`${info.file.name} file upload failed.`);
      }

    }
  }
};

const UploadComponent = ()=>(
  <Row type="flex" justify="center" align="top" style={{minHeight: '400px'}}>
    <Col span={8} style={{ textAlign: 'center' }}>
      <Dragger {...uploadProps}>
        <p className="ant-upload-drag-icon">
          <Icon type="inbox" />
        </p>
        <p className="ant-upload-text">Click or drag file to this area to upload</p>
        <p className="ant-upload-hint">Support for a single or bulk upload. Strictly prohibit from uploading company data or other band files</p>
      </Dragger>
    </Col>
  </Row>
)

在 Steps 切换时，直接加载组件显示出来（具体看官网的案例）

{steps[this.state.current].content}

好了，现在情况有变。要求：在上传多个文件时，确认多个文件全部完成后，才能按下一步

好吧，那只能先把按钮隐藏了。如果添加一个外部变量，在 render()里进行判断。

let isUploaded = false;
let uploadedFiles = [];

// 在 Upload 组件的 onChange 里添加
if (info.fileList.length === uploadedFiles.length) {
    isUploaded = true;
}

// 在按钮的代码处
let allowNext = false;

if (this.state.current < steps.length - 1){
  allowNext = true;

  if((this.state.current === 1) && (isUploaded === false)) {
    allowNext = false;
  } 
}

会发现，文件全部上传完后，这个下一步按钮照样不显示。 因为这个变量isUploaded并没有触发按钮组件的 allowNext变化

要改成用 setState 来修改变量。这样才能让变量触发按钮的显示/隐藏

先在按钮组件内增加一个方法，以及这个 state 变量的初始值

constructor(props) {
    super(props);
    this.state = {
      current: 0,
      isUploaded: false
    }
  }

finishUpload() {
    this.setState({isUploaded: true})
}

怎么把这个方法传入到 Upload 组件去呢？目前是个无状态函数，而且是直接显示啊。

增加一个方法wrapStepContent，接收两个参数，一个是当前第几步骤，一个是 props(注意，这是关键)

const wrapStepContent = (index, props) => {
  const components = [
    <ReadyComponent {...props} />,
    <UploadComponent {...props} />,
    <BeautifiedComponent {...props} />,
    <MergedComponent {...props} />
  ];

  return components[index];
}

然后把前面那个直接显示组件 {steps[this.state.current].content} 改成先传入一些值，然后返回带props的组件。 注意参数{finishUpload: ()=>this.finishUpload()} 就是往子组件里面传入这个方法，让子组件能调用，进而修改父组件的 state 变量

{wrapStepContent(this.state.current, {finishUpload: ()=>this.finishUpload()})}

把UploadComponent要改成能接受参数的形式

const UploadComponent = (props)=>(

最有意思的来了

因为 antd Upload 组件的传入的那个属性里，不知道刚传入的 props, 如何在 onChange 里调用？

下面的代码，把uploadProps 从一个对象const uploadProps = {}，改成一个可接收 props 参数的函数，在里面调用 props.finishUpload();

const uploadProps = (props) => {
  return {
    //...
    onChange(info) {
      //....
      
      if(info.fileList.length === uploadedFiles.length) {
        props.finishUpload();
      }
    }
  }
};

在组件调用的时候，（1）传入props到uploadProps函数，得到修改后 的uploadProps，（2）再与 props 一起合并（Object.assign）（3）最后一起展开为组件 Dragger 的属性。

const UploadComponent = (props)=>(
    //...
    <Dragger {...Object.assign(uploadProps(props), props)}>

现在可以回顾整个过程

  1. 在”顶级“组件里 定义 state 变量，和修改upload 状态的方法 （upload 状态）
  2. 把修改upload 状态的方法通过 props 传给 UploadComponent
  3. UploadComponent 组件把 props 同时传给子组件 Dragger，以及 Dragger 的一组内部属性，共享修改upload 状态的方法

简单来说其实就是跨父子组件的 props 传输与 子组件调用父组件方法修改父组件的属性。

难点就是把 props 除了传入组件，还传入了组件的另外一个属性/方法。(利用对象的合并与展开)

react-hot-loader 热更新 (create-react-app)

npm i react-hot-loader -D

在webpack.config.dev.js中entry第一行增加`'react-hot-loader/patch’

如果有.babelrc文件就增加"plugins": ["react-hot-loader/babel"]

如果是create-react-app项目就在webpack.config.dev.js文件module-rules-oneOf下新增代码"plugins”: [“react-hot-loader/babel”] 大约在143行号

然后在代码的入口entry 根节点包含热更新

//....
import App from './App';
import registerServiceWorker from './registerServiceWorker';

import { AppContainer } from 'react-hot-loader'; // 包裹根节点，想要渲染的内容


const root = document.getElementById('root')
const render = Component => {
    ReactDOM.hydrate(
        <AppContainer>
            <Component />
        </AppContainer>,
        root
    )
}
render(App)

if (module.hot) {
    module.hot.accept('./App', () => { // 当我们热更新的代码出现的时候，把App重新加载
        const NextApp = require('./App').default //因为在App里使用的是export default语法，这里使用的是require,默认不会加载default的，所以需要手动加上
        render(NextApp) // 重新渲染到 document 里面
    })
}
// ReactDOM.render(<App />, document.getElementById('root'));
registerServiceWorker();
