React 的 render props

React 16+

render props是一个组件间共享代码逻辑的小技巧, 通过props传递函数来实现。有许多库(比如React Router, React Motion)都使用了这个技巧。

组件有一个叫做render的prop(属性), 值是一个返回React元素的函数, 在组件内部调用这个函数渲染组件。(为了给原有组件“注入”其它组件的代码)

React不是原本就支持组件嵌套和传值吗？

class ComponentB extends React.Component() {
    render() {
        return (
            <div>Hello, I'm Component B</div>
        )
    }
}


export default class ComponentA extends React.Component {

    constructor(props){
        super(props)
    }
    
    render() {
        return(
            <div>
                <ComponentB />
            </div>
        )
    }

}

为啥要通过这么麻烦的方式给一个组件“注入”另一个组件的代码?

如果你一个组件不知道自己渲染什么东西，或者说你一个组件的基础功能是提供”可变数据源“，具体展示UI可以从外部注入，那么就可以用这个技术了.

(想想看，我们可以提供一个组件，它的 render props 可以根据需要，分别渲染 React 和 ReactNative 的组件，从而实现一些业务逻辑代码复用，同时又在 UI 层面根据平台有差异)

可变数据源组件

例如 一个Mouse组件 提供”可变数据源“， 作为一个基础数据的提供者它并不知道自己要渲染什么。而我们可用鼠标位置做许多用意思的东西，比如说跟随鼠标移动的动画，拖动页面元素移动。。。

class Mouse extends React.Component {
  constructor(props) {
    super(props);
    this.handleMouseMove = this.handleMouseMove.bind(this);
    this.state = { x: 0, y: 0 };
  }

  handleMouseMove(event) {
    this.setState({
      x: event.clientX,
      y: event.clientY
    });
  }

  render() {
    return (
      <div style={{ height: '100%' }} onMouseMove={this.handleMouseMove}>

        {/* ...but how do we render something other than a <p>? */}
        <p>The current mouse position is ({this.state.x}, {this.state.y})</p>
        {this.props.render(this.state)} // 这个比较关键
      </div>
    );
  }

特别注意上面的代码中 this.props.render(this.state) 这个组件的属性render，它就是一个函数，在这里渲染UI。Mouse 只负责把“可变数据源”作为参数给render函数，至于渲染的内容是什么？那就不是Mouse组件能控制了。

注意，虽然都叫 render, 但它是 this.props 的属性函数，而不是 组件生命周期的 render 函数

被注入UI组件

定义另外一个使用数据源的组件 Cat， 让Cat图片跟随鼠标位置的变化而移动。（这个Cat组件就是我们希望渲染在页面的UI）

这里 this.props.mouse 就差不多能猜到，后面会向Cat组件注入一个属性mouse，然后 Cat内部就能得到它，并获取到它的数据。

class Cat extends React.Component {
  render() {
  
    const mouse = this.props.mouse;
    
    return (
      <img src="/cat.jpg" style={{ position: 'absolute', left: mouse.x, top: mouse.y }} />
    );
  }
}

下面就是“绑定”(调用) 的方式

class MouseTracker extends React.Component {
  render() {
    return (
      <div>
        <h1>Move the mouse around!</h1>
        <Mouse render={mouse => (
          <Cat mouse={mouse} />
        )}/>
      </div>
    );
  }
}

之前我们在Mouse的render函数（生命周期函数）里调用了组件Props上的render属性函数（自定义的，叫啥名儿都行），那么我们在用到Mouse组件的时候，给Mouse组件一个render属性函数，这个函数使用参数（鼠标位置，通过Mouse组件的state传过来）来初始化Cat组件，然后返回。那么这样一来Cat组件就内嵌在Mouse组件的render函数里了。

也即是说一直到这个时候，我们才知道要往Mouse组件里渲染什么UI。

用途

一般不会这么用，但当两个平级组件之间需要 单向依赖的时候就很有用了：同级组件A和B，A组件需要跟随B组件的内部状态来改变自己的内部状态，我们就说A依赖B。

双向依赖怎么办？ 双向依赖就把状态维护交给父组件。

注意

如果使用render Props技术的组件（也就是有一个render函数属性的组件）是继承自React.PureComponent 的，那么在props的浅比较看来，新老props总是不一样的，那么就会每次都重新渲染，性能开销比较大。

所以推荐用法如下：继承React.Component，并使用类函数(而不是使用匿名函数)

class MouseTracker extends React.Component {
  // Defined as an instance method, `this.renderTheCat` always
  // refers to *same* function when we use it in render
  renderTheCat(mouse) {
    return <Cat mouse={mouse} />;
  }

  render() {
    return (
      <div>
        <h1>Move the mouse around!</h1>
        <Mouse render={this.renderTheCat} />
      </div>
    );
  }
}
