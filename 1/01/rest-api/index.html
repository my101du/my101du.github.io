REST API

permalink: wordpress-rest-api tags:

  • wordpress

简介

https://developer.wordpress.org/rest-api/

访问下以下地址，返回 json 数据内容，确保 REST API 服务是打开的（有的插件可以把它关闭）

www.sample.org/wp-json/wp/v2/posts

以及带参数的

www.sample.org/wp-json/wp/v2/posts?per_page=8&page=1&orderby=date&order=desc

自定义返回的字段

有时候获取 blog 列表，只是为了展示标题，但一次把全部的field 都拿回来了，体积太大

如何限制

https://stackoverflow.com/questions/41573953/how-to-get-specific-data-from-wp-json-rest-api

https://github.com/WP-API/WP-API/issues/446 (里面有 https://wordpress.org/plugins/wp-rest-api-filter-fields/ 等插件实现)

处理一些插件生成的 shortcode 代码

例如 visual composer 会得到 [vc_row] 这种短代码。

如何让它在服务器端就解析完毕，返回正常的 html 代码呢？

修改主题下的 functions.php 底部添加下面的代码。（可能还是会有部分的短代码会出问题）

/**
 * Modify REST API content for pages to force
 * shortcodes to render since Visual Composer does not
 * do this
 */
add_action( 'rest_api_init', function ()
{
   register_rest_field(
          'page',
          'content',
          array(
                 'get_callback'    => 'compasshb_do_shortcodes',
                 'update_callback' => null,
                 'schema'          => null,
          )
       );
});

function compasshb_do_shortcodes( $object, $field_name, $request )
{
   WPBMap::addAllMappedShortcodes(); // This does all the work

   global $post;
   $post = get_post ($object['id']);
   $output['rendered'] = apply_filters( 'the_content', $post->post_content );

   return $output;
}

如何把获取 css

// Register a new rest route
add_action( 'rest_api_init', 'my_rest_routes' );

function my_rest_routes() {

    register_rest_route(
        'wp/v2',
        '/output_css_uri/',
        [
            'methods'  => 'GET',
            'callback' => 'output_css_callback',
        ]
    );
}


// Callback function to output the css URI
function output_css_callback() {
    return get_stylesheet_uri();
}

用这个 http://example.com/wp-json/wp/v2/output_css_uri 获取 css 文件地址

(只能获得一个文件？) 并且打开看，里面内容很少，并不是“编译”后的完整站点样式文件

只能打开一个页面，打开开发者工具，打开 “network” 找到 css 把有关的文件地址复制到 react 应用 index.html 的 header 区域了

不过这样是影响“全局”样式了，希望只在部分页面里处理:
