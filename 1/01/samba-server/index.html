Samba文件共享服务

  samba 运维  评论 字数统计: 2064字 阅读时长: 5分

选择 NFS/Samba

碎文件多，nfs 有优势，否则，smb 兼容性好。

碎文件数量临界点，2w 个，具体可以自己谷歌国外 nfs 和 smb 的测评

Samba 服务

Samba 是在Linux和UNIX系统上实现SMB协议的一个免费软件，由服务器及客户端程序构成。Samba最大的功能就是可以用于Linux与windows系统直接的文件共享和打印共享，Samba既可以用于windows与Linux之间的文件共享，也可以用于Linux与Linux之间的资源共享。

Samba由两个主要程序组成，它们是smbd和nmbd。这两个守护进程在服务器启动到停止期间持续运行，功能各异。Smbd和nmbd使用的全部配置信息全都保存在smb.conf文件中。Smb.conf向smbd和nmbd两个守护进程说明输出什么以便共享，共享输出给谁及如何进行输出。

Samba提供了基于CIFS的四个服务：

1)、文件和打印服务

2)、授权与被授权

3)、名称解析

4)、浏览服务

https://www.tecmint.com/install-samba4-on-centos-7-for-file-sharing-on-windows/

samba server 基本配置

sudo yum install samba samba-client samba-common

# 防火墙
firewall-cmd --permanent --zone=public --add-service=samba
firewall-cmd --reload

sudo mkdir /home/share
sudo chmod -R 777 /home/share

sudo useradd smbuser

# 这就是关键了， 并不是 passwd  而是 smbpasswd
sudo smbpasswd -a smbuser

sudo vi /etc/samba/smb.conf

#找到 global 配置部分，添加一行
security=user

#来到最后，添加
[myshare]
path=/home/share
comment=my share directory
browseable=yes
writable=yes

[global]
    workgroup = WORKGROUP
    netbios name = centos
    security = user
[Anonymous]
    comment = Anonymous File Server Share
    path = /srv/samba/anonymous
    browsable =yes
    writable = yes
    guest ok = yes
    read only = no
    force user = nobody

重启服务

# 测试配置
testparm

# smb
systemctl start smb.service
systemctl enable smb.service

# nmb  干啥的？
systemctl start nmb.service
systemctl enable nmb.service

然后在客户端用 smbuser用户和对应的密码来连接共享目录（例如windows的 \192.168.0.x 和 OSX 的 cmd+k）

主机加入了一个域， 访问时无法连接

登录窗口选“更多选项”， 账号里开始打一个 \ 就不会自动读取到域了

性能调整

放在 [global]

# disable links and symbol links  不需要对符号链接也共享
follow symlinks = no
wide links = no

# enable some read/write tuning  启用读写缓存等等
read raw = yes
write raw = yes

aio read size = 16384
aio write size = 16384
write cache size = 262144

max xmit = 65536
large readwrite = yes

# Use sendfile for reading files efficiently:以下几个我测试貌似影响不大
use sendfile = yes
getwd cache = yes

# disable locking, because only 2 share can be written.
strict locking = no
fake oplocks = yes
oplocks = no

follow symlinks ，对我来说不需要，Windows好象不支持这种符号连接，我也不需要在共享里有这个东西，所以关闭

wide links，同上，不需要

aio read/write size，是和异步读写有关的，设大一点比较好

oplocks和locking这两个决定samba是否对文件进行lock测试，如果一个用户以write方式打开了这个文件，那么其它用户就 不能再以write打开，这两个东西很费内存，也同样会降低samba的性能。 另外，我有很多个share，其中只有两个是所有人都可以写的，其它的都只有一个用户能写，对于后者，不需要打开这个，因为共享设计就是别人无法写。 那么就在global里面把这几个和lock有关的参数关掉，在那两个所有人可以写的share再打开。

所有基它的参数，都是samba里面提到的performance tunning的参数我就不一个一个的说了，要了解它的可以去 http://us1.samba.org/samba/docs/man/Samba-HOWTO-Collection/  看 看

完整的参数 http://us1.samba.org/samba/docs/man/Samba-HOWTO-Collection/

NFS

如果 windows 下也用了samba, 因为无法修改端口，则端口映射无法使用。改成 NFS

服务端 CentOS

安装

# 客户端也是这个。。。
yum -y install nfs-utils nfs-utils-lib

systemctl enable nfs-server.service
systemctl enable rpcbind
systemctl enable nfs-lock
systemctl enable nfs-idmap  


systemctl start nfs-server.service
systemctl start rpcbind
systemctl start nfs-lock
systemctl start nfs-idmap 

# 修改目录权限(或者把某个用户加入到组。)
chown nfsnobody:nfsnobody /var/nfs

chmod -R 777 /var/www

配置 /etc/exports 文件,如

# [要共享的目录] [共享给客户端IP(选项)] [客户端2 ...  可以是一个 ip 或网段 全部选项 https://linux.die.net/man/5/exports
/var/www 192.168.100.222(rw,async,no_subtree_check,no_root_squash)
/var/nfs        192.168.1.0/24(rw,sync,no_subtree_check)

# 修改完毕后，要重载配置
exportfs -a

防火墙：

如果没有配置好，会出现 mount.nfs: Connection timed out

因为 nfs 除了默认开放端口 111 2049 还有一些随机端口，很不方便，因此配置NFS服务使用固定端口。

编辑 /etc/sysconfig/nfs 配置文件

# TCP port rpc.lockd should listen on.
LOCKD_TCPPORT=32803
# UDP port rpc.lockd should listen on.
LOCKD_UDPPORT=32769
 
MOUNTD_PORT=892
STATD_PORT=662

在Linux NFS服务器上执行以下命令获得NFS端口信息

rpcinfo -p


#....略其他 111  2049 892  662 等
100021    4   udp  32769  nlockmgr
100021    1   tcp  32803  nlockmgr

找到后，运行如下

firewall-cmd --permanent --add-port=2049/tcp
firewall-cmd --permanent --add-port=2049/udp
firewall-cmd --permanent --add-port=111/tcp
firewall-cmd --permanent --add-port=111/udp
firewall-cmd --permanent --add-port=892/tcp
firewall-cmd --permanent --add-port=892/udp
firewall-cmd --permanent --add-port=662/tcp
firewall-cmd --permanent --add-port=662/udp
firewall-cmd --permanent --add-port=32803/tcp
firewall-cmd --permanent --add-port=32769/udp

firewall-cmd –reload

客户端

linux

apt-get install nfs-common portmap

# or centos
yum -y install nfs-utils nfs-utils-lib

# 挂载 mount -t nfs
mount 12.34.56.789:/home /mnt/nfs/home
mount 12.34.56.789:/var/nfs /mnt/nfs/var/nfs


df -h

# 检查已经 mount 的
mount

# 启动自动挂载,添加内容
vi /etc/fstab

#一个案例 10.211.55.9:/data    /data  nfs auto,rw,vers=3,hard,intr,tcp,rsize=32768,wsize=32768      0   0
12.34.56.789:/home  /mnt/nfs/home   nfs      auto,noatime,nolock,bg,nfsvers=3,intr,tcp,actimeo=1800 0 0
12.34.56.789:/var/nfs  /mnt/nfs/var/nfs   nfs     auto,noatime,nolock,bg,nfsvers=3,intr,tcp,actimeo=1800 0 0

# 卸载
umount /directory name

在终端中使用 showmount -e IP\Domain来查看 Centos 主机的共享状态

使用命令挂载即可

$ sudo mount -t nfs 192.168.100.222:/var/www /private/nfs

客户端 windows

打开控制面板 > 程序 > 启用或关闭 Windows 功能，找到NFS服务打开子目录勾选NFS客户端与管理工具。 NFS客户端：通过界面操作挂在NFS 管理工具：通过命令行挂在NFS

查看远程电脑NFS共享的目录

showmount -e 远程电脑的IP

挂载 打开我的电脑点击此电脑 > 映射网络驱动器

需要读写权限的需要修改注册表 通过修改注册表将windows访问NFS时的UID和GID改成0即可，步骤如下 1、在运行中输入regedit，打开注册表编辑器； 2、进入HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\ClientForNFS\CurrentVersion\Default条目； 3、选择新建—-QWORD值，新建AnonymousUid，AnonymousGid两个值，值为0； 4、重启电脑 注册表才会生效；

卸载 不要使用资源管理器的“断开网络驱动器”

问题

如果连接时 Centos 端提示

nfsd: request from insecure port (192.168.7.130:49232)!

则将’选项’配置成 (rw,async,insecure) 即可
