安装 SSH 服务

http://blog.csdn.net/netwalk/article/details/12952051

安装服务端

Ubuntu缺省没有安装SSH Server，使用以下命令安装： sudo apt-get install openssh-server 然后确认sshserver是否启动了：（或用“netstat -tlp”命令） ps -e|grep ssh 如果只有ssh-agent那ssh-server还没有启动，需要/etc/init.d/ssh start，如果看到sshd那说明ssh-server已经启动了。  如果没有则可以这样启动： sudo/etc/init.d/ssh start 　　事实上如果没什么特别需求，到这里 OpenSSH Server 就算安装好了。但是进一步设置一下，可以让 OpenSSH 登录时间更短，并且更加安全。这一切都是通过修改 openssh 的配置文件 sshd_config 实现的。

SSH配置

ssh-server配置文件位于/etc/ssh/sshd_config，在这里可以定义SSH的服务端口，默认端口是22，你可以自己定义成其他端口号，如222。然后重启SSH服务：

sudo /etc/init.d/sshresart   通过修改配置文件/etc/ssh/sshd_config，可以改ssh登录端口和禁止root登录。改端口可以防止被端口扫描。

sudo cp/etc/ssh/sshd_config /etc/ssh/sshd_config.original  
sudochmod a-w /etc/ssh/sshd_config.original

编辑配置文件 /etc/ssh/sshd_config  

找到#Port 22，去掉注释，修改成一个五位的端口：
Port 22333
找到#PermitRootLogin yes，去掉注释，修改为：
PermitRootLogin no
配置完成后重起：
sudo/etc/init.d/ssh restart  

SSH服务命令

停止服务：sudo /etc/init.d/ssh stop 启动服务：sudo /etc/init.d/ssh start 重启服务：sudo /etc/init.d/sshresart

更改 ssh 默认端口

服务器默认使用了 22 端口作为 ssh 服务的端口，这个端口成为入侵者重点关注的突破口，因此需要将它改成一些难以猜测的端口，例如 13682 这样的。

特别注意

如果你使用了防火墙服务，必须要先在防火墙规则里添加好你要修改的新端口，并且暂时不要关掉原来的22端口，确认用新端口可以登录后再关掉 22 端口。以免改完后无法登录！（我有一次在机房配置好后就回公司，在公司远程修改了 22 端口却忘了提前在防火墙开放端口，结果把自己挡在外面了，只好重新跑回到机房去配置）

# 先把防火墙里的新端口开放
$ vi /etc/sysconfig/iptables

# 复制 22 这行，改成 13682 等端口
-A INPUT -p tcp -m state --state NEW -m tcp --dport 22 -j ACCEPT
-A INPUT -p tcp -m state --state NEW -m tcp --dport 13682 -j ACCEPT

# 使防火墙规则生效
$ services iptables restart
$ iptables -L
ACCEPT     tcp  --  anywhere             anywhere             state NEW tcp dpt:ssh
ACCEPT     tcp  --  anywhere             anywhere             state NEW tcp dpt:13682

# 很重要的一步，关掉 SELinux(教程最前面有说明)，或者安装`policycoreutils-python` 来支持 `semanage` 命令，放行新端口
$ setenforce 0

$ yum -y install
$ semanage port -a -t ssh_port_t -p tcp 13682
$ semanage port -l | grep ssh
ssh_port_t                     tcp      13682, 22

如果使用 firewalld 替代 iptables

# 查看开放了哪些端口
firewall-cmd -list-all

# 开放新的端口
firewall-cmd --add-port=13682/tcp --permanent

systemctl reload firewalld

# 编辑 ssh 服务的配置文件，找到这两行，改成新的配置参数，第一个是端口，第二是禁止 root 登录
# 首先备份一下
$ cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak

$ vi /etc/ssh/sshd_config

# 重载 sshd 服务的配置参数(我发现还需要重启服务器才有效果……)
$ service sshd reload

$ reboot

# 最后再编辑 iptables 文件，删掉之前的 22 端口那行的配置，重启 iptables 服务，关闭 22 端口

不允许 root 远程登录

另外，我们还要禁止 root 这个用户来登录，以免攻击者猜测到或者管理员泄露了 root 密码后，还能保障一定的安全。

首先要创建一个非 root 用户

(如果不创建，等下设置生效后就进不去系统了。。。)

useradd eric
passwd eric

然后

$ vi /etc/ssh/sshd_config

# 从 PermitRootLogin yes 修改
PermitRootLogin no 

$ service sshd restart

当再次通过 ssh 远程登录时，需要在命令行中指定Port参数。

# 测试 root 直接登录（使用默认 22 端口），提示连接被拒绝，这是系统的防火墙在起作用
$ ssh root@192.168.56.101
ssh: connect to host 192.168.56.101 port 22: Connection refused

# root 使用指定新端口登录，提示权限被拒绝，这是 sshd 服务的`PermitRootLogin no`权限设置在起作用
$ ssh -p 13682 root@192.168.56.101
Permission denied, please try again.

# 使用 -p 指定新端口，以及用 newuser 这个我们之前创建的普通用户可以正常登录
$ ssh -p 13682 newuser@192.168.56.101
#The authenticity of host '[192.168.56.101]:13682 ([192.168.56.101]:13682)' can't be established.
#ECDSA key fingerprint is SHA256:MBkrnI7mctKMFkMovitFmvzpq4+Vj4uvLrbvGZXH/P4.
#Are you sure you want to continue connecting (yes/no)? yes

空闲超时退出时间间隔

如果用户连接了服务器，然后中途又因为其他事情离开了机器，这时候如果有其他人来到电脑前进行恶意破坏，那就会造成很大的损失了。 我们无法保证每个用户都有离开随时锁住屏幕的习惯，因此需要在服务器上设置一个超时时间，当服务器检测到用户很久没有操作后，就把这个空闲用户强制注销。

$ vi /etc/ssh/sshd_config
ClientAliveInterval 600
ClientAliveCountMax 0

要求用户使用公钥登录，禁止使用输入用户名和密码的方式登录

虽然我们禁用了 root 用户远程登录服务器，但是仍然有两个弊端：

  1. 一些别有用心的人仍然可以通过不断输入账号和密码去猜测和尝试
  2. 合法用户每次登录的时候，如果密码很复杂的话，输入容易出错

所以最好让服务器给每台信任的设备（例如你天天使用的笔记本电脑）“发个证”，每次登录的时候，服务器看一下这个证件就可以了，而不用繁琐地验证用户名和密码。

注意 用户一定要保管好自己的设备（和里面的“证件”），因为一旦丢失了设备或让别人进入了你的操作系统，就可以畅通无阻地以你的身份拿着“证件”登录到服务器了。

有两种免密登录方式

  1. 服务器上生成 id_rsa （钥匙） 和 id_rsa.pub (锁), 然后把 id_rsa 复制很多份，发给每个同事，让他们登录的时候指定“钥匙”

命令行

# -i 指定私钥，默认条件下使用 ~/.ssh/id_rsa
# -l 指定用户名 
# -p 指定端口，默认为 22
ssh -i ~/.ssh/id_rsa -p 2222 -l wangao 192.168.56.101

或 xshell/sftp 等工具的“指定密钥”

但一旦同事离职/电脑丢失，这个钥匙就相当于泄露了。。。

  1. 在每台需要登录的 客户端电脑 上生成一个自签名证书，然后把证书内容放到 服务器 上的“信任设备列表”里

推荐用这种方式，当员工辞职，管理员只需要从信任设备列表里删除这个证书内容就可以了

下面来详细说明下如何使用公钥来登录。这里假设被登录的服务器是“Server”，客户端是“Client”

在 Client （例如笔记本）上用 ssh-keygen 命令创建公钥**

# 在 Client 上运行
# ssh-keygen:生成秘钥 -t指定算法 -f 指定生成秘钥路径 -N 指定密码
$ ssh-keygen -t rsa

输入后，会提示已经创建了 .ssh/id_rsa、id_rsa.pub 两个文件，其中第一个为密钥，第二个为公钥。

将 Client 上生成的 公钥 文件 id_rsa.pub 复制到 Server 的 .ssh 目录下

首先要在 Server 上创建一个.ssh目录

注意，如果你禁用了 root 登录，那么就要在用于 SSH 登录的用户主目录下创建，例如 /home/newuser/.ssh

用用户、密码方式登录 Server 机器(这里是 newuser)

# 用 newuser 账号登录后
$ mkdir /home/newuser/.ssh
chown newuser:newuser /home/newuser/.ssh

注意如果你是在 root 用户下通过 newuser 创建用户的 .ssh 文件夹，需要 修改 owner，否则后面通过公钥文件验证会失败

然后回到 Client 机器， 将前面生成的公钥文件上传到 Server 的 .ssh 目录里面去

$ scp -P 13682 .ssh/id_rsa.pub newuser@192.168.56.101:~/.ssh/

回到 Server，发现文件已经存在

$ ll .ssh/
total 4
-rw-r--r-- 1 newuser newuser 398 Aug 20 06:08 id_rsa.pub

将 Client 的公钥文件添加到 Server 的“已验证钥匙”列表文件底部（.ssh/authorized_keys）

# 注意，现在是在 Server 机器上
$ cat ~/.ssh/id_dsa.pub >> /home/newuser/.ssh/authorized_keys


# 确保在 newuser 登录
whoami

# 设置authorized_keys权限
$ chmod 600 ~/.ssh/authorized_keys

# 设置.ssh目录权限
$ chmod 700 -R ~/.ssh

直接使用简化版的 ssh-copy-id 命令，复制 client 的公钥内容到 server 的 authorized_keys 文件底部

# 这里用户名是 test, 就会发送到目标服务器的 /home/test/.ssh/authorized_keys 文件
ssh-copy-id test@192.168.0.3

#输入目标服务器的用户密码

修改 sshd 配置，关闭密码登录

编辑 /etc/ssh/sshd_config 文件，进行如下设置：

RSAAuthentication yes
PubkeyAuthentication yes

另外，如果你希望 root 能通过 ssh 登录的话，请留意 root 用户能否通过 SSH 登录：

PermitRootLogin yes

当你完成全部设置，并以密钥方式登录成功后，再禁用密码登录：

PasswordAuthentication no

最后，重启 SSH 服务：

service sshd restart

回到 Client，测试无密码访问

# 注意现在是在 Client 上
$ ssh -p 13682 newuser@192.168.56.101

# 无需密码登录成功
Last login: Tue Dec 10 22:34:02 2013

跳板机

我们无法保证客户端的情况

  1. 个人电脑会不会遗失在家或被盗
  2. 个人电脑系统会不会崩溃
  3. 个人电脑的网络安全问题
  4. 员工离职后设备内密钥的清空
  5. 服务器维护太多的客户端公钥非常麻烦

因此，更好的办法是建立一台可信的“跳板机”，用户先登录这台机器，然后使用公钥的方式从跳板机登录到服务器。这样服务器就只要维护一些可信的 跳板机 IP 和公钥即可。

只允许部分用户通过 SSH 访问

当系统中添加的用户越来越多，我们就要设置只允许其中几个用户才能使用 SSH 登录。

只允许列表中的用户名登录

$ vi /etc/ssh/sshd_config
AllowUsers root user1  user2

限制只有某些个IP才能远程登录服务器

一些公司里人员流动比较频繁，当运维人员离职后，除了修改密码外，还要考虑到机器可能存在被外部访问的风险。因此设置一个 IP 地址列表，把公司的 IP 以及认为是安全的网络环境 IP 地址添加进去，然后禁止任何其他的 IP 地址登录。

$ vi /etc/hosts.deny     #在其中加入sshd:ALL
$ vi /etc/hosts.allow    #在其中进行如下设置：sshd:192.168.1.1     #(只允许192.168.1.1这个IP远程登录服务器)

常见错误

报错 Too many Authentication Failures for user root

在 ssh 或者 scp 的命令中添加参数

$ scp -o PubkeyAuthentication=no user@host

复制完 publich key 内容后，SSH 连接时还是需要密码

如果手动复制 id_rsa.pub 文件的内容到远端的 authorized_keys 文件，发现 SSH 连接时还是需要密码，可能是因为 vi 编辑的时候，把行号复制进去了。建议使用scp上传源文件，然后用 cat >> 附加到文件末尾的方式，避免复制的时候格式错误。

设置“客户端超时自动断开”

修改下面的操作可能会降低服务器的安全性（例如你在客户端登陆后，忘记退出，被别人继续登录上去操作）

vim /etc/ssh/sshd_config

# 修改下面两个值（如果不存在就添加）
ClientAliveInterval 30       （每30秒往客户端发送会话请求，保持连接）
ClientAliveCountMax 3      （去掉注释即可，3表示重连客户端3次失败后，重启SSH会话）

#重启sshd服务，使配置生效
/etc/init.d/sshd restart   
service sshd restart
#如果是 ubuntu
/etc/init.d/ssh restart

使用 ssh/config 结合 alias 的方式

编辑 ~/.ssh/config文件

host server1
    hostname 192.168.3.1
    username devops
    port 2222

host server2 server_test
    hostname 192.168.3.2
    username devops
    port 2222

host就是“别名”，可以有多个名称，将来可以直接用别名来登录

ssh server1

# 任意一个别名都可以
ssh server2
ssh server_test

如果某台服务器需要获取一个密钥来登录,在后面添加IdentityFile

host server018 mongodb-box mongo
    hostname 192.168.3.1
    username devops
    port 2222
    IdentityFile ~/.ssh/id_dsa_devops

使用命令别名的方式（推荐，因为上面那个不知道怎么输入“密码”）

vim ~/.bash_aliases

# 在里面添加所有别名
alias scotch='ssh nick@scotch.io'
alias example2='ssh root@example.com'
alias example3='ssh userxyz123@64.233.160.0 -p 56000'
alias amazon1='ssh ec2-user@ec2.amazon.com -i /path/to/special/privatekey/amazon.pem'

# 然后使之生效
source ~/.bash_aliases

# 然后就可以直接连接了
scotch

因为 ssh 命令没有“密码”的参数，只能用 sshpass 再加上 ssh, 并且由于密码可能有特殊字符，要用单引号包裹进来 例如

alias my_vm="sshpass -p 'Pass&Word_123' ssh root@1.2.3.4 -p22"

注意： 每次修改内容，都要运行 source ~/.bash_aliases

使用自写脚本，和一个字典

创建一个“字典文件” ssh_pass

第一位是“别名”，第二位是“IP/主机”，第三位是“密码”，第四位是“端口” 第五位是“用户名”

my_vm 10.211.55.11 123456 22 root
my_aliyun 1.2.3.4 PassWord123 22 root
my_bandwagon 5.6.7.8 WordPass456 27331 root

然后创建一个脚本，从字典里读取对应的内容

vi /usr/local/bin/autossh

内容如下

#!/bin/bash  
  
RC_ERR_NO_HOST=11  
RC_ERR_NO_PASSWORD=21  
RC_ERR_NO_PROT=31  
RC_ERR_NO_USER=41
RC_ERR_NO_SHORTNAME=51
RC_SUCCESS=0  
  
pass_path=~/zzy_work/sec_files/sshp_pass
  
shortname=$1  
  
# arguments   
if [ -z $shortname ]; then  
    echo "请检查连接的短名称"  
    exit $RC_ERR_NO_SHORTNAME  
fi  
  
# read file  
host=`grep $shortname\  $pass_path | cut -d' ' -f 2`  
if [ -z $host ]; then  
    echo "请检查 host 或 ip"  
    exit $RC_ERR_NO_HOST 
fi 

# read file  
pwd=`grep $shortname\  $pass_path | cut -d' ' -f 3`  
if [ -z $pwd ]; then  
    echo "请检查密码"  
    exit $RC_ERR_NO_PASSWORD  
fi  

port=`grep $shortname\  $pass_path | cut -d' ' -f 4`  
if [ -z $port ]; then  
    echo "请检查端口"  
    exit $RC_ERR_NO_PORT  
fi  

user=`grep $shortname\  $pass_path | cut -d' ' -f 5`  
if [ -z $user ]; then  
    echo "请检查用户名"  
    exit $RC_ERR_NO_USER  
fi 
  
exec sshpass -p $pwd  ssh $user@$host -p$port  
exit $RC_SUCCESS

然后给运行权限

chmod +x /usr/local/bin/autossh

使用的时候，需要先用 ssh 的完整命令连接一次服务器

ssh root@1.2.3.4

提示是否在 ~/.ssh/know_hosts里记录服务器，输入 yes

然后 exit退出，就可以直接用“字典”里的别名登录了

autossh my_vm
