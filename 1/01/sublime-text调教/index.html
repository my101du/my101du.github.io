Sublime Text 是一个代码编辑器……后面的内容我不想复制了，你们自己百度吧。 因为我是一个靠写代码为生的人，每天工作的时候，使用时间最长的除了浏览器就是编辑器了，因此选择编辑器只有一个要求：第一是启动和响应速度快，第二是内存占用小，第三是能折腾插件。在Windows下，我用得最多的就是 NotePad++，花式写PHP很是流畅。后来换 Mac 后，试用了若干编辑器（TextMate、TextWrangle等等，这个有空会再写一篇），最终留下了 Sublimte Text 3(其实2也很不错，但是在识别GBK文件的时候有一点点瑕疵所以就放弃了)。

安装插件所需要的库（以 sublime text 3 为例）

2017 09 14 正式版已经不需要手动运行这个命令了

直接打开 Package Control 命令面板，输入 install ，会提示安装 Package Control。然后就可以开始安装插件了。

其他 3 低版本

点击菜单view，Show Console，输入下面的内容，回车，这时候可能会卡住一会，稍等一下直到执行完毕。

import urllib.request,os,hashlib; h = 'df21e130d211cfc94d9b0905775a7c0f' + '1e3d39e33b79698005270310898eea76'; pf = 'Package Control.sublime-package'; ipp = sublime.installed_packages_path(); urllib.request.install_opener( urllib.request.build_opener( urllib.request.ProxyHandler()) ); by = urllib.request.urlopen( 'http://packagecontrol.io/' + pf.replace(' ', '%20')).read(); dh = hashlib.sha256(by).hexdigest(); print('Error validating download (got %s instead of %s), please try manual install' % (dh, h)) if dh != h else open(os.path.join( ipp, pf), 'wb' ).write(by)

重启 Sublime Text。

如果在菜单Preferences可以看到Package Control，就表示安装成功，接下来可以安装插件了。

如果通过上面的方式安装失败，可以点击下面的页面，下载 Package Control 包，直接放到安装文件的 Packages/目录（可以通过菜单里的Browse Packages找到这个目录）

https://packagecontrol.io/Package%20Control.sublime-package

安装插件

运行 Cmd+Shift+P，输入install，选中 install package即可安装插件了。

如果要显示当前有哪些插件，运行 Cmd+Shift+P，然后输入list，根据自动提示选择即可；同样要删除一个插件，输入remove跟着操作即可。

source map 的问题

压缩/混淆后，如果在浏览器报错， chrome开发者工具是很难进行排查的。

部署前端之前，开发者通常会对代码进行打包压缩，这样可以减少代码大小，从而有效提高访问速度。然而，压缩代码的报错信息是很难Debug的，因为它的行号和列号已经失真。这时就需要Source Map来还原真实的出错位置了。

需要一个 source map 文件来帮助定位错误

首先在文件底部添加一行.

//# sourceMappingURL=jquery.min.map

注意不要在插件里开启了删除注释

"keep_comments": true,
"comments_to_keep": "/sourceMappingURL/",

如果最新的uglifyjs --version 已经是最新的 3.x.x 版本，这个插件默认的命令会报错

需要打开插件来编辑一下，/Users/zzy/Library/Application Support/Sublime Text 3/Packages/Minify/Minify.py

找到这里，直接把后面的全部删掉

改成保留一个参数就足够了

--source-map content=inline,includeSources,url=inline

再次运行命令，正常生成

配置：

打开 Sublime Text > Preferences > Package Setting > Plant​Uml​Diagrams > Setting - User，在其中添加如下配置：

{
    "plantuml_server": "http://www.plantuml.com/plantuml/",
    "jar_file": "path/to/plantuml.jar",
    "charset": "UTF-8"
}

Plant​Uml​Diagrams 的运行机制是先调用 plantuml_server 配置的服务进行转换，如果不行再调用 jar_file 的 jar 包来转换。

上文中的 plantuml_server 是 PlantUML 的官方服务，如果涉及到一些项目机密等情况，你也可以提供一个自建的服务

让 phpfmt 插件能够运行在 PHP5.6 或更低版本下

phpfmt 是一个非常强大的 PHP 代码格式化插件，可以一键帮你生成符合 PSR 规范的代码、进行等号对齐等等。但是最新的版本要求 PHP7.0 以上，如果你用了 PHP5.6或更低的，会在运行 phpfmt 时报错。

由于一些特殊原因，我们的网站可能不能运行在 PHP7 （例如一些老的类，用了类同名构造函数而不是 __constructor，却又不方便修改），因此我们需要修改下这个错误提示，让它能忽略版本检测而正常运行 phpfmt。

打开 Preferences，Browse Packages，找到 phpfmt插件的文件夹，找到phpfmt.py文件，搜索 “5.6"字样，把那一行很长的if后面部分删除（就是下面这部分）

 or 'PHP 5.6' in res.decode('utf-8') or 'PHP 5.6' in err.decode('utf-8')

保存，就既不会报PHP版本不支持，又能使用代码格式化功能了。
