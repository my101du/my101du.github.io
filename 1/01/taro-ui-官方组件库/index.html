Taro-UI 官方组件库

https://taro-ui.aotu.io/#/docs/introduction

之前的“组件库”，一般是和 微信小程序 组件对应的一些“基础组件”，需要自己从0开始添加样式

而这里的 Taro-UI ，有点类似 Antd，已经是一套成熟的多端 UI 组件库，自带样式，可以直接使用。Web、ReactNative、微信小程序等各种端大行其道，当业务要求同时在不同的端都要求有所表现的时候，针对不同的端去编写多套代码的成本显然非常高，这时候只编写一套代码就能够适配到多端的能力就显得极为需要。

如果 Taro-UI 里没有合适的直接可用的组件，可以用 Taro 组件库里的“原始组件”来添加 CSS 实现

npm install taro-ui

import {AtButton} from 'taro-ui'

render...
  <AtButton type='primary'>Add</AtButton>

注意很多组件的属性，只支持微信小程序平台，H5、rn 并不支持

官方demo(一些事件的处理范例)，例如 TabBar 的 onClick 切换具体事件

https://github.com/NervJS/taro-ui-demo

自定义主题

目前自定义主题的功能还在开发中，如果您有修改组件样式的需求，可以通过以下方式进行修改：

globalClass 全局样式类 配置 customStyle 属性（仅有部分组件支持，请查看组件文档）

在组件构造器中的 options.addGlobalClass 字段设置为 true（Taro UI 的组件均开启了此特性）

/* CustomComp.js */
export default CustomComp extends Component {
  static options = {
    addGlobalClass: true
  }
  
  render () {
    return <View className="red-text">这段文本的颜色由组件外的 class 决定</View>
  }
}


/* 组件外的样式定义 */
.red-text {
  color: red;
}

issue 覆盖无效 https://github.com/NervJS/taro-ui/issues/66

小程序的 addGlobalClass 只能使 page 页面上的样式可以影响 组件 所以目前的解决办法只有把 样式写在 page 上面 或者用 customStyle

使用第三方字体图标库

可自行下载 Ionicons 或 Font Awesome 等拓展字体图标库,然后

(1) 配置 postcss 小程序端样式引用本地资源内联 config/dev[prod].js

//....
    postcss: {
        // 小程序端样式引用本地资源内联
        url: {
          enable: true,
          limit: 102400000000
        }

(2) 编写字体图标库 css

/* icon.scss */
@font-face {
  font-family: 'FontAwesome';
  /* 自行安装第三方字体图标库 */
  src: url('./assets/fonts/fontawesome-webfont.eot?v=4.7.0');
  src: url('./assets/fonts/fontawesome-webfont.woff2?v=4.7.0') format('woff2'), url('./assets/fonts/fontawesome-webfont.woff?v=4.7.0') format('woff'), url('./assets/fonts/fontawesome-webfont.ttf?v=4.7.0') format('truetype');
  font-weight: normal;
  font-style: normal;
}
/* 根据第三方字体图标库编写 */
/* 举例：fa 就是 prefixClass 的值，下面的的图标 css class 命名都要用 fa- 开头  */
.fa {
  display: inline-block;
  /* 以下的 font 与上面 @font-face 的 font-family 要一致*/
  font: normal normal normal 14px/1 FontAwesome;
  font-size: inherit;
  text-rendering: auto;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}
.fa-clock:before {
  content: "\f00c";
}

(3) 在 app.js 全局引入 icon.scss

/* app.js */
import './icon.scss'

(4) 在 设置-项目设置-调试基础库 设置版本 2.2.3 以上

(5) 使用 AtIcon className 前缀，用于第三方字体图标库，比如想使用’fa fa-clock’ 的图标，则 传入

<AtIcon prefixClass='fa' value='clock' size='30' color='#F00'></AtIcon>

其他

跨域与代理

package.json 末尾里添加一段, 无效

"proxy": "https://www.sample.org"

查看“配置详情”章节，可以为 h5 添加 webpack 的一些自定义配置，打开 /config/dev.js

devServer: {
    //historyApiFallback: true,
    //hot: true,
    //inline: true,
    //host: '10.0.0.9',
    //contentBase: path.join(__dirname, "dist"),
    //compress: true,
    //port: 9000,
    proxy: {
        '/api': {
            target: 'http://www.qq.com:3000',
            secure: false,
            changeOrigin: true
        }
    }
}

如果是跨域，记得加上选项 changeOrigin: true

这样，请求

/api 就会自动转发到 http://www.qq.com:3000去，解决了无法在 h5 模式下请求其他域名的跨域问题

但是，这个配置一般是本地开发阶段或者基于 html5 的前端项目时用到，当编译成 ReactNative 等应用后，反而不需要代理，否则会导致/api 的请求就失效了（因为 webpack devServer 没有了）

所以对于生成环境的配置， 为了复用这个请求 /api 的代码，又请求真实的远程地址

https://zhuanlan.zhihu.com/p/29161762

因此定义一个常量，例如 API_HOST

在 config/dev.js 本地开发环境，需要用代理时， 它的值为 /api

而 config/prod.js 生产环境，不需要代理，它的值设置为 http://www.sample.org/api

taro 里叫 defineConstants ， 没有说明 dev prod default 的优先级？
