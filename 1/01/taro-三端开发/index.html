taro 三端开发

主要是因为 taro-ui 不支持 RN, 因此务必首先兼容 react-native, 用 flex 等来布局

(现在小程序发展这么快，如果没有特别要用 ReactNative 生成 app, 可以考虑拿到 H5 版本，然后套一个壳。)

毕竟 taro-ui 自带的组件那么多，自己写太麻烦了。

主要参考

https://developers.weixin.qq.com/community/develop/article/doc/00088cd4d34120236928d031f56413

项目库 https://github.com/js-newbee/taro-yanxuan

以及 http://morecoder.com/article/1109165.html

一些分析对比总结

  1. uni-app 虽然能结合 vue weex 实现多端生成，而且有 uni-ui(组件/模板)，比 Taro 的 RN 端多样式库方面的支持，但是社区质量，以及未来支持，都感觉不如 Taro 的支持力量。
  2. Taro 如果不想用自带 RN 来生成原生应用（expo），可以生成 H5，然后用 Ionic 封装起来，也是一种思路（因为 RN 也还有挺多坑的）
  3. Taro 开发 RN，因为不能用 ReactNative Base 等这些UI库，自己写 sass 还是挺麻烦的。而且为了兼顾 RN, 相应地在小程序、H5 上受局限也更大

因此，Taro 专注小程序和 H5, 能用到 Taro-UI 和其他， 要开发 App ，就直接用 RN (而不是 Taro 下的 RN)了。

对于 2. 有人提问：https://github.com/NervJS/taro/issues/2296

一些问题

dev:h5 下 taro-ui报错

./node_modules/taro-ui/dist/h5/components/countdown/index.js 105:11 Module parse failed: Unexpected token (105:11) You may need an appropriate loader to handle this file type.

方法： 修改 config/index.js 在 h5下加

esnextModules: ['taro-ui'],

(当然，不使用 taro-ui 也可以)
