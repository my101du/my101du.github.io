测试工具 Jest

  jest  评论 字数统计: 1385字 阅读时长: 3分

资料 https://www.jianshu.com/p/8b1301ff45fa

使用 Jest(基于 jasmine) 和 Enzyme

Jest will automatically define NODE_ENV as test

安装

如果使用了creat-react-app脚手架来创建项目，已经包含了。

如果没有，则要安装库

yarn add --dev jest enzyme

修改 package.json,添加一个命令，以便运行npm test(或 yarn test)

"scripts": {
    "start": "react-scripts start",
    "test": "jest" //添加的
},

如果你的项目使用了 babel

需要安装对应支持。（regenerator-runtime在 npm 的新版中已经不需要单独安装了）

npm install --save-dev babel-jest

如果使用了 ES6 和 React.js 特性（确保已经安装过 babel-preset-es2015 和 babel-preset-react 这两个 presets），需要明确在.babelrc文件里启用（没有这个文件则要创建）

If you’ve turned off transpilation of ES2015 modules with the option { “modules”: false }, you have to make sure to turn this on in your test environment.

{
    "presets": [
        ["es2015", 'react', { "modules": false }]
    ],
    "env": {
        "test": {
          "presets": [["es2015"], "react"]
        }
  }
}

Note: babel-jest is automatically installed when installing Jest and will automatically transform files if a babel configuration exists in your project. To avoid this behavior, you can explicitly reset the transform configuration option:

// package.json
{
  "jest": {
    "transform": {}
  }
}

如果使用了 webpack

https://facebook.github.io/jest/docs/en/webpack.html

基础

首先创建一个sum.js 文件

function sum(a, b){
    return a+b;
}

module.exports = sum

然后创建专门对sum.js进行测试的文件sum.test.js

const sum = require('./sum.js')

test('this is a test function', ()=>{
    expect(sum(1, 2)).toBe(3);
})

在当前目录运行测试命令

yarn jest

# 结果
PASS  ./sum.test.js
✓ adds 1 + 2 to equal 3 (5ms)

如果在全局命令行中使用 jest

https://facebook.github.io/jest/docs/en/cli.html

npm install -g jest

jest my-test --notify --config=config.json

Matchers 匹配

全部： https://facebook.github.io/jest/docs/en/expect.html

常用

  • toBe // 表示===精确等于
  • toEqual // 在进行对象的值是否相等时使用，会递归检查

“是否存在” “条件满足”

  • toBeNull matches only null
  • toBeUndefined matches only undefined
  • toBeDefined is the opposite of toBeUndefined
  • toBeTruthy matches anything that an if statement treats as true
  • toBeFalsy matches anything that an if statement treats as false

数字

  • toBeGreaterThan
  • toBeGreaterThanOrEqual
  • toBeLessThan
  • toBeLessThanOrEqual

toBe 和 toEqual 同样可用

对于浮点数，使用toBeCloseTo，不要使用toEqual

  • expect(value).toBeCloseTo(0.3);

字符串

正则

  • toMatch(/stop/)

数组包含

  • expect(shoppingList).toContain(‘beer’);

抛出异常

function compileAndroidCode() {
  throw new ConfigError('you are using the wrong JDK');
}

test('compiling android goes as expected', () => {
  expect(compileAndroidCode).toThrow();
  expect(compileAndroidCode).toThrow(ConfigError);

  // You can also use the exact error message or a regexp
  expect(compileAndroidCode).toThrow('you are using the wrong JDK');
  expect(compileAndroidCode).toThrow(/JDK/);
});

相反情况

使用not可以对上面的匹配取相反情况

  • to.not.toBe
  • to.not.toEqual

异步

基本的

在 test 的第二个参数（函数）上传入done，在异步操作执行完后调用done()

test('the data is peanut butter', done => {
  function callback(data) {
    expect(data).toBe('peanut butter');
    done();
  }

  fetchData(callback);
});

Promises

Just return a promise from your test, and Jest will wait for that promise to resolve.If the promise is rejected, the test will automatically fail.

注意没有done了，否则会在运行测试时报

Error: Timeout - Async callback was not invoked within timeout specified by jasmine.DE FAULT_TIMEOUT_INTERVAL.

test('the data is peanut butter', () => {
  expect.assertions(1);
  
  //fetchData(instead of using a callback)returns a promise
  return fetchData().then(data => {
    expect(data).toBe('peanut butter');
  });
});

//If you expect a promise to be rejected use the .catch method. 
test('the fetch fails with an error', () => {
  expect.assertions(1);
  return fetchData().catch(e =>
    expect(e).toMatch('error')
  );
});

.resolves / .rejects

You can also use the .resolves matcher in your expect statement, and Jest will wait for that promise to resolve. If the promise is rejected, the test will automatically fail.

test('the data is peanut butter', () => {
  expect.assertions(1);
  return expect(fetchData()).resolves.toBe('peanut butter');
});

test('the fetch fails with an error', () => {
  expect.assertions(1);
  return expect(fetchData()).rejects.toMatch('error');
});

Async/Await

To write an async test, just use the async keyword in front of the function passed to test

test('the data is peanut butter', async () => {
  expect.assertions(1);
  const data = await fetchData();
  expect(data).toBe('peanut butter');
});

test('the fetch fails with an error', async () => {
  expect.assertions(1);
  try {
    await fetchData();
  } catch (e) {
    expect(e).toMatch('error');
  }
});

combine async and await with .resolves or .rejects

test('the data is peanut butter', async () => {
  expect.assertions(1);
  await expect(fetchData()).resolves.toBe('peanut butter');
});

test('the fetch fails with an error', async () => {
  expect.assertions(1);
  await expect(fetchData()).rejects.toMatch('error');
});

Actually, I find that it does work, but it specifically addresses self-signed certificates. It does not allow expired or invalid certificates. To allow any certificate, you have to add this line near the top of your code;

与 localstorage

安装配置

安装jest-localstorage-mock

yarn add --dev jest-localstorage-mock

修改package.json,直接引用包名

"jest": {
  "setupFiles": [
    "jest-localstorage-mock"
  ]
}

或者使用 setUpFiles

"jest": {
  "setupFiles": [
    "./__setups__/localstorage.js"
  ]
}

/__setups__/localstorage.js内容是

require('jest-localstorage-mock');

如果在 create-react-app 创建的项目中使用

require("jest-localstorage-mock");

在 test 中使用

支持 localStorage and sessionStorage object

输出一些内容

在test()方法体中，使用console.log()不会输出任何东西。改成it()

it('test search', ()=>{
    data = {message: 'request successufl'};

    // now output to terminal
    console.log(data.message);
    
    expect(data.message).toMatch(/request successful/);
});
