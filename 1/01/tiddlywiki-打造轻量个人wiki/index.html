TiddlyWiki 打造轻量个人wiki

优势：一个文件搞定，笔记之间的链接

问题： 体积不断增长 保密性？

中文版

https://tiddlywiki.com/languages/zh-Hans/empty.html

有问题，很多按钮打不开。。。。

外观 Appearence

Theme tweak

Fluid Story fixed sidebar,(正文左侧更大面积)

保存

可以使用 PHP 的 store.

全部走一个服务器，不需要在设备之间转换

下载 https://code.google.com/archive/p/bidix/downloads

store.php 这里面找 _th\lib

编辑密码 $USERS = array( ‘UserName1'=>'Password1’, etc)

上传 store.php

打开 setting, saving, 点击 TiddlySpot

go to the Saving tab of the control panel ,

输入用户名 Username 密码

Server URL The full URL to the store.php file on your server Upload filename The filename used to save the TiddlyWiki (defaults to index.html) Upload directory The relative path from store.php to the directory used for saving the file Backup directory The relative path from store.php to the directory used for backups

还需要修改 Backup directory 保存备份（会不会越来越大啊！！！， 每保存一次 2M, 改一个字就一次？ 一天1000次就是 2G??） （可以在 savding 里取消 auto save….）

在服务器上，全部位于同一目录中，并且所有权限暂时设置为777（包括目录本身）：

修改 php 限制php_value upload_max_filesize 4M php_value post_max_size 6M

If you get an error message regarding split(), you may need to change references to split in store.php to function explode .

定期把服务器上这个经常更新的 html 文件，定期备份到网盘或自己本地硬盘即可

git 仓库 省了自己用 PHP 了 ~~

密码方式也可以

或者 setting-developer settting -> personal token

修改完了点保存，

如果出现401 错误， 说明当前上网的问题。就翻墙下。

如果成功提示 Saving wiki with method save through saver github 。。。

如果你服务器在国内，很可能也被墙了。。。 换 PHP 方式吧
