js 是一种“弱语言”（脚本），对变量的类型检查不严格。

class Student{
	fullName: string;
	
	constructor(public firstName,public middleInitial, public lastName){
	  this.fullName = firstName + " " + middleInitial + " " + lastName;
	}
}

interface Person {
  firstName: string;
  lastName: string;
 }

 function greeter(person: Person){
  return "Hello" + person.firstName +  " " + person.lastName;
  }

  var user = {firstName: "jane", lastName: "User"};
  document.body.innerHTML = greeter(user);

运行

tsc greeter.ts

得到greeter.js文件，可以看到 class Student语法被编译成js的function形式。

var Student = (function () {
    function Student(firstName, middleInitial, lastName) {
        this.firstName = firstName;
        this.middleInitial = middleInitial;
        this.lastName = lastName;
        this.fullName = firstName + " " + middleInitial + " " + lastName;
    }
    return Student;
}());
function greeter(person) {
    return "Hello" + person.firstName + " " + person.lastName;
}
var user = { firstName: "jane", lastName: "User" };
document.body.innerHTML = greeter(user);

对比 Flow

在 React 项目中启用 TypeScript

webpack 帮助代码打包（可选将所有依赖都放入到一个 js 文件）

declaration files： @types/react @types/react-dom ，通常我们 import 如 react,会自动在 react 包里面找，但是不是所有包都包含了 declaration files, 所以 TypeScript 还会在 @types/react 里面查找（以后不需要考虑这个）

安装 typescript 到“开发时”依赖（–save-dev） awesome-typescript-loader 读取 TypeScript 标准配置文件 tsconfig.json ，让 webpack 编译 TypeScript 代码；（或者使用 ts-loader https://github.com/s-panferov/awesome-typescript-loader#differences-between-ts-loader） source-map-loader待查（与 sourcemap 和 debug 有关）

这里把 typescript 安装为 dev 依赖，也可以使用npm link typescript来链接到全局的 typescript

# Global install the webpack
npm install -g webpack

# Go to the project path
cd ~/projects/test

# Install the react library
npm install --save react react-dom @types/react @types/react-dom

# Install the typescript
npm install --save-dev typescript awesome-typescript-loader source-map-loader

配置文件

tsconfig.json文件包括了所有输入文件和配置参数

{
    "compilerOptions": {
        "outDir": "./dist/",
        "sourceMap": true,
        "noImplicitAny": true,
        "module": "commonjs",
        "target": "es5",
        "jsx": "react"
    },
    "include": [
        "./src/**/*"
    ]
}

更多内容：https://www.typescriptlang.org/docs/tsconfig-json.html

创建 React 组件

新建文件 src/componets/Hello.tsx (注意，tsx 不是 js 或 jsx)

这是一个无状态“函数式”组件

import * as React from "react";

export interface HelloProps { compiler: string; framework: string; }

export const Hello = (props: HelloProps) => <h1>Hello from {props.compiler} and {props.framework}!</h1>;

当然可以换成“类”的样子

import * as React from "react";

export interface HelloProps { compiler: string; framework: string; }

// 'HelloProps' describes the shape of props.
// State is never set so we use the '{}' type.
export class Hello extends React.Component<HelloProps, {}> {
    render() {
        return <h1>Hello from {this.props.compiler} and {this.props.framework}!</h1>;
    }
}

好了 现在在一个页面 index.tsx 里来使用 Hello.tsx 组件。 注意传入的 props (compiler 和 framework)

import * as React from "react";
import * as ReactDOM from "react-dom";

import { Hello } from "./components/Hello";

ReactDOM.render(
    <Hello compiler="TypeScript" framework="React" />,
    document.getElementById("example")
);

创建一个 webpack 配置文件 webpack.config.js

module.exports = {
    entry: "./src/index.tsx",
    output: {
        filename: "bundle.js",
        path: __dirname + "/dist"
    },

    // Enable sourcemaps for debugging webpack's output.
    devtool: "source-map",

    resolve: {
        // Add '.ts' and '.tsx' as resolvable extensions.
        extensions: [".ts", ".tsx", ".js", ".json"]
    },

    module: {
        rules: [
            // All files with a '.ts' or '.tsx' extension will be handled by 'awesome-typescript-loader'.
            { test: /\.tsx?$/, loader: "awesome-typescript-loader" },

            // All output '.js' files will have any sourcemaps re-processed by 'source-map-loader'.
            { enforce: "pre", test: /\.js$/, loader: "source-map-loader" }
        ]
    },

    // When importing a module whose path matches one of the following, just
    // assume a corresponding global variable exists and use that instead.
    // This is important because it allows us to avoid bundling all of our
    // dependencies, which allows browsers to cache those libraries between builds.
    externals: {
        "react": "React",
        "react-dom": "ReactDOM"
    }
};

注意看 externals ，不需要把这两个依赖打包到 bundle 文件，这会增加编译时间，并且浏览器也不会缓存这两个文件（因为它们是不会经常变化的）

新建一个 index.html 把文件引入

<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8" />
        <title>Hello React!</title>
    </head>
    <body>
        <div id="example"></div>

        <!-- Dependencies -->
        <script src="./node_modules/react/umd/react.development.js"></script>
        <script src="./node_modules/react-dom/umd/react-dom.development.js"></script>

        <!-- Main -->
        <script src="./dist/bundle.js"></script>
    </body>
</html>
