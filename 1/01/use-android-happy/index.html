让Android手机更好用

  技能  评论 字数统计: 5308字 阅读时长: 11分

从 Android 1.5 版本开始，到现在的 6.x 版，我使用过 N 个安卓手机。最开始是三星的 i5700，后来是中兴 V880，然后是一款联想，后来又是一款中兴的大屏机（这时候“剁手兴”的名号已经震天响了），接着是 Moto 的一款全键盘仿黑莓的，以及营业厅赠送的两部低端三星电信机，现在是三星 J3 和小米4C，以及一款非常喜欢的 Moto XT901，还有几台过渡几个月或损坏或卖掉记不太清楚了。因此除了自己编译 ROM外，各种系统的优化、软件的使用都是非常熟悉，甚至还修理过两台手机的屏幕排线和听筒模块。也记录一下那段废寝忘食上网找 ROM、root的日子吧。

ROOT 与 Bootloader 解锁

ROOT

ROOT 操作就像给苹果手机“越狱”，以便能更全面地控制自己的手机。安卓系统 ROOT 后的自由度比苹果的更大，所以玩家们习惯拿到手就 ROOT。比如国内很多的手机厂商喜欢预装很多垃圾APP，甚至还有隐藏的恶意扣费后台程序，ROOT后就能彻底干掉它们了。

最开始的 ROOT 方式很麻烦，要通过 adb 这个命令行工具连接手机，然后一个个命令敲进去，我记得有一次把机器给弄成了砖。

后来就出来了很多软件，比如 ROOT大师，King Root 等等，有的刷机工具也自带 ROOT 功能，极大简化了操作。步骤基本上就是：

  1. 打开手机的 开发人员选项，USB调试 开关（开发人员选项如果是隐藏的，一般是到“关于”里，点击 ·版本号 若干次就自动显示了）
  2. 手机连上电脑
  3. 安装ROOT软件
  4. ROOT软件自动安装手机的驱动和对应的 APP（这个APP其实后期可以删除的，一般都是什么系统清理工具）
  5. 打开软件开始ROOT

手机重启几次后，就 ROOT 成功了，但是因为安卓手机型号太多太多，这些软件基本上只能把市场上常见的机型覆盖到，如果是特别冷门的机器，就不行了。

BootLoader 解锁

一些国外的手机会有这个锁，我一般都是让商家帮忙弄好，很少自己操作过。

刷 ROM

线刷与卡刷

所谓 线刷 就是用 USB 线连接电脑和手机，在电脑里运行刷机工具（一般是厂家提供的）；卡刷 就是不用连着电脑，把 ROM 包放在手机闪存上，按住手机上的几个键重启进入 Recovery 模式，选择 ROM 包进行刷机（这种方式除了刷 ROM，有时候还用于刷入一些第三方的软件或库，例如 GMS）。

由于安卓系统的高度可定制，不同厂家自己都会有自己的 ROM（美其名曰叫个性化，其实就是想塞自己的 APP 来赚钱）。所以要想使用一些优秀的第三方系统，就只能通过刷 ROM 的方式来替换手机出厂的默认系统了。

这个过程有点像给默认为 Windows 7 系统的电脑重新安装 Windows 8。

现在优秀的 ROM 很多，我比较喜欢

  • CM（国外的一个团队做的，在原生 Android 上扩充了一些功能）
  • 魔趣（国内人做的，和CM很相似，应该都是用的同一个底包，比较适合中国用户的习惯）
  • MIUI（小米的系统应该是最好用的，但是适用机型不多）

因为刷第三方 ROM并不如 ROOT 那么简单。首先是需要时间，你需要去寻找、对比各种 ROM 的特性和参数，因为这个东西不像官方系统，一般都是个人或者小团队自己做的，在稳定性和安全性上并不会给你很大的保障，要仔细去鉴别。

其次是刷的过程中，如果有断电、或者操作失误，容易造成手机故障（俗称的变砖），这时候就比较麻烦了。

如果想简单，就直接下载 刷机精灵 这种软件来操作，但是它们可选的 ROM 不多。

我喜欢下载 ROM 的 zip 包后，放在手机的闪存上，进入 Recovery Mode 刷机。经常提到的 三清，就是删除手机的 user data，cache data，以及进行一次factory set。现在的 Recovery也做得越来越智能和傻瓜化了。想起以前给 i5700 刷机的时候，还要安装三个电脑上的软件，load 4个包，插着USB线分7个步骤……

不过呢，现在觉悟了，刷ROM的过程除了能体会一下新界面的新鲜感，和些许速度提升、内存占用减少，纯属浪费时间。当然了，还是要向广大做出这么多优秀的 ROM的爱好者们致敬！因为热爱，所以执着。

安装gms实现google服务同步（日历、邮件等）

由于各种坑爹的原因，国内厂商的手机都把 Google 有关的服务和app（简称 gms）给删除了。其实他们也是为了用户着想，因为即使不删除，GFW的原因你也没法用……

安装gms包

最早的时候，要补上这个包，得找到对应机型和版本的包，进入 Recovery 刷进去，如果弄错了版本和型号，还容易手机变砖。现在有个 APP，好像就叫“一键安装Google服务”，先安装它，重启后就可以在里面安装google 的各种APP了

shadowsocks

这是一个翻墙软件，这里不做太多介绍，详情可以看另外一篇文 越过长城，走向世界

添加google帐号

正常翻墙后，就可以进入“账号”里面添加 google 账号了，然后开启日历、联系人等等各种同步即可享受 google 的优质服务。

绿色守护和服务管理

由于国内的软件开发商比较没有节操，常常是“全家桶”的推广模式，装了一个 APP，它全家都一起来了，所以有些老人家的手机上你可以看到百度卫士、百度手机管家、360手机助手、腾讯手机管家……轮番自动启动，手机卡得像得了关节炎，电量刷刷往下掉。

由于这些所谓的“助手”都有“杀死程序”的功能，那么我一直有个疑问，他们都能杀对方，谁最后活下来？

所以还是用一个比较中立的APP来管理这些流氓吧。“绿色守护”是我比较喜欢的一个APP，曾经买过它的捐赠版（其实就是付费……），通过把所有程序都放到黑名单里面，就可以进行唤醒阻断和自动休眠了。

后来用了小米手机后，它自带的权限管理和神隐模式非常好用，然后平时特别注意使用的时候别不小心勾选了安装垃圾应用，基本上也没遇到过全家桶问题了。

联通打开了4G，但无法上网

一般国产安卓里，会有一个蜂窝数据开关，但是 AOSP 系统有一次不小心把它关闭了，一直没有4G信号，还以为手机坏了。

后来才发现这个控制手机流量上网放的位置很奇怪

进入流量使用状况，在里面打开移动网络开关

省电软件要不要用？

有一些号称可以省电的APP，其实就是通过定时关闭网络、调整亮度、关闭 CPU 核心等等。通过对比测试，我觉得没有太大的必要，只需要一个自动休眠后台进程的APP就可以了。一个智能手机，需要不停地开关网络也太不智能了，以及在唤醒时由于CPU的频率没有恢复导致的卡顿挺让人烦心的。不如买个续航长一些的手机吧。

解决MOTO XT901 电池耗电快的问题（有LTE制式的电信机器也可以试试）

在拔号界面，拔*#*#4636#*#*，手机信息，首选网络里改成CDMA AUTO(PRL),退出，再到手机 设置界面 – 更多 –网络选择里 这时应该己经是CDMA 项，点开再选一下CDMA，退出，再重启看看，己经固定在CDMA 模式了。

关闭所有动画，让操作行云流水

我个人不是很喜欢手机上的过渡动画，晃啊晃的太浮夸了，最喜欢的就是即点即到的那种感觉。关闭所有系统动画的操作是：

  1. 进入开发人员选项
  2. 拉到下面，找到 窗口动画缩放等三个动画选项，全部改为 0 或关闭

adb 命令（刷入 recovery 等）

安装 adb

# macos
brew cask install android-platform-tools

基本命令

# 列出连接设备
$adb devices

# 安装 apk
$adb install example.apk

# 备份
# https://www.linkedin.com/pulse/backup-restore-your-android-phone-using-adb-ajibola-okubanjo
# apk – Backs up your apps, noapk – Does not backup apps.
# shared – Backs up data on the SD card. noshared – Does not backup data on the SD card.
# package参数可以选择备份单独的应用，如果你不是备份某个应用，使用-all备份整个系统
# -system|-nosystem 决定-all标签是否包含系统应用，默认的是-system，根据情况可选择是否用-nosystem
adb backup -apk -shared -all -f android-backup-file.adb

# 恢复备份
adb restore android-backup-file.adb

# 进入连接手机的命令行
$adb shell
->ls -al /sdcard

# 重启(分别是普通重启，进入fastboot，进入recovery)
$adb reboot
$adb reboot bootloader
$adb reboot recovery

# 刷第三方 recovery
$adb reboot bootloader（先重启进入 bootloader）
$fastboot flash recovery /local/recovery.img

# ！！！据说一定要这样 fastboot boot，否则会被官方的覆盖回去。。。导致在刷完后
$fastboot boot /local/recovery.img

# 稍等片刻自动重启 自动进入 recovery

# 在图形界面里三清！！！
# 选择清除选项，在“滑动确认恢复出厂设置”那里向右滑动，完成双清
# 然后选择 advance ... 全选彻底清除所有数据

# 上传文件到手机 （例如 ROM 压缩包，Magisk root 工具， Gapps 等）
# 注意，要依次进行，先刷入系统重启，安装完后，再刷入 magisk 等
$adb push /local/rom.zip /sdcard/

# 然后在图形界面里"install"选择 /sdcard/rom.zip 刷系统
# 刷完后，重启时提示是否清空 cache/davlk 选“是”

# 如果在刷系统时发生错误 this package is for device: xxxx this device is . 无法识别当前设备型号
# 重新进入 recovery
# 选择 install
# 下面点击“image” 选输入 img 文件
# 把本地的 recorvery.img 复制到设备
# 刷入，选择 type 为“Recovery”
# 重启设备

# 如果还是无效，刷 3.2.1 等版本的 recovery, 我的小米4c 安装 RR6(Oreo 8.1)就是这样解决的，3.1.1 的img 不行

# 重启的时候问是否要把 twrp 安装进系统，滑动确认安装

# fastboot 模式 
$fastboot reboot

# 从 fastboot 中 重新进入 bootloader
$fastboot reboot bootloader

小米刷谷歌套件

  • 首先要向官方申请解锁 BootLoader，否则是不可以刷机的。
  • 申请通过后，下载解锁工具，对手机解锁
  • 下载小米专用的线刷工具和线刷包，音量下 + 电源，进入 FastBoot 模式，刷上 MIUI 的开发版
  • 然后 root 机器（在安全中心——授权管理——Root权限）
  • 然后取消 system 保护（见上面）
  • 下载 “GO谷歌安装器”（体积较大 4M），安装时可能会提示“system无法写入”
  • “Go谷歌安装器”有时候在 wifi 下无法安装 gapps，要切换到流量。。。
  • 再下载“谷歌安装器”（方形图标，只有几百KB 的一个），按提示安装所有的软件
  • 这时候进入 google play 可能会频繁提示已经停止运行（就是俗称的 FC）
  • 然后再次打开“GO谷歌安装器”，找到并下载里面的“FC修复工具”，进行修复（需要先给 root 权限）
  • 修复完成后，检查下“GO谷歌安装器”里是不是会有 Google play 和 Google 服务 两个软件包的升级提示，进行升级（估计是哪个小体积的“谷歌安装器”下载的文件比较古老了，但至少它能跳过 system 检查强制安装进去）
  • 然后全部升级后，打开 VPN 或者其他翻墙工具
  • 进入 google play 商店，输入账号密码
  • OK

去wifi 感叹号

20171204更新：在安卓8.1/8.0/7.1.2里如何消除x号（在老一点点版本是感叹号）呢？ 1.首先开启usb调试，然后用数据线连接电脑和手机。 2.然后解决好您的adb驱动问题，具体教程见：http://www.pixcn.cn/thread-1084-1-1.html 3.在电脑开始菜单-运行 输入cmd，打开命令提示符 依次输入下面语句 (以下办法支持安卓8.1/8.0/7.1.2/7.1.1) adb shell settings put global captive_portal_https_url https://www.google.cn/generate_204

然后开启飞行模式，然后关闭飞行模式解决！亲自测试有效！

下面语句对安卓7.1/7.0有效 adb shell settings delete global captive_portal_server
adb shell settings put global captive_portal_detection_enabled 0

20150223更新： 谷歌原生安卓系统当连接到移动数据网络或者WIFI网络的时候，其NetworkMonitor模块会向特定的服务器发起一个http的请求并利用收到的响应进行网络状态判断。由于谷歌的服务器被天朝屏蔽，所以导致没有返回值，这个时候谷歌安卓系统就会在信号或者wifi上打一个感叹号。

以下办法对安卓6.x/5.x有效果，对安卓7.0无效 解决的办法比较简单，修改接收请求的服务器地址： 首先搞定您pc的adb环境，然后连接您的手机和电脑，在命令提示符用下面命令来修改将服务器地址修改成http://www.g.cn

adb shell settings put global captive_portal_server www.g.cn

然后开启飞行模式，然后关闭飞行模式解决！

如果您想恢复的话，使用下面的语句：

adb shell settings delete global captive_portal_server
adb shell settings put global captive_portal_detection_enabled 1
