remote develper (via SSH 为例， 还有 WSL, Container)

在 本地 创建 public key ( ~/.ssh/id_rsa.pub)

ssh-keygen -t rsa -b 4096

复制 id_rsa.pub 内容到 远程服务器的 ~/.ssh/authorized_keys 文件里面去

# 本地是 mac/linux
ssh-copy-id your-user-name-on-host@host-fqdn-or-ip-goes-here

# 本地 windows, 使用 PowerShell
$REMOTEHOST="your-user-name-on-host@host-fqdn-or-ip-goes-here"

scp "$env:USERPROFILE\.ssh\id_rsa.pub" "${REMOTEHOST}:~/tmp.pub"
ssh "$REMOTEHOST" "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat ~/tmp.pub >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys && rm -f ~/tmp.pub"

提高安全性，不要在所有 SSH 机器上使用同一个 public key，因此需要在本地生成 key 时指定生成的文件

# mac/linux
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa-remote-ssh

# windows
ssh-keygen -t rsa -b 4096 -f "$env:USERPROFILE\.ssh\id_rsa-remote-ssh"

注意要删除里面的注释，这里仅仅是作为参考帮助（因为 json 不支持！！！）

{
  "protocol": "sftp", // 使用 sftp 协议
  "host": "10.8.118.139", // 服务器 ip
  "port": 22, // 访问端口号
  "username": "user_name", // 用来登录服务器的用户名
  "remotePath": "abs path of remote file", // 被同步文件在服务器上的绝对路径
  "uploadOnSave": true, // 在本地执行保存操作后，自动同步到服务器
  "syncMode": "update",
  "passive": false,
  "debug": false,
  "privateKeyPath": "/Users/username/.ssh/id_rsa", // 本地保存的可以用来登录服务器的 ssh key
  //"passphrase": null,

  //"interactiveAuth": true,

  "watcher": {
    "files": "**/*",
    "autoUpload": false,
    "autoDelete": false
  },
  "ignore": [
    // 设置不被同步的文件和路径
    "\\.vscode", // .vscode 路径
    "\\.idea",
    "*.pyc", // 拓展名为 .pyc 的文件
    "\\output",
    "\\.git"
  ]
}

注意，如果你的 mac 上 .ssh/id_rsa 是比较新的非 PEM 格式。要把它转换成 PEM 格式。

cd ~/.ssh

# 备份下先
cp id_rsa id_rsa.bak

# 转换成 PEM 格式
ssh-keygen -p -m PEM -f ~/.ssh/id_rsa
