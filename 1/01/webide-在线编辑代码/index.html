code-server

sudo docker run -it -p 127.0.0.1:8080:8080 -v "${HOME}/.local/share/code-server:/home/coder/.local/share/code-server" -v "$PWD:/home/coder/project" codercom/code-server:v2

docker 版有权限问题。。报错

docker: Error response from daemon: OCI runtime create failed: container_linux.go:346: starting container process caused “chdir to cwd ("/home/coder/project") set in config.json failed: permission denied” : unknown.

https://github.com/cdr/code-server/issues/459 提到解决方法

Fixed by manually create the folders /home/coder/project (无效)

# 这个目录保存等下容器要生成的 extensions 等，所以要可写
sudo mkdir -p ~/.local/share/code-server/extensions

sudo chown -R username ~/.local/share/code-server

# 一定要进入某一个当前用户可写的项目目录，否则 ${PWD}会因为无权限报上面的错误，例如我在 home 目录就遇到了
cd Your-Project-Home

docker run....

# 当然写成固定的路径也可以 /home/wwwroot/projects:

# 如果要外网访问， -p 改成 0.0.0.0:8080

会给出一个临时密码，如 5bab9e8dca3323294c02553b 要使用固定的密码，在启动时加参数

--PASSWORD xxxxx

如果要部署 https give code-server a certificate with code-server –cert followed by the path to your certificate. Additionally, you can use certificate keys with –cert-key followed by the path to your key. If you pass –cert without any path code-server will generate a self-signed certificate.

-d 后台运行

最后

sudo docker run -dit --name code-server -p 0.0.0.0:8080:8080 -e PASSWORD=123456 -v "${HOME}/.local/share/code-server:/home/coder/.local/share/code-server" -v "/home/wwwroot:/home/coder/project" codercom/code-server:latest

访问后会比较慢。。。（可能也和服务器配置、网速有关？）

升级 docker image. (从 help 里可以看到版本和官方 release 的差异)

sudo docker pull codercom/code-server:latest
sudo docker stop code-server
sudo docker rm code-server
sudo docker run -dit....

进去后，如果有提示升级，马上升到最新版。。。

进入后，一直

Attempting to reconnect in 15 seconds…

控制台看到：

ERR Error: WebSocket close with status code 1006

和

WebSocket connection to ‘ws://xx.xx.xx.xx:8080/?reconnectionToken=9ce641f5-12f6-41bb-9edc-5aa795499b6e&reconnection=true&skipWebSocketFrames=false’ failed: Error in connection establishment: net::ERR_EMPTY_RESPONSE

需要到防火墙打开端口。(宝塔 + 腾讯云/阿里云安全组，出 + 入 都要开)

如果是走 frp. 除了 http 协议外，还要添加一个 tcp 的协议

[WS]
#访问协议
type = tcp
#内网web服务的端口号
local_port = 8082
remote_port = 8082

还是没用，要在 nginx 反向代理里增加下面的内容

map $http_upgrade $connection_upgrade {
    default upgrade;
    '' close;
}

server {
    localtion / {
        # ... 略
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
    }
}

https://www.xncoding.com/2018/03/12/fullstack/nginx-websocket.html

https://www.hi-linux.com/posts/42176.html

回到服务器， docker ps -a 发现 Exit 了。。 全部关掉这些。 重新启动（可能是启动太多次，冲突了。。资源不够什么的）

运行一会又不动白屏了。 回到服务器一看。。。哎哎。 容器又是 Exited

重新启动几次。 打开文件不多的时候还可以，文件、插件多了内存就不够了。。。。

问题

ctrl + shift + c 打开 chrome 的 console, docker 必崩。。。。 所以要把这个快捷键绑定到 code-server 某个事件上

binary 版

https://github.com/cdr/code-server/releases

chmod +x code-server 解决问题。

反应很快，比docker 体验好(不过打开项目后，打开几个文件装几个插件，内存占用就差不多了。。。)。 也没有权限问题。

解决关闭ssh窗口进程杀掉问题 nohup > 2>&1 & 注意！！nohup命令执行后，不要直接关闭终端，使用exit命令退出会话

使用中问题：不知道是不是内存不够还是1M水管/国外网络延迟，intellisense 非常慢，基本不可用状态。

terminal 是 docker 环境的。

不过可以 ifconfig 查看到 host 的 ip 然后 ssh 登录上去到宿主机即可。

自己修改 docker 文件。例如增加 php 等。 便于安装 phpformarter 等需要执行路径的

最后总结，操作体验十分接近 vscode, 机器性能强且 局域网访问速度足够的话很快。 但如果通过 frp 的穿透，带宽不够 速度和方便性远不如 cloud9 （直接连接 server 不走 ftp 1M 带宽也挺快，估计是 frp 中转的延迟太大了。）

self-hosted cloud9

不知道用了什么黑科技，运行起来挺快的，普通打开几个文件后，占用内存好低。（一点都不像也是 node 的）

git clone https://github.com/c9/core.git c9sdk
cd c9sdk
scripts/install-sdk.sh

node server.js

--settings       Settings file to use
--help           Show command line options.
-t               Start in test mode
-k               Kill tmux server in test mode
-b               Start the bridge server - to receive commands from the cli  [default: false]
-w               Workspace directory
--port           Port
--debug          Turn debugging on
--listen         IP address of the server
--readonly       Run in read only mode
--packed         Whether to use the packed version.
--auth           Basic Auth username:password  (-a)
--collab         Whether to enable collab.
--no-cache       Don t use the cached version of CSS


# http://localhost:8181/ide.html 


# 默认只能在本地访问， 如果要在外面也能访问 并且加上一个用户验证（比较弱的验证。。。）
node server.js -l 0.0.0.0 -p 8080 -a username:password


# 默认是在 /cloud9sdk 目录下启动 workspace. 但我们一般希望是在自己的项目如 /var/www 下
# 你可以创建多个命令来实现，一个端口对应一个项目目录
node server.js -w /var/www/project1 -p 8081
node server.js -w /var/www/project2 -p 8082

git pull origin master

过滤一些目录，如node_modules

在 project_setting 里面， find in files , ignore…

node_modules/ /vendor

必须要关掉页面，重新进入 cloud9 才能生效

修改了 运行项目目录、用户后提示 Failed to write to ‘state.settings’. File or folder ‘state.settings’ does not exist.

如果你是用 newuser 这个用户启动项目的： “node server.js xxxxx”, 一定要把项目的所有权也改成该用户。 ‘chown newuser /home/wwwroot/project’, 否则会报这个错误

AWS Cloud9 (收购了 cloud9 改成云服务了)

https://console.aws.amazon.com/cloud9

可以连接到 amazon 的 ec2 或自己的 linux(只用webide， 不需要付费了。)

连接到自己的 linux server， 需要把 public SSH key 复制到服务器用户目录的 .ssh/authorized_keys…. 里面去

会提示你正在 install cloud9 ide 所需要的 package.(glibc-statics node-pty Nak sqlite node-qty 等) 国内服务器可能比较慢

AWS Cloud9 无需额外付费。如果您为 AWS Cloud9 开发环境使用 Amazon EC2 实例，则您仅为用于运行和存储您的代码的 计算和存储资源（例如 EC2 实例、 EBS 卷）付费。您还可以通过 SSH 将您的 AWS Cloud9 开发环境连接到现有 Linux服务器（例如本地服务器）而无需额外付费。

问题 因为是“远程访问”， 所以文件树展开速度非常慢（不过高亮很快 毕竟服务器本地先加载后渲染）。 国外 AWS -> 国内服务器 而且智能提示有点傻。。。经常不显示

可以修改 server 的 region 改成亚洲的试试（默认是美国） 无法安装 vscode 的各种牛逼插件。。。 只能在 project setting 里启用自带的 js library 支持。
