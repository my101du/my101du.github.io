构建工具，grunt或者gulp 打包工具 webpack 他的目的就是把有依赖关系的各种文件打包成一系列的静态资源

开始

下载与安装(全局)

npm install -g webpack

配置文件

webpack.config.js 这个是webpack的管理配置文件

// 以CMD的格式导出模块
module.exports = {
    // 入口文件，指向app.js
    entry: './app.js',
    // 出口文件
    output: {
        // 文件名，将打包好的导出为bundle.js
        filename: './bundle.js'
    }
}

主要分为

  • entry 用哪个文件作为项目的入口
  • output 处理完成的文件放在哪里
  • module 要用什么不同的模块来处理各种类型的文件

如入口文件 app.js

import bar from './bar'

bar()

入口文件里面引入了模块 bar.js

export default function bar () {
    console.log('bar')
}

然后在 html 里引入入口文件

// page.html 这个是html目录文件，这个文件引入入口文件
<html>
<head>
    <title>1、最简单的webpack实例</title>
</head>
<body>
<script src="./bundle.js"></script>
</body>
</html>

执行 webpack 打包

# 默认配置文件
webpack

# 或者指定不同的配置文件 （如 dev prod 等）
webpack --config webpack.config.js

# 自动生成 打完后的bundle.js文件

npm 脚本- 指令

例如常常输入的 npm run dev npm run test等，实际上是用到了 package.json 里的 scripts 属性

{
    "name": "simple-command",
    "version": "0.0.1",
    "scripts": {
        "test": "webpack --config webpack.config.js"
    }
}

入口、多入口，出口文件

如果只有一个入口文件

entry: './app.js',

相当于

entry: {
    main: './app.js'
}

多入口

一个项目里，允许有A、B两个html文件，他们之间是不同的入口文件（比如一个是用户入口页，一个是管理入口页）。first.js 和 second.js 共享 common.js 模块

. |____first.html | |____first.js | | |____common.js |____second.html | |____second.js | | |____common.js

因此配置多个人口就用这个格式

entry: {
    first: './first_entry.js',
    second: './second_entry.js'
},

因为这是“把多入口文件打包到一个文件里”，因此运行的时候会报错

Multiple assets emit to the same filename

需要将outoput改成 [name].js, 名字根据入口文件决定, [name]表示文件名自动匹配入口文件的key

output: {
    filename: './dist/[name].js'
}

分别打开first.html和second.html来查看效果

出口

path 与绝对路径

除了只写 filename 外，还可以把目录和文件分开写。

注意， path 必须使用 __dirname变量来指定当前目录的绝对路径（指从根目录到当前目录的路径（不带最后一个/））而不能用相对路径./dist

output: {
    path: __dirname + '/dist',
    filename: 'dist.js'
}

占位符、hash 文件名

前面的“多入口”里已经用到了 [name]这个占位符。 有时候需要每次 build 的时候，都生成不同的文件名，避免浏览器会因为 bundle.js 文件名相同，导致每次就加载旧的缓存文件，这时候可以使用 [hash],打包后得到类似的文件 ：dist.5099da45ae9fc763852d.js

filename: 'dist.[hash].js'

注意： 即使文件没有改变，每次生成的结果也不同， 如果想让模块没有改变时，hash值不改变，应该改成 chunkhash

filename: 'dist.[chunkhash].js'

插件 HtmlWebpackPlugin： 自动创建html模板，并让该html文件引入打包好的js文件

这样会产生一个问题，每次 build 都会得到一个新的文件， 那 html 文件里引入的 dist.xxxxx.js 也要去改动，太麻烦了

安装一个 webpack 插件html-webpack-plugin， 并且在当前项目安装 webpack

npm install --save-dev html-webpack-plugin

npm install --save webpack

// 引入插件
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    //...
        
    // 将插件添加到webpack中
    plugins: [
        new HtmlWebpackPlugin({
            title: 'My HTML'
        })
    ]

运行 npm run test，会发现在dist文件夹下，除了之前的js文件，还出现了一个html文件，而这个html文件引入了我们打包好的js文件。

然后你还会想到，我除了自己自己开发的js 代码，很可能还需要

  • 以 CDN 的方式引入 jQuery，那么这个生成的 html 文件里怎么生成它们呢？
  • 自定义生成的 html 文件名？

那需要查看 HtmlWebpackPlugin 的文档 https://github.com/jantimon/html-webpack-plugin#configuration，增加一些配置

plugins: [
    // 这里是添加的插件
    new HtmlWebpackPlugin({
        title: 'title', // html的title（就是title标签里的东西）
        filename: 'index.html', // 重写后的html文件名，默认是index.html
        template: './demo.html',    // 这个就是那个模板文件，不会改动原有的内容，而是在原来html文件的末尾，将打包编译好的文件添加进去
    })
]

让它能够从”模板“里额外加入打包的js文件， 如 demo.html 如下

<!doctype html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport"
          content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">
    <meta http-equiv="X-UA-Compatible" content="ie=edge">
    <title>引入了bootstrap</title>
    <link rel="stylesheet" href="https://cdn.bootcss.com/bootstrap/3.3.7/css/bootstrap.min.css"
          integrity="sha384-BVYiiSIFeK1dGmJRAkycuHAHRg32OmUcww7on3RYdg4Va+PmSTsz/K68vbdEjh4u" crossorigin="anonymous">
</head>
<body>
<h1>
    <a href="http://www.baidu.com">baidu</a>
</h1>
<style>
    .table {
        width: 500px;
    }
</style>
</body>
</html>

Loader

引入css的时候，我们可能用css-loader,处理一些小图片，我们为了减少请求数，所以需要将图片转为base64字符串，这个时候要用url-loader,处理使用es6等高版本js代码的，我们需要使用babel-loader….

作用：对模块的源代码进行转换 例如输出一个有 es6、7 语法的 js 文件，将其编译转换为 es5 语法

先通过npm安装loader，引入package.json中

npm i -s css-loader

然后配置 webpack, 将 loader 添加到 module 下的 rules里面进行配置

module.exports = {
    //... entry, output...
    module: {
        rules: [
        // loader放在rules这个数组里面
        ]
    }

匹配方式

webpack 从入口文件（ entry ）开始解析代码，遇见 es6 的 import，或者 require 语法，导入对应的模块。

每次加载一个新模块时（包括入口文件）（每个文件都被 webpack 视为模块），就会触发能匹配成功的loader。 比如 test: /\.css$/ 会匹配成功 .css 结尾的文件，然后模块被该 loader 所处理

处理方式

链式

如下，首先被 css-loader 进行处理，打包成模块，然后被 style-loader 处理(顺序是从后往前执行的)，在 加载 js 文件时，将 css 代码插入到 html 文件之中

{
    test: /\.css$/,
    use: [
        'style-loader',
        'css-loader'
    ]
},

异步/同步

支持查询参数/options

{ test: /\.png$/, use: [ "url-loader?mimetype=image/png" ] }

相当于

{
    test: /\.png$/,
    use: [
        {
            loader: 'url-loader',
            options: {
                mimetype: 'image/png'
            }
        }
    ]
}

loader 也有插件

比如 postcss-loader 的 autoprefixer 插件，用于自动添加 css 兼容性前缀

常用 loader 列表

需要查看各自的文档

  • babel-loader：用于将es6、es7等语法，转换为es5语法；
  • css-loader：用于处理css文件（主要是处理图片的url）；
  • style-loader：将转换后的css文件以 style 标签形式插入 html 中；
  • postcss-loader：一般用于添加兼容性属性前缀；
  • less-loader：以 less 语法来写 css ；
  • url-loader：用于将图片小于一定大小的文件，转为 base64 字符串；
  • file-loader：url-loader 不能转换 base64字符串 的文件，被这个处理（主要用于设置打包后图片路径，以及CDN等）；
  • html-withimg-loader：用于加载html模板；

babel-loader

实际上包括一大堆需要安装的（webpack 也在里面）

npm install --save babel-loader babel-core babel-preset-env webpack

创建babel规则文件.babelrc

{
  "presets": [
    [
      "env",
      {
        "modules": false,
        "targets": {
          "browsers": [
            "> 1%",
            "last 2 versions",
            "not ie <= 8"
          ]
        }
      }
    ]
  ]
}

然后就可以在代码里用 ES6 语法了(像这里的箭头函数)

let foo = () => {
    console.log('1')
}
foo()

运行 npm run test 得到 dist.js 文件，代码已经被成功转为非es6语法

var foo = function foo() {
  console.log('1');
};
foo();

但是无法转换 Promise、Set 等等。。。。又要做额外的工作

安装插件

npm install babel-runtime --save
npm install babel-plugin-transform-runtime --save-dev

修改 .babelrc。

{
  "presets": [
    "babel-preset-env"
  ],
  "plugins": [
    "transform-runtime"
  ]
}

babel-runtime对应 plugins 里的 transform-runtime, 它干什么的？

当添加了 babel-loader 后，默认运行 webpack 命令，会给每个文件都添加辅助代码（以便可以使用新的特性的代码），因此会导致文件过大。为了禁用 babel 每个文件的 runtime 注入，引入 babel-plugin-transform-runtime 并且使所有辅助代码从这里引用。

最后需要把 babel-loader 的规则添加到 webpack.config.js 里去

{
    test: /\.js$/,
    exclude: /node_modules/,
    loader: 'babel-loader'
}

支持新特性

es6的Promise，Set，以及es7中的async/await都可以在代码里使用了。

但对于一些新特性（还没有正式添加到 ES7 规范里面） 正常来说 runtime 是不支持的。因此要安装babel-preset-stage-2 来指定 stage 版本，通常安装到 stage-2 就足够了，如果还不行，再安装 stage-1 或者 stage-0

npm install --save babel-preset-stage-2

在 .bablerc 文件的 presets 数组里，babel-preset-env下方添加stage-2

{
  "presets": [
    "babel-preset-env",
    "stage-2"
  ],
  "plugins": [
    "transform-runtime"
  ]
}

Webpack & Babel

参考 https://github.com/huangtengfei/blog/issues/17

Webpack 是一个前端模块加载兼打包工具，可以对 JS、CSS 和 图片 等都作为模块来使用和处理。对不同类型的需要编译的文件，需要使用相应的加载器（比如用 babel 转译 ES6）

另外，由于 Webpack 需要一个入口文件来进行分析和处理，需要先将 React 组件引入到一个主文件。

编辑 webpack.config.js 配置文件

var path = require('path');
var webpack = require('webpack');

var ROOT_PATH = path.resolve(__dirname);
var APP_PATH = path.resolve(__dirname, './components/index.js');
var BUILD_PATH = path.resolve(__dirname, './build');

module.exports = {
  entry: APP_PATH,
  output: {
    path: BUILD_PATH,
    filename: 'bundle.js'
  },
  module: {
    loaders: [{
      test: /\.jsx?$/,
      loaders: ['babel-loader?presets[]=es2015,presets[]=react']
    }]
  }
}

最后，需要将编译打包后的文件 bundle.js 引入到 index.html 中

构建

执行 webpack.config.js 中的构建任务，生成 bundle.js，这个操作可以在 package.json 中配置：

"scripts": {
  "build": "webpack"
}

执行 npm run build 完成构建

启动服务器

但这种方式显得略 low，一是它是双击以文件的形式打开 HTML 页面，二是每次有更改都要手动执行 npm run build 重新打包。

一种更好的方式是启动一个静态资源服务器，监听文件内容修改并自动打包。在这里用的是前面安装好的 webpack-dev-server ，在 package.json 中配置：

"scripts": {
  "dev": "webpack-dev-server --devtool eval --progress --colors --hot --content-base build"
}

简单解释一下 dev 中各个参数的含义：webpack-dev-server 在 localhost:8080 建立一个 Web 服务器；–devtool eval 映射编译好的源码，用于调试；–progress 显示代码打包进度；–colors 表示在命令行中显示颜色； –content-base 来指定 server 启动后的内容目录。

执行 npm run dev 启动 server，此时打开 http://localhost:8080 ，即可看到效果。修改一下 Hello 或者 World 组件中的内容，刷新页面，你会发现浏览器中内容也相应改变了。

自动刷新

前面实现了对文件修改的监听和自动打包，但浏览器还需要手动刷新。其实可以在 Webpack 的配置文件中增加一个入口点，实现自动刷新。

entry: [
    'webpack/hot/dev-server',
      'webpack-dev-server/client?http://localhost:8080',
      APP_PATH
  ],

处理样式

不管什么类型的资源，Webpack 都需要相应的 loader 来处理。对于普通的 css，需要 css-loader 和 style-loader 两种 loader，前者会遍历所有 css 并对 url() 处理，后者将样式插入到页面的 style 标签中。对于预编译 css 语言，还需要额外的 loader，比如 less-loader 或 sass-loader，这里用 less 举例。

首先，安装 loader ：

npm install less-loader css-loader style-loader --save-dev

然后，在 webpack.config.js 中配置 loader，注意 loader 的处理顺序是从右到左的：

{
    test: /\.less$/,
    loader: 'style!css!less'
}

配置完之后要重新运行一下 npm run dev

接着，在 Hello 和 World 组件中各添加一个样式文件，这里用 Hello/index.less 举例：

@lightgrey: #ccc;

h1 {
  background: @lightgrey;
}

最后，在 components/index.js 中引入 less 文件：

import './Hello/index.less';
import './World/index.less';

处理图片

为了减轻网络请求的压力，有时候会有将小图片转成 BASE64 字符串的需求，这个可以通过 url-loader 来实现。

同处理样式中的操作，首先安装 url-loader ：

npm install url-loader --save-dev

配置 webpack.config.js，然后重新重新运行一下 npm run dev

{
    test: /\.(png|jpg)$/,
    loader: 'url?limit=50000'
}

接着，在 Hello 组件下建一个 imgs 文件夹，并放入一张小于 50K 的 bg.png 做背景图片，然后修改 Hello/index.less 文件：

h1 {
  background: url('./img/bg.png');
}

切换到浏览器，审查元素，看一下图片是不是转成 BASE64 字符串了呢~

补充

loader 的写法

在使用 Webpack 打包小节 webpack.config.js 的编写中，babel-loader 最初我是这样写的：

loaders: [{ test: /.jsx?$/, loader: ‘babel-loader’, query: { presets: [‘es2015’, ‘react’] } }] 后来为了使用热加载，引入了 react-hot ，loader 要改为 loaders，即：

… loaders: [‘babel-loader’, ‘babel-loader’], query: { presets: [‘es2015’, ‘react’] } …
这时在执行构建的时候，会报错 Error: Cannot define ‘query’ and multiple loaders in loaders list，于是改成这种写法。不过后来直接用 webpack-dev-server 的 –hot 选项，也没引入 react-hot 了，但此处写法仍保留，便于后面扩展 loader 。

babel 升级

Failed to compile.

./src/index.js Error: The ‘decorators’ plugin requires a ‘decoratorsBeforeExport’ option, whose value must be a boolean. If you are migrating from Babylon/Babel 6 or want to use the old decorators proposal, you should use the ‘decorators-legacy’ plugin instead of ‘decorators’.

npx babel-upgrade --write --install

自动更新 package.json 里的 babel/cli babel/core 等版本 6->7

并且 package.json 或者 babelrc 里的配置也会变成新的语法

npm-check -u

babel 支持 mobx 的 @observe 等装饰器语法

默认会报错，提示用 export @dec class 来替代

只需要修改 package.json 里 eslintConfig 配置，或者 .eslintrc 文件

"eslintConfig": {
    "extends": "react-app",
    "parserOptions": {
      "ecmaFeatures": {
        "legacyDecorators": true
      }
    }
  },

或者修改 .babelrc (推荐单独)里

{
  "presets": [
  	"@babel/preset-env",
  	"@babel/preset-react",
  ],
  "plugins": [
	  [
	  	"@babel/plugin-proposal-decorators", 
	  	{ 
	  		"legacy": true 
	  	}
	  ]
	]
}
