WordPress (hereinafter referred to as “WP”) content management, as well as some specific plug-in related operations.

Login

login address

Suppose you want to log in the WP site is www.sample.org, then the background is usually default login address is

www.sample.org/wp-admin

However, in order to improve the security of the system and prevent network hackers from using the scanner to burst passwords, the background address is usually changed to something else, for example

www.sample.org/S3v9_KkR2

login interface

Understand content management

WP has the two most common “content”:

  • Page
  • Article

Page can usually be considered as fixed URLs, generally not much. For example,

  • About business / about
  • Contact / contact
  • …

Article usually can be considered as website constantly updated, increased content, generally with the development of the website, the number will be more and more, and the link format has certain rules. E.g:

  • News / 1, news / 2, news / 3 ….
  • Company Monthly Report / 201701, report / 201702, report / 201703 …

  • Content Area: You can copy & paste any web content into the edit box. (If you want to upload your own local pictures, videos, etc., see the later chapters media files)

Set status, privacy, release time

As shown above, several parameters of the page can be modified to achieve a “delayed” or “internal” publish page.

  1. “State”, if it is a draft, the article will not be released
  2. “Privacy”: pages can be public; public (but need a password); privately owned three
  3. “release time”: can be set to a future time, temporarily hidden, by the time, the page is automatically released.

Modify the permanent link for future review of the content

If you do not summarize the article’s permanent link in some way, the content may increase. (Imagine having a meeting, held once a year, is not clear at all if they are titled / company-annual-meeting)

As shown in the figure, in front of automatically generated permanent link news/2018/01 this prefix, respectively, news categories, year, month. This basically avoids the problem of duplicate names

Select the appropriate article category

To show the headlines of the latest news in the right area on the front page and to better categorize articles, select the appropriate category for the article.

general

  • The company’s major news (acquisitions, annual reports), etc., into Corporate News
  • New product launches, some Press Release from other BUs, to Product News
  • If it is the show, routine notification, etc., into Events

Caching and compression

When publishing pages / articles, the WP Rocket plugin allows users to customize the cache and compression parameters for a particular piece of content.

Sometimes, you must turn off the cache

If there are some pages that have some programs / content that need live update, or some add-ons will dynamically process the contents, there must be checked “Do not cache the page”

E.g:

  1. Home page (after adding a new news, let it’s title appear immediately)
  2. News listings page (same as above)
  3. Enable multi-language version of the article, the page (because the internal need to dynamically read the parameters, select the appropriate text content data)

manually clear the cache
