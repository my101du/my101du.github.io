Markdown

wordpress.com 默认是支持 markdown 的。在后台的“设置”——“撰写”里面启用

但我们自己搭建的，要安装插件

（1）使用Jetpack插件支持Markdown，但是这个 Jetpack 比较大。Jetpack支持的是Markdown Extra，和原始Markdown有稍许差别

（2）WP-Markdown，据说有时候解析一些符号有问题 stores the processed HTML

（3）[推荐]WP Editor.md 边编辑边实时预览

（4）TinyMCE的增强插件（TinyMCE Advanced）设置中增加 markdown 支持，但是无法“自动”解析，如果从别处复制来，要手动回车

（5）https://github.com/flyhttp/wp-gruber-markdown

WordPress 装 Mytory Markdown 本地写 MD 文件后推送到 Github，WordPress 填文件地址就行 最大亮点是 MD 更新只需推送 Github，WordPress 会自动更新日志。 另外有一个 For Dropbox 版本，可以用 Dropbox 储存 MD 文件。

具体可以参考 使用 Mytory Markdown on WordPress 构建 Markdown 发布系统 https://cloudlet.info/t/643

代码高亮

Crayon Syntax Highlighter插件来支持代码高亮（https://www.zhihu.com/question/28276750）

而Crayon Syntax Highlighter和Jetpack完全兼容

在 Crayon Syntax Highlighter 设置中不要勾选“捕获 反引号 为 标签”，以防和Jetpack插件冲突。
