wp cli 命令行操作

permalinnk: wp-cli-of-wordpress tags:

  • wordpress

是一个命令行操作 Wordpress 工具。例如安装插件、删除操作等。

https://github.com/wp-cli/wp-cli

可以在本地运行测试，也可以使用 travis CI 在远程运行

本地运行测试

安装 PHPUnit

全局

wget -O phpunit https://phar.phpunit.de/phpunit-7.phar
chmod +x phpunit
./phpunit --version

mv ./phpunit /usr/local/bin/

对某个项目用 composer

composer require --dev phpunit/phpunit ^7

./vendor/bin/phpunit --version

安装&升级 WP-CLI

# download
curl -O https://raw.githubusercontent.com/wp-cli/builds/gh-pages/phar/wp-cli.phar

# check it's working
php wp-cli.phar --info

# make the file executable
chmod +x wp-cli.phar
sudo mv wp-cli.phar /usr/local/bin/wp

# test!
wp --info

# update
wp cli update

命令行 支持 tab 自动完成（Tab completions）

下载 https://raw.githubusercontent.com/wp-cli/wp-cli/master/utils/wp-completion.bash

然后在 ~/.bash_profile 里引用source /path_of_it/wp-completion.bash它。并运行source ~/.bash_profile命令使之生效

如果使用zsh,则添加下面的（load and start bashcompinit before sourcing）

autoload bashcompinit
bashcompinit
source /FULL/PATH/TO/wp-completion.bash

创建插件

略，假设名字叫 my-plugin

自动生成插件的 tests 文件

下面的命令会生成所有 phpunit 所需文件，包括 .travis.yml 等。

wp scaffold plugin-tests my-plugin

初始化本地测试环境

install-wp-tests.sh默认会在 /tmp下下载、创建一个 wordpress 副本（latest 表示最新版，你可以自己改成 3.x 4.x 来测试兼容性），以及一个测试数据库wordpress_test（在运行时修改数据，）

cd wwwroot/wordpress/wp-content/plugins/my-plugin

# 根据实际情况修改 mysql 的验证信息
bash bin/install-wp-tests.sh wordpress_test root '' localhost latest

运行测试

phpunit

出现这样的提示

PHP Fatal error: Class PHPUnit_Util_Test may not inherit from final class (PHPUnit\Util\Test) in /private/var/folders/7g/hk8r20fd003ftkl_q69mkds40000gn/T/wordpress-tests-lib/includes/phpunit6-compat.php on line 18

是因为 wordpress 的版本（或者 PHP）与 PHPUnit 的版本不匹配

例如我这个项目是 Wordpress 4.9.4 的版本。 全局安装的 PHPUnit 是（phpunit –version）7.5.2

（1）全局的 phpunit 降级

wget -O phpunit https://phar.phpunit.de/phpunit-6.phar

（2）方法2：在 wordpress 插件的目录下, 单独指定安装 PHPUnit 6

composer require --dev phpunit/phpunit ^6

再运行vendor下的 phpunit ，就不报这个错误了.

如果代码里有

session_start();

session_regenerate_id(true);

会提示

Warning: session_start(): Cannot send session cookie - headers already sent by (output started at /private/var/folders/7g/hk8r20fd003ftkl_q69mkds40000gn/T/wordpress-tests-lib/includes/bootstrap.php:72) in ……

改成(屏蔽错误)， 更好的处理方法是判断在 TEST 环境下不运行

@session_start();
@session_regenerate_id(true);

写测试文件

在tests目录下 test-wp-meta-verify.php

在方法前面添加test前缀，则被视为单元测试，另外最佳实践是为测试的类添加 Test 前缀

class WP_Meta_VerifyTest extends WP_UnitTestCase
{
    public function setUp()
    {
        parent::setUp();

        $this->class_instance = new WP_Meta_Verify();
    }

    public function test_google_site_verification()
    {
        $meta_tag = $this->class_instance->google_site_verification('B6wFaCRbzWE42SyxSvKUOyyPxZfJCb5g');
        $expected = '<meta name="google-site-verification" content="B6wFaCRbzWE42SyxSvKUOyyPxZfJCb5g">';

        $this->assertEquals($expected, $meta_tag);
    }

    public function test_bing_site_verification()
    {

    }
}

支持 wordpress 自有方法的 WP_UnitTestCase 类

注意

WP_UnitTestCase这个类，和其他php应用使用的原生PHPUnit_Framework_TestCase不同，它还支持 wordpress 自身的特有函数。

WordPress Test Assertions

//Assertions for Errors

$this->assertWPError($actual, $message)
$this->assertNotWPError($actual, $message)
$this->assertIXRError($actual, $message)
$this->assertNotIXRError($actual, $message)


//Assertions to test WP_Query for conditional tags

$this->assertQueryTrue($args)

//E.g. $this->assertQueryTrue('is_single', 'is_feed') means is_single() and is_feed() must be true to pass.

WordPress Object Factories

factories:

可以简化创建 post page user taxonomies 等

WordPress Trac 查看更多 factories https://core.trac.wordpress.org/browser/trunk/tests/phpunit/includes/factory

//create() – returns the object ID of the created object
//create_and_get() – creates and return the entire object

//create a tag
$tag_id = $this->factory->tag->create( array( 'slug' => 'tag-slug' ) );

$cat_with_slud_id = $this->factory->term->create( array( 'taxonomy' => 'tax', 'slug' => 'tax-slug-2' ) );
//create posts
$page_id = $this->factory->post->create( array( 'post_title' => 'My Page') ); $this->factory->post->create_many(15);

setUp()等方法

它在所有单元测试之前运行。这里$this->class_instance = new WP_Meta_Verify();表示创建一个WP_Meta_Verify类的实例。

然后就可以在其他方法里使用了。例如test_google_site_verification()中$meta_tag = $this->class_instance->google_site_verification

类似的方法还有tearDown()，setUpBeforeClass()，tearDownAfterClass()等

public function setUp() {
		parent::setUp();  
		
		$this->editor_user_id = $this->factory->user->create( array( 'role' => 'editor' ) );
		//set the current user to the newly created user
		wp_set_current_user( $this->editor_user_id );
	}
		
	public function tearDown() () {
		parent::tearDown();  
	}

表单提交

function test_nds_custom_meta_update_for_editors() {
        $factory_editor_id = $this->factory->user->create( array('role' => 'editor') );                                              
        $get_editor_user_meta = get_user_meta($factory_editor_id, 'preferred_browser', true );
        
        //for new users with editor role the default meta value for preferred_browser is chrome
        $this->assertEquals($get_editor_user_meta, 'chrome');
        
        //spoof $_POST
        $_POST['preferred_browser'] = 'opera';
        
        //make sure we are admin
        wp_set_current_user(1);
        
        //call the update method
        nds_update_usermeta_field_browser($factory_editor_id);
        $get_editor_user_meta = get_user_meta($factory_editor_id, 'preferred_browser', true );
        
        //test the meta value was updated correctly
        $this->assertEquals($get_editor_user_meta, 'opera');
}
