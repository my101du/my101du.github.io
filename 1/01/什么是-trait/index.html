什么是 Trait

Trait 是一个在 PHP5.4 以后才出现的概念。

因为 PHP 只能单继承。

class A {
    public $a;
}

class B {
    public $b;
}


// 正常
class C extends A {
    
}

// 错误，不支持多继承
class D extends A, B

那么如果有一个 方法 需要在很多的类中使用时，通常我们写一个 基础类，在基类中实现这个方法，然后所有类都继承这个基类。

比如说，手机这个基类，有开机，关机等动作。

然后三星手机和小米手机和华为手机都继承这个基类，那么就都有了 开机，关机动作 也就是说 一般使用“继承”时，这些类应该都比较类似。

但是，我们现在有另外几个类收音机，行车记录仪器， 他们也有开机，关机动作，如果让它们继承手机这个基类，虽然实现了目的，但太令人费解了。

能不能让一个方法在很多的类中使用，同时又不会被强制放在某个类里面呢？

Trait 是一种代码复用技术，为PHP的单继承限制提供了一套灵活的代码复用机制，可以混入一个或多个现有的PHP类中。

使用 Trait

在下面的例子中

虽然 A_1 和 B_1 分别继承了不同的类（A 和 B），但是可以 user 相同的 Trait T

trait T {
    function t1() { /*1*/ }
    function t2() { /*2*/ }
}

class A {

}

class B {

}

class A_1 extends A {
    use T;
    
    /* ... */
}

class B_1 extends B {
    use T;
    
    /* ... */
}

优先级

从基类继承的成员会被 trait 插入的成员所覆盖。优先顺序是来自当前类的成员覆盖了 trait 的方法，而 trait 则覆盖了被继承的方法。

自身方法 > trait的方法 > 继承的方法

trait HelloWorld {
    public function sayHello() {
        echo 'Hello World!';
    }
}

class TheWorldIsNotEnough {
    use HelloWorld;
    public function sayHello() {
        echo 'Hello Universe!';
    }
}

$o = new TheWorldIsNotEnough();
$o->sayHello();//输出是 Hello Universe! 因为自身的方法覆盖了 trait 的方法

多个trait的使用

trait Hello {
    public function sayHello() {
        echo 'Hello ';
    }
}

trait World {
    public function sayWorld() {
        echo 'World';
    }
}

class MyHelloWorld {
    use Hello, World;
    public function sayExclamationMark() {
        echo '!';
    }
}

$o = new MyHelloWorld();
$o->sayHello();
$o->sayWorld();
$o->sayExclamationMark();

覆盖 trait 的方法

https://stackoverflow.com/questions/11939166/how-to-override-trait-function-and-call-it-from-the-overridden-function

Trait 不是类，所以不能使用继承类的方式去重写。以下是 Trait 里继承函数的写法

trait A {
    function calc($v) {
        return $v+1;
    }
}

class MyClass {
    use A {
        calc as protected traitcalc;
    }

    function calc($v) {
        $v++;
        return $this->traitcalc($v);
    }
}
