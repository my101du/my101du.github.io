任务调度（计划任务）

slug: laravel-crontab tags:

  • laravel

任务调度

以往需要运行一个计划任务，必须 SSH 登录服务器通过crontab -e来创建，有很多的弊端

1.当需要创建和维护很多个计划任务时，要不断 SSH 登录服务器，特别繁琐 2. cron 任务列表里大量类似的 * * */5列表，阅读起来很困难 3. 系统的 crontab 没办法进行版本控制

因此可以使用 Laravel 的任务调度功能。

Laravel 的任务调度中心

只需要把它（仅一条）添加到系统的计划任务里即可，每分钟运行一次，由它来接管 Laravel 应用中其他所有的计划任务。

* * * * * php /path/to/artisan schedule:run >> /dev/null 2>&1

添加新任务（定义调度）

首先找到在 app/Console/Kernel.php 文件的 schedule 方法，里面已经存在一个样板案例了（每小时执行）。注意这里是command方法，表示运行Artisan 命令

//$schedule->command('inspire')->hourly();
                 
// 定时运行命令
$schedule->command('emails:send --force')->daily();
$schedule->command(EmailsCommand::class, ['--force'])->daily();

然后在后面添加一个计划任务，每天运行清理数据表任务，注意这里是call方法，参数为一个闭包函数

$schedule->call(function () {
            DB::table('recent_users')->delete();
        })->daily();

如果你的php.ini允许，还可以使用exec方法运行操作系统命令

$schedule->exec('node /home/forge/script.js')->daily();

运行失败：检查权限 & 时区

注意，需要先测试一下，因为可能会因为权限问题无法执行。

$ /path/to/artisan schedule:run
# 提示无权限
zsh: permission denied: /path/to/artisan

# 添加执行权限
$ chmod +x /path/to/artisan

$ /path/to/artisan

# 再运行
No scheduled commands are ready to run.
Running scheduled command: Closure

另外，如果 php.ini 的时区设置和你的电脑时区设置不一样，可能会发现没有按时执行，这时需要添加一个时区选项。 (如果 php.ini 没有设置 date.timezone 的值，会默认使用 UTC)

$schedule->call(function () {
        DB::table('recent_users')->delete();
    })->dailyAt('11:22')
    ->timezone('Asia/Shanghai');

频率设置参数与条件限制

  • ->cron('* * * * * *'); 在自定义的 Cron 调度运行该任务
  • ->everyMinute(); 每分钟运行该任务
  • ->everyFiveMinutes(); 每五分钟运行该任务
  • ->everyTenMinutes(); 每十分钟运行该任务
  • ->everyThirtyMinutes(); 每三十分钟运行该任务
  • ->hourly(); 每小时运行该任务
  • ->daily(); 每天午夜运行该任务
  • ->dailyAt(‘13:00’); 每天的 13:00 运行该任务
  • ->twiceDaily(1, 13); 每天的 1:00 及 13:00 运行该任务
  • ->weekly(); 每周运行该任务
  • ->monthly(); 每月运行该任务
  • ->yearly(); 每年运行该任务
  • ->timezone(‘America/New_York’); 设置时区

然后可以结合一些参数实现更加精确的调度

  • ->weekdays(); 限制任务在工作日
  • ->sundays(); 限制任务在星期日
  • ->mondays(); 限制任务在星期一
  • ->tuesdays(); 限制任务在星期二
  • ->wednesdays(); 限制任务在星期三
  • ->thursdays(); 限制任务在星期四
  • ->fridays(); 限制任务在星期五
  • ->saturdays(); 限制任务在星期六
  • ->when(Closure); 限制任务基于一个为真的验证

例如下面的

$schedule->call(function () {
    // 在每个礼拜一的 13:00 运行一次...
})->weekly()
->mondays()
->at('13:00');

时间范围限制

$schedule->command('reminders:send')
                    ->hourly()
                    ->between('7:00', '22:00');
                    
$schedule->command('reminders:send')
                    ->hourly()
                    ->unlessBetween('23:00', '4:00');

验证为真

->when(function () {
	//并且 when 参数指定的 闭包 返回 true，且没有其它限制条件存在
    return true;
});

避免前一个任务没有结束时的重复执行

默认情况，即便之前相同的任务主体仍未结束，现有计划任务依旧会被运行。为了避免这个问题，你可以使用 withoutOverlapping 方法：

$schedule->command('emails:send')->withoutOverlapping();

输出任务结果

//sendOutputTo 把结果发送到某个文件
//appendOutputTo 把结果附加到某个文件末尾
//emailOutputTo 发邮件
$schedule->command('foo')
         ->daily()
         ->sendOutputTo($filePath)
         ->appendOutputTo($filePath)
         ->emailOutputTo('foo@example.com');

注意

  1. 对于emailOutputTo，你必须先通过sendOutputTo方法将其输出到一个文件。同时，在邮件发出之前设置 Laravel 的 电子邮件服务 2.emailOutputTo与sendOutputTo 方法只适用于 command 方法，并且不支持call 方法。

任务挂勾（执行前、执行后运行其他方法）

$schedule->command('emails:send')
         ->daily()
         ->before(function () {
             // 任务将要开始...
         })
         ->after(function () {
             // 任务已完成...
         });

任务挂勾有个特殊的“Ping网址” 通过 pingBefore 与 thenPing 方法，调度器能自动的在一个任务完成之前或之后 ping 一个指定的网址。该方法在你计划的任务进行或完成时，可用来有效的通知一个外部服务，例如 Laravel Envoyer：

$schedule->command('emails:send')
         ->daily()
         ->pingBefore($url)
         ->thenPing($url);

使用 pingBefore($url) 或 thenPing($url) 功能需要 Guzzle HTTP 函数库的支持。可在 composer.json 文件中加入以下代码来安装 Guzzle：

“guzzlehttp/guzzle”: “~5.3|~6.0”
