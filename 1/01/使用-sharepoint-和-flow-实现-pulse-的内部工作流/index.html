使用 SharePoint 和 Flow 实现 Pulse 的内部工作流

一些概念

一个工作流的例子

按我的理解，工作流简单来说就是一个数据在不同的节点之间，根据条件或用户操作，不断变动状态。

举一个简单例子“Eric 要请假”来便于理解

角色

  • IT 部门，经理是 Bob, 有一个员工 Eric
  • 行政部门，经理是 Jenny，还有一个助理 Bess

节点

  • Eric 先填写一个请假单， 处于开始节点
  • Eric 提交这个请假单，自动发给他的经理 Bob, 处于等待经理批准节点
  • 如果 Bob 通过，请假单自动发给人力资源 Jenny 审核，变成经理通过，等待人力资源节点
  • 如果 Bob 拒绝，请假单的处于节点经理拒绝，整个工作流结束
  • Jenny 或者 Bess 通过，则处于 人力资源通过节点
  • 如果 Jenny 或者 Bess 拒绝，则处于人力资源拒绝节点，工作流结束

数据

请假单是由一系列字段组成的表单，例如请假人，部门，时间等等。 对应地就是一个数据表，先根据字段设计好表结构

Eric, Tim, Taylor 每次请假都填一个新的请假申请单，那么就是表的 N 条“记录”，是一个 List

这个请假单长什么样子，是在网页上运行，还是在手机上运行？ 需要设计表单

转换与条件

在节点之间变化，是通过条件来触发的, 包括自动或手动。并且可以在转换发生时产生事件（例如发送邮件通知给相关的人）

例如 Bob 点了“允许”按钮， 或者到时间了自动拒绝等等，然后决定节点是否跳转到下一个节点

SharePoint 与 List & Item 管理数据表结构和存储记录

在 SharePoint 里处理保存网页（Page），文件（视频等等），还能创建 “App”，其中有种叫 List. 就能用来保存工作流的数据。

Form/App: 这个申请表单的体现形式，例如有输入框、日期选择器等

SortByColumns根据哪几个值排序

  • 第一个是个 Filter()函数
  • 第二个是 Title(就是对应 list 的 title 字段)
  • 第三个是。。。

Filter()第一个参数是 list 的名称，第二个是 StartWith()查询在上面那个输入框里的内容是否能匹配 title 的值

SortByColumns(
    Filter(
        PulseTravelBusinessTest,
        StartsWith(
            Title,
            TextSearchBox1.Text
        )
    ),
    "Title",
    If(
        SortDescending1,
        Descending,
        Ascending
    )
)

例如这里需要改成列表所有的项目中，某个字段（‘Immediate Supervisor’.Email）的值里有字符串 ”当前用户的email” User().Email

SortByColumns(
    Filter(
        PulseTravelBusinessTest,
        StartsWith(
            'Immediate Supervisor'.Email,
            User().Email
        )
    ),
    "Title",
    If(
        SortDescending1,
        Descending,
        Ascending
    )
)
