Voyager

和 Laravel-Admin 类似，但是更漂亮一些， 并且： 能够图形化得到模型关联、能够图形化创建表。。。

怎么没早发现它呢！laravel-admin 星星还没它多呢。。。

https://docs.laravelvoyager.com/

优势

  1. 解决了 laravel-admin 的路径问题（运行在子目录下链接错误），它能正常识别到
  2. 配置更灵活
  3. 比较美观

问题

  1. 因为引入了 google fonts （来实现图标等）或者别的原因，点击切换的时候很卡
  2. 界面不够紧凑（列表间距很大 等等）

安装

安装包

composer require tcg/voyager

我出现了这个提示，但还是安装成功了，后面问题待测试。

Failed to decode response: zlib_decode(): data error Retrying with degraded mode, check https://getcomposer.org/doc/articles/troubleshooting.md#degraded-mode for more info

（奇怪，后来全部清空， remove 包重新安装又好了？？）

正式安装 voyager（干净/测试数据）

php artisan voyager:install

# 如果加参数，则会安装一些测试数据，包括一个管理员（如果user 没有数据的情况），page, posts settings 等
php artisan voyager:install --with-dummy

测试数据里的管理员信息

email: admin@admin.com password: password

之前安装过一些用户权限相关的包，这里会提示数据表重复了。

提示 roles 表已经存在

里面有数据

找到 voyager 目录里的 migrations /vendor/tcg/voyager/migrations/

通过检查，spatie/laravel-permission包中用的是 auth_roles。

想到之前安装过一个用户权限有关的包EnTrust

https://github.com/Zizaco/entrust

正好对应这几个表

After the migration, four new tables will be present:

  • roles — stores role records
  • permissions — stores permission records
  • role_user — stores many-to-many relations between roles and users
  • permission_role — stores many-to-many relations between roles and permissions

因此如果不再使用的话，就添加个 migration 删除它们好了

php artisan make:migration delete_entrust_package_not_used_table

注意修改迁移文件名，时间要比 voyager 的 migrations 早，例如改为 2015年12月30日 2015_12_30_023929_delete_entrust_package_not_used_table.php

内容

public function up()
{
    Schema::dropIfExists('roles');
    Schema::dropIfExists('permissions');
    Schema::dropIfExists('role_user');
    Schema::dropIfExists('permission_role');
}

然后删除之前可能创建的这些文件。不然后面又会创建出来

在单独运行这个文件

php artisan migrate --force --path=./database/migrations/2015_12_30_023929_delete_entrust_package_not_used_table.php

migrations 都成功运行了，但好像官方的 migration 丢了个字段

自己补上吧。。。生成修改 roles 表字段（添加 description）的migration

php artisan make:migration add_field_description_to_roles_tabl

修改得到的文件 2019_01_21_025553_add_field_description_to_roles_table.php

public function up()
{
    Schema::table('roles', function ($table) {
        if (!Schema::hasColumn('roles', 'description')) {
            $table->string('description')->nullable()->after('name')->default('');
        }
    });
}

/**
 * Reverse the migrations.
 */
public function down()
{
    if (Schema::hasColumn('roles', 'description')) {
        Schema::table('roles', function ($table) {
            $table->dropColumn('description');
        });
    }
}

运行迁移

php artisan migrate

再次安装 voyager (自动运行 seeds)

php artisan voyager:install --with-dummy

哎 这位作者 怎么回事。。。缺两个字段（seed 和 migration 不对应）

再重复下吧，给 permission 添加字段（两个）

//up
Schema::table('permissions', function ($table) {
    if (!Schema::hasColumn('permissions', 'description')) {
        $table->string('description')->nullable()->default('');
    }
    if (!Schema::hasColumn('permissions', 'name')) {
        $table->string('name')->nullable()->default('');
    }
    if (!Schema::hasColumn('permissions', 'display_name')) {
        $table->string('display_name')->nullable()->default('');
    }
});

//down
if (Schema::hasColumn('permissions', 'description')) {
    Schema::table('permissions', function ($table) {
        $table->dropColumn('description');
    });
}

if (Schema::hasColumn('permissions', 'name')) {
    Schema::table('permissions', function ($table) {
        $table->dropColumn('name');
    });
}

if (Schema::hasColumn('permissions', 'display_name')) {
    Schema::table('permissions', function ($table) {
        $table->dropColumn('display_name');
    });
}

运行 n 遍，还发现很多问题。 算了就不带 --with-dummy 参数了

还是不行！！！

最后，删除所有表， （migrate:refresh）, 重新 remove 包 require 包

再次运行 voyager:install 终于成功了

php artisan voyager:admin your@email.com
