var jsonData = psd2json('./target.psd', './outdir');                // return JSON and output /outdir/target.json /outdir/target/*.png

PS 在虚拟机或低配置显卡里报错 vRam 512M 问题

CC 2017 2018 一打开文件，就提示3D 火焰什么的 要求 vRam 512，然后确认，会提示 无法完成请求，因为不是指定文档。。。

（虚拟机里，这个首选项要鼠标一直往下滚，或者按方向下键才出来。。。。 还以为新版不见了。。）

Paralles 中设置显存为 512M (但这样内存捉急了) 在 VirtualBox 这免费开源的里面反而正常！！！ 即使显存是 32M 也不会有问题， 估计是 Paralles 显卡驱动的问题（更新无效。）

无效 将“编辑–首选项–性能–高级选项”里面的“绘图模式”设置为基本。（不开到 512M 这个是灰色的。。。） 编辑-首选项-性能 内存调小

无解， 那就 Parallels 用 512M 显存吧。真浪费，分配的内存才 1G

换成“绿色” CC.。。。 然后在弹出 vRam 512 的时候选“不再显示”，继续打开，是Ok 的。（高版本的“安装版”继续操作提示文件格式错误什么的，打不开）

但是 Cutterman 要求 CC2014及以上（无法加载面板）， CC2015 绿色版打开 cutterman 报错“因为程序错误。。”

CC2015 特别安装版(装完要登录 Adobe ID。。。。不然进不去。。)

CC2014 安装板 （重装系统再装 cutterman, 不然一直提示无法加载扩展）

最后

使用 512M 显存！ CC 2018

上面的可忽略

如果提示“ps无法完成请求，因为它不是所指类型文档”

不要在 PS 里打开文件， 直接右键文件，用 Photoshop 打开 就可以了！！！！ （不用管这个内存的问题）

怀疑是 Parallels 对远程网络文件的判断有问题，Virtual 可以像“本地”一样直接打开。

批量修改图片(大小/颜色)

打开 PS 的“动作”面板（ALT + 9）,

先打开图片。 （确保这个打开动作不会“录制“成动作的一部分）

新建动作，把步骤操作一遍包括打开、储存、关闭，完成后停止动作

然后开始“批处理”（文件-自动-批处理），选择 动作、源、目标的文件夹
