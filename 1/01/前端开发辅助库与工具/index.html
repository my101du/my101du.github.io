使用代码高亮 highlight

默认没有对内部解析出来的代码进行高亮

不能用前面的 react-syntax-highlighter，下面的代码会报错（因为 SyntaxHighlighter 内部只能是字符串，而不是React组件对象，且 Markdown 组件的 source 也需要是一个字符串）

<SyntaxHighlighter language='markdown' style={docco}><ReactMarkdown source={input} /></SyntaxHighlighter>

官方方法在这里：结合 highlight.js 来处理

https://github.com/rexxars/react-markdown/issues/3

首先是在 <Markdown>组件内添加 renderers属性

<Markdown
            className="result"
            source={this.state.markdownSrc}
            skipHtml={this.state.htmlMode === 'skip'}
            escapeHtml={this.state.htmlMode === 'escape'}
            renderers={{code: CodeBlock}}
          />

解释如下

renderers - object An object where the keys represent the node type and the value is a React component. The object is merged with the default renderers. The props passed to the component varies based on the type of node. With one exception: if the key is text, the value should be a function that takes the literal text and returns a new string or React element.

这个属性来定义不同的节点类型用不同的方式来解析：key 节点类型，value 值是一个 react 组件（所有的节点类型可以在文档看到）

对应上面就是 用 CodeBlock 组件来解析 code 节点类型

仔细看 CodeBlock 组件的代码 （ref 的解释参看 https://hackernoon.com/refs-in-react-all-you-need-to-know-fb9c9e2aeb81）

    this.highlightCode();
  }
