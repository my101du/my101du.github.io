简化 Form Html 表单的代码输入 打开 https://laravelcollective.com ，根据 Laravel 版本来使用文档，例如： https://laravelcollective.com/docs/5.1/html

这里以 Laravel 5.5 使用的 laravelcollective/html 5.5 为准。

安装

安装最新版

composer require laravelcollective/html
composer install

添加到 providers 和 alias 数组(config/app.php)

'providers' => [
    // ...
    Collective\Html\HtmlServiceProvider::class,
    // ...
  ],
  
  
  'aliases' => [
    // ...
      'Form' => Collective\Html\FormFacade::class,
      'Html' => Collective\Html\HtmlFacade::class,
    // ...
  ],

创建表单 Form

表单 open /close 标记

不指定 method，默认为 post。 如果是 put，delete 这种需要“模拟”的 html 表单方法，会自动创建_method隐藏字段

{!! Form::open(['url' => 'foo/bar', 'method' => 'put']) !!}
    //
{!! Form::close() !!}

除了使用url方法，也支持指向到 routes 或 controller actions.

echo Form::open(['route' => ['route.name', $user->id]])
echo Form::open(['action' => ['Controller@method', $user->id]])

如果在表单中要上传文件，添加一个 files 选项到参数数组中

echo Form::open(['url' => 'foo/bar', 'files' => true])

补充

如果要添加 id 等属性

{!! Form::open(['url' => route('portal.createOrUpdateQuote'), 'files' => true, 'id'=>'quote-form']) !!}

向表单添加 CSRF Token

保护应用免受 cross-site request forgeries

使用 Form::open 会 自动添加. 也可以手动添加

echo Form::token();

在 route 中应用它

Route::post('profile',
    [
        'before' => 'csrf',
        function()
        {
            //
        }
    ]
);

模型与表单绑定

基于一个 model 的内容来创建一个表单

echo Form::model($user, ['route' => ['user.update', $user->id]])

现在，你只要创建一个表单元素，例如 text input，会自动从模型中找到这个元素的 name 的值并设置。

注意 如果你有 Session flash data 也和表单里某个 field name 相同，会优先于 model 的数据！完整的优先级是

Session Flash Data (Old Input) Data From Current Request (via Request::input method) Explicitly Passed Value Model Attribute Data

注意

和Form::open()一样，使用 Form::model时, 记得使用 Form::close 来关闭

使用 Accessors 让表单中显示的数据和实际 model 中的数据不同

首先知道 http://laravel.com/docs/5.2/eloquent-mutators#accessors-and-mutators 相关知识。

form[CamelAttributeName]Attribute

例如通过在模型中创建一个 formDateOfBirthAttribute 方法，可以为 date_of_birth 属性定义个 accessor. 会在 Form::model() 执行时自动被 HTML Form Builder 调用，来填充表单元素

namespace App;

use Carbon\Carbon;
use Illuminate\Database\Eloquent\Model;

class User extends Model
{
    public function getDateOfBirthAttribute($value)
    {
        return Carbon::parse($value)->format('m/d/Y');
    }

    public function formDateOfBirthAttribute($value)
    {
        return Carbon::parse($value)->format('Y-m-d');
    }
}

表单控件

除了下面的在 php 代码里用 echo函数，在 Blade 模板里应该使用类似下面的语法

{!! Form::label() !!}

Label

echo Form::label('email', 'E-Mail Address', ['class' => 'awesome']);

Text, TextArea, Password, Hidden Field

echo Form::text('username');
// with a default value
echo Form::text('email', 'example@gmail.com');

// hidden 和 textarea 类似

// 密码字段
echo Form::password('password', ['class' => 'awesome']);

// 其他 如 email, number 等
echo Form::email($name, $value = null, $attributes = []);
echo Form::number('name', 'value');
echo Form::date('name', \Carbon\Carbon::now());

// 如果允许上传文件，注意要先在 Form Open 里把 files option set to true
echo Form::file($name, $attributes = []);

CheckBox & Radio

echo Form::checkbox('name', 'value');
echo Form::radio('name', 'value');


// 选中状态
echo Form::checkbox('name', 'value', true);
echo Form::radio('name', 'value', true);

Dropdown list

echo Form::select('size', ['L' => 'Large', 'S' => 'Small']);

// selected default value
echo Form::select('size', ['L' => 'Large', 'S' => 'Small'], 'S');

// create a first option with null value
echo Form::select('size', ['L' => 'Large', 'S' => 'Small'], null, ['placeholder' => 'Pick a size...']);

// grouped dropdown list
echo Form::select('animal',[
    'Cats' => ['leopard' => 'Leopard'],
    'Dogs' => ['spaniel' => 'Spaniel'],
]);

// range
echo Form::selectRange('number', 10, 20);

// month
echo Form::selectMonth('month');

Button & Submit

echo Form::submit('Click Me!');

自定义

自定义“宏”（生成自己的表单组件）

注册一个自定义的宏

Form::macro('myField', function()
{
    return '<input type="awesome">';
});

然后就可以使用了

echo Form::myField();

自定义组件

自定义组件和自定义宏很类似，但是它不是直接在闭包函数里写 html，实际上使用了 Blade 的 Template 特性，把一堆元素放在一起组成一个“组件”，保存为一个 Blade 模板。

比如熟悉 Bootstrap 的开发者肯定经常写这样的一大段，特别烦。

<div class="form-group">
	<label class="control-label">test</label>
	<input type="text" name="username" class="form-control" />
</div>

我们把上面那两个表单元素写成一个组件，放在 resources/views/components/form/text.blade.php 文件里。

// resources/views/components/form/text.blade.php
<div class="form-group">
    {{ Form::label($name, null, ['class' => 'control-label']) }}
    {{ Form::text($name, $value, array_merge(['class' => 'form-control'], $attributes)) }}
</div>

注册组件，registering your Components inside a Service Provider’.(这部分内容请参看 Service Provier 章节)，例如 FormServiceProvider

步骤如下，首先创建一个新的 ServiceProvider

php artisan make:provider FormServiceProvider

然后在 config/app.php里注册它

'providers' => [
    // Other providers
    App\Providers\FormServiceProvider::class,
],

最后在boot方法中创建新的自定义组件

Form::component('bsText', 'components.form.text', ['name', 'value' => null, 'attributes' => []]);

然后就可以直接用 Form Facede 调用这个模板了

{{ Form::bsText('first_name') }}

<!--生成对应结果-->
<div class="form-group">
    <label for="first_name">First Name</label>
    <input type="text" name="first_name" value="" class="form-control">
</div>

生成 URL

简化 <a href=""> 的写法

// url
echo link_to('foo/bar', $title = null, $attributes = [], $secure = null);

// assets
echo link_to_asset('foo/bar.zip', $title = null, $attributes = [], $secure = null);

// named route
echo link_to_route('route.name', $title = null, $parameters = [], $attributes = []);

//to a action
echo link_to_action('HomeController@getIndex', $title = null, $parameters = [], $attributes = []);
