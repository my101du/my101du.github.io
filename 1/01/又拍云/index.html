又拍云

共享IP的渣渣虚拟主机，主机商又不提供SSL证书安装服务，那只能通过CDN在传输过程中全站HTTPS，这种方式更简单。

以下介绍的就是使用又拍云的CDN服务配置全站HTTPS。

[又拍云联盟]的站长扶持计划，只要在博客页脚加上upyun的logo，通过审核后，每个月有免费的15G流量和10G存储空间，对于我们这种个人博客基本够了

除了国内的又拍云还有国外的cloudlfare同样支持免费CDN服务，我以前用过一次，效果不是很理想。

全网加速服务

创建一个服务项目 源站类型：当然填写自主源了,源站指的就是我们主机上的数据。 加速域名：当然是自己博客的域名：前提是要备案

回源配置 CDN的关键就是通过CDN服务商提供的CNAME记录，使用户每次请求都是访问距离最近的缓存服务器，缓存服务器如果已经有数据了，直接返回数据。如果缓存服务商没有请求的数据，就继续向真实主机发起数据请求。而向真实主机请求数据的过程就是回源的过程。

云存储服务

创建一个云存储服务 可以认为是某个域名，专门保存图片。主域名仍然是自己的IP

图片类型 文件类型 视频点播类型

可以使用客户端 sdk api 等来进行管理

一般把它单独绑定一个域名，例如 img.sample.org 然后其他应用统一调用里面的文件。

全站 CDN 服务

像是把云存储服务给封装了一下，关闭了文件管理，能够自动进行全站 CDN 加速。 主域名也变成了七牛 CDN 的域名

服务创建完成

操作 admin 后台提示 403 forbidden 关闭又拍云的 WAF 保护

又拍云联盟 https://www.upyun.com/league

HTTPS

又拍云的 HTTPS 证书启用并绑定到 CNAME 域名后。

如果 WordPress 等动态站点自己的服务器上没有使用 SSL

回源： 设置为 用协议跟随

但是 wp 的内部会报错 /include/ /theme/ css 说加载了不安全（非 https 协议）

建议申请 Let’s Encrypt 类型的证书，可以自动续签 (同时宝塔上可以使用 let’s encrypt 自动续签。 -不知道有问题没有。。。) 那个 Trust Asia 的一年后要手动处理。

mixed content

wordpress -> theme -> header

需要在“缓存规则”里，把下面的部分添加到“不缓存”

页面压缩

据测试，当“页面压缩”功能开启的时候会过滤掉后台页面重要的JavaScript段落，这直接导致了一系列故障从而使得文章无法发布。

终极解决办法：（在常用电脑上直接绑定 host 到真实 ip） 修改你系统内的Hosts文件，将你的域名强制定向到你的源服务器，这样既可以解决因为启用CDN导致的Wordpress后台任何显示异常问题。

当然，其用此方法仅会导致本机访问网站的时候CDN失效，并不影响其他人通过CDN的访问。

虽然改了 discuz 的地址，但是

此时注意到 URL 已经变成了 http: 因此需要在 upyun 里的 https 里“强制https”

discuz 还是有 mix content ， 小锁不显示问题

因为头像，

1 进入后台 站长， 修改 ucneter 设置。 可能保存发现不生效。把 头像调用改成“静态调用”，（这时头像404） 再改回动态

2.进入 /uc_center, 应用管理，修改 论坛的 https, 然后更新数据缓存

最后

在 freessl.cn 上申请 key & pem 类型证书（用 dns 验证 txt 记录）

然后分别在宝塔上部署（自定义）， 又拍云使用 “自有 ssl” 一年后续签

key manager 辅助，添加 域名服务上（如 腾讯云），拿到 secretid 和 accesskey 等 （保存到 key manager） -> 新建密钥（直接使用主账号， 虽然新建子用户安全性高，但是要输入手机啥的）
