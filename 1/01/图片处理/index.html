图片处理

 评论 字数统计: 909字 阅读时长: 2分

功能: 图片处理 slug: laravel-image-handler tags:

  • laravel

Intervention

同样的问题 如果处理图片太大，会出现超过php 内存限制的错误 需要修改 PHP的memory_limit

1、修改php.ini memory_limit = 128 这种方法需要重启服务器，很显然，此方法对虚拟机有限制

2、通过ini_set函数修改配置选项值 @ini_set('memory_limit', "256M");

关于

http://image.intervention.io/

用户头像上传、个人的照片上传等。

除了用 autoload 方式，还可以直接和 Laravel 集成

安装

composer require intervention/image

配置

导出配置文件

php artisan vendor:publish --provider="Intervention\Image\ImageServiceProviderLaravel5"

打开config/image.php可以看到默认使用 GD 库来处理文件，如果希望使用 Imagick 库，需要修改 driver。因为 Imagick 支持处理 BMP ICO PSD 等更多格式

别名和prodivder引入

查看文件 vendor/intervention/image/provoider.json

因此在 config/app.php 的

providers添加

Intervention\Image\ImageServiceProvider::class,

aliases 添加

'Image'        => Intervention\Image\Facades\Image::class,

API

SimpleImage

问题 在循环里下载几张 4M 图片后，出现过内存占用超过 130M 导致 nginx 返回 500 错误的问题

https://github.com/claviska/SimpleImage

安装

composer require claviska/simpleimage

没有像 Intervention 提供 provider 和 alias 参数,不需要添加 provider 等

直接使用

try {
  // Create a new SimpleImage object
  $image = new \claviska\SimpleImage();

  // Magic! ✨
  $image
    ->fromFile('image.jpg')                     // load image.jpg
    ->autoOrient()                              // adjust orientation based on exif data
    ->resize(320, 200)                          // resize to 320x200 pixels
    ->flip('x')                                 // flip horizontally
    ->colorize('DarkBlue')                      // tint dark blue
    ->border('black', 10)                       // add a 10 pixel black border
    ->overlay('watermark.png', 'bottom right')  // add a watermark image
    ->toFile('new-image.png', 'image/png')      // convert to PNG and save a copy to new-image.png
    ->toScreen();                               // output to the screen

  // And much more! 💪
} catch(Exception $err) {
  // Handle errors
  echo $err->getMessage();
}

上面图片库处理一些“分辨率”高的图片时分配内存问题 500 错误

经过分析，有的图片 4M 多，也不会报内存错误，但是有的图片 3M 都会报错。

即使改成 256M 限制也不行

检查发现和分辨率有关！ 可能超过某个值，就会出现“预”分配内存问题 （这个之前在 node 的图片处理库 sharp 也遇到过），看来和 php 的 memory_limit 并没有什么太大的关系

那么这个限制怎么计算？ 这文章可以看到，算法是 width x height x 3， 也就是说这张 7067x5290 的图，需要内存

https://stackoverflow.com/questions/2827908/php-gd-allowed-memory-size-exhausted

但如果无法预测图片大小怎么办？难道给它打到 1GB ？ 如果还是不够呢？

在线计算工具

http://www.dotsamazing.com/en/labs/phpmemorylimit

计算公式

在本来的 memory_limit 基础上，加上处理图片需要的内存大小（因为除了图片处理，php代码也要占用内存，即可能是 默认的 128M + 512M）

https://alvarotrigo.com/blog/allocate-memory-on-the-fly-PHP-image-resizing/

注意，代码里与一个“初始化内存值”，否则如果跑个循环， size 会一直增大

public function setMemoryLimit($filename)
{
    //this might take time so we limit the maximum execution time to 50 seconds
    set_time_limit(50);

    //reset to the default value
    $defaultMemoryLimit = "256";
    ini_set('memory_limit', $defaultMemoryLimit . 'M');

    //getting the image width and height
    $width = 0;
    $height = 0;
    list($width, $height) = getimagesize($filename);
    
    $size = $defaultMemoryLimit + floor(($width * $height * 4 * 1.5 + 1048576) / 1048576);

    //We don't want to allocate an extremely large amount of memory, define a limit
    $maxAllowedMemory = "1024";
    
    if ($size > $maxAllowedMemory) {
        $size = $maxAllowedMemory;
    }

    //updating the default value
    ini_set('memory_limit', $size.'M');
    echo 'allocate new memory size:' . $size.'M<br />';
}

在处理多张图片的时候，输出如下

allocate new memory size:257M allocate new memory size:371M allocate new memory size:365M allocate new memory size:361M allocate new memory size:470M allocate new memory size:470M allocate new memory size:260M
