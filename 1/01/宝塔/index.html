免费的 nginx WAF

https://github.com/alexazhou/VeryNginx

官方的 nginx 防火墙要收费。 但有替代方案:VeryNginx的WAF防火墙

使用浏览器验证。可以避免被采集。也可以用来防御CC攻击，基本跟宝塔的收费插件nginx差不多功能，还带图形界面

宝塔面板已经包含openresty，lua-nginx-module，http_stub_status_module，http_ssl_module。所以只需要安装VeryNginx单软件即可，千万不要用网上其他教程的方法，否则会导致宝塔的Nginx无效。

git clone https://github.com/alexazhou/VeryNginx.git cd VeryNginx #进入文件夹 python install.py install verynginx #只安装VeryNginx本体

执行以上代码后如果没有错误，就说明已经安装完成了

测试VeryNginx是否正常运行：

打开你的网址/verynginx/index_zh.html ，用户名密码都是verynginx，记得进去后第一时间修改。 尝试添加自定义动作，浏览器校验，设置个cookie验证上去。点右下角的保存，刷新几次你的网址，看看是否多了一个叫verynginx_sign_cookie的cookie值。 至此VeryNginx就算正常运行了。其他设置请自行研究。

必须要用到且了解此功能才决定自己是否要开启! 开启后，以任何方式访问面板，将先要求输入BasicAuth用户名和密码 开启后，能有效防止面板被扫描发现，但并不能代替面板本身的帐号密码 请牢记BasicAuth密码，一但忘记将无法访问面板 如忘记密码，可在SSH通过bt命令来关闭BasicAuth验证

手动安装 php 扩展

例如 ldap 默认是没有的，在面板的“扩展安装”里也找不到这个，就只能手动安装了

https://www.bt.cn/bbs/forum.php?mod=viewthread&tid=2802&extra=

Ubuntu 特别简单，apt-get install php7.1-ldap CentOS yum install php-ldap （没有 7.1）

所以根据 https://centos.pkgs.org/7/centos-sclo-rh-x86_64/rh-php70-php-ldap-7.0.10-2.el7.x86_64.rpm.html

改成 71

# Install CentOS SCLo RH repository:
yum install centos-release-scl-rh

#Install rh-php70-php-ldap rpm package:
yum install rh-php71-php-ldap

然后把 /opt/rh/rh-php71/root/usr/lib64/php/modules/ldap.so 复制到宝塔PHP实际的 extension 里面去

其他的要从源码编译安装了

#下载源码  解压 进入
wget https://github.com/swoole/swoole-src/archive/v1.9.8.tar.gz

tar zxf v1.9.8.tar.gz && cd swoole-src-1.9.8

# 使用phpize命令用于准备php扩展库的编译环境 ，自己修改路径
/www/server/php/70/bin/phpize

# 编译安装前的准备 指定php配置目录
./configure --with-php-config=/www/server/php/70/bin/php-config

# 安装
make && make install

修改php配置文件即可完成此扩展到的添加支持
