项目结构

结合微信官方 quick start 的例子与个人需求，将项目结构如下分好：

├── app.js
├── app.json
├── app.wxss 
├── config.js // 配置文件
├── image // 图片目录
├── pages // 页面目录
├── utils // 实用 untils 类
└── wxParse // 第三方库wxParse

“utils”文件夹

“utils” 文件夹里核心的文件是api.js,在这个js文件里提供程序需要调用 WordPress REST API 的公用接口方法

api.js 提供的接口方法包括：

（1）获取文章列表。 （2）获取文章详情。 （3）获取页面列表。 （4）获取页面详情。 （5）获取文章分类。 （6）获取文章评论。 （7）获取文章第一张图片。

“pages”文件夹

包含小程序里所有的功能页面：首页列表（index）、文章详情（detail）、按分类、搜索的文章列表（list），页面详情（page），关于页面（about）….

按照微信小程序的开发规则，每个页面功能都会包括三个文件：js，json ，wxml，wxss。简单来说:js文件控制小程序加载的程序以及发送数据请求，同时把获取的数据提供给wxml来显示,json文件是配置文件，wxml是小程序的前端显示页面（相当于web程序的 html），wxss就是样式文件（相当于web程序的css）有关这些文件更详细的说明见官方的文档。

例如首页（文章列表页面）启动小程序时，会展示最新的 5 篇文章，然后通过下拉流式加载更多文章。

用到的 WordPress REST API 就是 your-site.com/wp-json/wp/v2/posts?per_page={num}&page={num}。

数据请求

需要微信小程序去调用WordPress REST API去获取数据，在微信小程序里提供的发送http请求的api是：wx.request,通过这个接口发送请求，获取数据并赋值（setData）给小程序的页面数据对象（data），小程序前端（wxml文件）基于这个页面数据对象来渲染显示页面。

文章详情解析 html 标签代码成小程序支持的wxml

文章页使用到的 API 地址是 your-site.com/wp-json/wp/v2/posts/{id}。类似地，通过 wx.request 接口访问 URL，然后渲染数据到 WXML 页面上。

因为 wp 保存的是 html 标签代码，将 HTML 直接输出到小程序中，是会报错的，将富文本转为微信小程序可识别的 WXML 的问题

WxParse

(小程序现在出了富文本组件（rech-text），个人评价么，暂时还比不上 wxParse。当前支持的标签有限（如pre标签不支持）且不支持绑定事件，暂时还是先用着wxParse。)

引入库和样式

//在使用的View中引入WxParse模块
var WxParse = require('../../wxParse/wxParse.js');

//在使用的Wxss中引入WxParse.css,可以在app.wxss
@import "/wxParse/wxParse.wxss";

数据绑定

let article = res.data.content.rendered;
WxParse.wxParse('article', 'html', article, self, 5);
 
self.setData({
    wxParseData: article.nodes
});

然后在 wxml 里引用

  1. 首先引入模板 wxParse.wxml
  2. 然后在<template>标签的 is属性里写"wxParse”, data属性把刚刚生成的解析后字符串信息填写进去
<import src="../../wxParse/wxParse.wxml"/>
<view class="entry__cotent ">
    <template is="wxParse" data="{{wxParseData:article.nodes}}"/>
</view>

注意，如果用于数组，前面WxParse.wxParse方法的第一个参数要修改成数组形式

WxParse.wxParse('short_description[' + item.id + ']', 'html', item.short_description, that, 5);

wxml 使用的时候也改成

<template is="wxParse" data="{{wxParseData:short_description[item.id].nodes}}" />

}

根据小程序的登录过程：

  1. 在小程序里 wx.login() 发送登录请求到微信服务器，如果允许登录，微信服务器生成一个 code 返回给小程序
  2. 然后在小程序里，把这个 code 发送到 开发者服务器
  3. 在开发者服务器中，请求[临时登录凭证校验接口](https://api.weixin.qq.com/sns/jscode2session?appid=APPID&secret=SECRET&js_code=JSCODE&grant_type=authorization_code，得到 openid & session_key 或者 openid & session_key & unionid
  4. 开发者可以用上面这几个数据来实现后端的用户身份验证（例如以openID为userid自动在站内注册一个用户, 例如 wp 的wp_insert_user()函数）

然后，The auth endpoint will send the API Keys in JSON format to the callback_url
