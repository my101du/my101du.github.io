微信支付插件

slug: wechat-pay tags:

  • wordpress

init.php 入口

定义常量

XH_WC_WEIXINPAY ‘XH_WC_WEIXINPAY’ XH_WC_WeChat_ID ‘xhwechatwcpaymentgateway’ XH_WC_WeChat_DIR XH_WC_WeChat_URL

加载语言包

load_plugin_textdomain( ‘wechatpay’, false, dirname( plugin_basename( FILE ) ) . ‘/lang/’ );

在“插件”列表的当前插件下增加一个链接

直接跳转到 WooCommere 里的“设置”，“结算”，“微信支付” add_filter( ‘plugin_action_links_’ . plugin_basename( __FILE__ ), ‘xh_wechat_wc_payment_gateway_plugin_edit_link’ );

##在激活插件时，更新数据表 _postmeta

register_activation_hook _payment_method改成值 “xh-wechat”

add_action(init, ‘xh_wechat_wc_payment_gateway_init’)

在这个函数xh_wechat_wc_payment_gateway_init里

  1. 引入类文件 class-xh-wechat-wc-payment-gateway.php
  2. 创建 $api = new XHWechatWCPaymentGateway()
  3. 运行 check_wechatpay_response 检查有没有微信服务器发送过来的数据
  4. add_filter woocommerce_payment_gateways 通过woocommerce_wechatpay_add_gateway添加一个支付网关
  5. add_action 到自定义 (这两个“tag” 没找到)
    • wp_ajax_XH_WECHAT_PAYMENT_GET_ORDER ——— get_order_status
    • wp_ajax_nopriv_XH_WECHAT_PAYMENT_GET_ORDER —— get_order_status
  6. add_action 在运行 WooCommerce 一些动作的时候调用
    • 如woocommerce_receipt_ —– receipt_page
    • woocommerce_update_options_payment_gateways_ ———- process_admin_options（这个函数没找到）
  7. 插入 js 文件 wp_enqueue_scripts (qrcode.js 和 checkout.js)

类 class-xh-wechat-wc-payment-gateway.php （扩展自 WC_Payment_Gateway 类）

__construct

定义这个类的一些描述信息

$this->id，icon，has_fields,method_title, method_description

调用 init_form_fields 来设置”字段“ （商户id，appid, 汇率等等）

略（包括一些 text 组件）

init_settings 后读取配置参数

$this->title = $this->get_option ( ‘title’ ); // description

引入 lib/ 目录 下的微信文件(见下面每个文件详细说明)

初始化 $this->config

new WechatPaymentConfig ($this->get_option(‘wechatpay_appID’), $this->get_option(‘wechatpay_mchId’), $this->get_option(‘wechatpay_key’));

process_payment($order_id) 处理订单付款信息

获得 order 订单数据，返回跳转地址

$order = new WC_Order ( $order_id );
return array (
    'result' => 'success',
    'redirect' => $order->get_checkout_payment_url ( true )
);

woocommerce_wechatpay_add_gateway

把类自己 $this 添加到所有 woocommmerce 支付网关数组里面去

get_order_title

订单标题截取，对return apply_filters('xh_wechat_wc_get_order_title', mb_strimwidth ( $title, 0,32, '...','utf-8'));有疑问。

xh_wechat_wc_get_order_title 这个 tag 是用在？

get_order_status

获取订单状态（paid 还是 unpaid）， and the “return url” of this order

wp_enqueue_scripts

$orderId = get_query_var ( 'order-pay' );

如果某个订单的 payment_method 等于 $this->id （常量XH_WC_WeChat_ID值）, 并且当前在支付页面。引入两个 js 文件

check_wechatpay_response()

检查微信服务器推送过来的 xml 数据。

如果满足条件 验证签名

WechatPaymentResults::Init ( $xml ) 解析 xml 数据

查看 WxPay.Data.php 接口调用结果类 WechatPaymentResults

从 xml 里得到 $transaction_id 和 $order_id

$input = new WechatPaymentOrderQuery ();

$input->SetTransaction_id ( $transaction_id );生成 订单查询输入对象

WechatPaymentApi::orderQuery ( $input, $this->config );

调用 WechatPaymentApi 类（WxPay.Api.php） orderQuery 方法， 从 api.mch.weixin.qq.com/pay/orderquery 查询订单

得到结果再使用 WechatPaymentResults::Init($response,$WxCfg); 解析出来 $result

如果 result_code 或 return_code 等于 ‘FAIL’, 抛出异常（错误码与信息） 如果 trade_state 不等于 ‘SUCCESS’，抛出异常(order not paid)

假如这个微信支付成功了（前面两步都没有问题，且微信服务器上已经支付成功）

在 Woocommerce 里设置支付完成 $order->payment_complete ($transaction_id);

生成一个支付提醒“成功”

$reply = new WechatPaymentNotifyReply ();
$reply->SetReturn_code ( "SUCCESS" );
$reply->SetReturn_msg ( "OK" );
    
WxpayApi::replyNotify ( $reply->ToXml () );

process_refund( $order_id, $amount = null, $reason = ‘') 处理退款

$trade_no =$order->get_transaction_id();

这里用到了汇率（会变化的数据）来确定退款

$input = new WechatPaymentRefund (); 初始化一个退款输入对象（WxPay.Data.php）然后设置一些属性值

WechatPaymentApi::refund 然后发送请求到微信服务器请求退款

function receipt_page($order_id) 收据？

$input = new WechatPaymentUnifiedOrder (); 生成一个统一下单输入对象，接着设置一些值（Out_trade_noorder_id 等 汇率）

注意 SetTrade_type 是 “NATIVE” 和 SetProduct_id

$input->SetTrade_type ( "NATIVE" );
$input->SetProduct_id ($order_id );

调用 WechatPaymentApi::unifiedOrder (统一下单) 这个里面可以看到 JSAPI 还是 NATIVE， 是不是就是这里删除了 SDK 支付的代码？

重要： 在这里显示了二维码图片（以及 xh-wechat-payment-pay-url 微信支付链接）！

如果是调用 SDK 这里应该显示一个”支付“按钮

支付页面（二维码）分析

URL: /checkout/order-pay/3138/?key=wc_order_5b486b598d5f1

二维码实际链接(页面刷新一次，会更新一次)： weixin://wxpay/bizpayurl?pr=jJ5JI39 二维码图片： 根据上面链接，使用 js/qrcode.js 生成

js 持续发送请求到后端，检查是否已经扫码支付：

/wp-admin/admin-ajax.php orderId=3138&action=XH_WECHAT_PAYMENT_GET_ORDER

返回结果：

如果是 unpaid 未支付, 隔 2 秒持续再次请求一次 {status: “unpaid”,…} status:“unpaid” url:“https://new.renqizx.com/checkout/order-received/3138/?key=wc_order_5b486b598d5f1"

如果是 paid 已支付, 跳转到到返回数据中的 data.url
