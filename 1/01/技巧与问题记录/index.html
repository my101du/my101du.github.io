数据查询

如果查询一个数据表，需要让某个一字段作为“key”

//Use the Eloquent
$roles = Role::all()->keyBy('name');

//If you are not using Eloquent, you can create a collection on your own:
$roles = collect(DB::table('roles')->get())->keyBy('name');

//If you're using Laravel 5.3+, the query builder now actually returns a collection, so there's no need to manually wrap it in a collection again:
$roles = DB::table('roles')->get()->keyBy('name');

Blade 模板

如何在 php 文件中也能使用针对 html 的插件呢？

例如 bootstrap 插件，输入 bs3-nav 快速生成代码的。 只能支持 .html 后缀，然而 Laravel 的模板文件是以 .blade.php结尾的。

这就要为 php 文件添加一个“副”类型了

默认点击某个语言，是“切换”解析，会影响当前文件

点击这个添加

然后再编辑一下配置

"files.associations": {
        "*.blade.php": "html"
    },

现在 ok 了

问题

解决： Please provide a valid cache path.

执行 composer install 后， 出现 Please provide a valid cache path. 问题。

于是手动创建了缓存目录， 执行 php artisan optimize 还是报这个错误。

storage 下的 app, framework, logs 都存在，且都是 777 bootstrap/cache 存在。

但 Please provide a valid cache path. 还在。

继续在 storage/framework 下面创建 sessions， views, cache 文件夹

然后 chmod -R 777 storage/

推荐在 sessions、 views、cache 目录下放入.gitignore文件保持住文件夹

最近新建了一个 laravel 5.5项目，发现本地运行好好的，部署到服务器就发现出问题了。

在访问后台 admin 页面时提醒出错：

Please provide a valid cache path

经过一番排查，这个可以算作是 gitignore 的问额，也可以不算233333

之前在本地创建项目时，使用的是 github 默认的 laravel 的 gitignore。所以很多文件没有创建，系统缓存没办法创建，于是就出现了这样的问题。

最后的解决办法是手动创建了下面这些目录：

  • storage/framework
  • storage/logs
  • storage/framework/cache
  • storage/framework/sessions
  • storage/framework/views
  • bootstrap/cache

最后就 ok 了。

env 配置问题

突然env()得不到数据了

因为某些原因使用了 config:cache ，导致返回空字符串。运行下面的命令，清空 config 缓存，重新生成就好。

php artisan config:clear

数据库问题

提示 SELECT list is not in GROUP BY clause and contains nonaggregated column

在将一个表的数据按需求导入另一个表时，使用了group语句如下

DB::insert("INSERT INTO sap_product_main " .
            "(`bu`, `class`, `part_number`, `updatetime`, `updatedate`, `desc`) " .
            "SELECT `bu`, `class`, `part_number`, `updatetime`, `updatedate`, `product_desc` FROM sap_sync_full GROUP BY `part_number`");

如果用 MySQL5.7 版本，会报错如下

Syntax error or access violation: 1055 Expression #8 of SELECT list is not in GROUP BY clause and contains nonaggregated column

解释如下：

https://stackoverflow.com/questions/36228836/syntax-error-or-access-violation-1055-expression-8-of-select-list-is-not-in-gr

模型在插入数据时，自动给某个字段使用当前时间为默认值

假设在数据表 tickets 里有个字段 due_data(datetime 类型)，但没有设置“默认值”.希望在代码里当用户插入新记录时，如果没填写 due_data 的值，自动使用当前的时间。

方法1： 在migration文件里设置一个默认值，使用 SQL 语法

...->default(\DB::raw('DATE_ADD(NOW(), INTERVAL 24 HOUR)'));

方法2，在 controller 插入的时候

if(!$request->has(due_date)) {
    $ticket->due_date = Carbon::now()->addDays(1);
}

方法3， 在 Model 里设置

如果默认值是一个常量，可以用 $attributes

protected $attributes = [
    'status' => self::STATUS_UNCONFIRMED,
    'role_id' => self::ROLE_PUBLISHER,
];

但是我希望用 Carbon 来动态生成日期，编辑器会报错。 常数表达实例有运算符？

protected $defaultDate = Carbon::now();
