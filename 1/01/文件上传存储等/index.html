文件(上传&存储等)

permalink: file-store-of-laravel tags:

  • laravel

Laravel 使用 Flysystem 扩展包提供强大的文件抽象层。除了使用本地的磁盘外，还可以使用各种云存储（driver），并能够轻松地切换这些存储方式。

学会使用Storage这个 facade

设置

修改配置文件config/filesystems.php，能够设置多个“磁盘”（一个磁盘对应一个驱动和位置）

本地 最基本的就是本地磁盘local驱动，配置文件中，默认的root是对应着storage/app，下面的操作会把 file.txt 文件保存到 /storeage/app/file.txt

Storage::disk('local')->put('file.txt', 'Contents');

其他

在使用 S3 或 Rackspace 驱动之前，你需要通过 Composer 安装适当扩展包： 这些在国内用不了，国内的可能就是 七牛 又拍 阿里云 等

  • Amazon S3: league/flysystem-aws-s3-v3 ~1.0
  • Rackspace: league/flysystem-rackspace ~1.0

一种驱动可以有多个磁盘

'disks' => [
    'local' => [
        'driver' => 'local',
        'root'   => storage_path(),
    ],
    'uploads' => [
        'driver' => 'local',
        'root'   => public_path() . '/uploads',
    ],
]

使用

获得磁盘实例

通过 Storage facade 的 disk 方法获得磁盘实例，然后可以使用链式调用对磁盘执行各种方法。

如果不使用disk方法，会使用默认的磁盘实例。在使用中，可以随时切换磁盘实例

Storage::put(
            'avatars/'.$user->id,
            file_get_contents($request->file('avatar')->getRealPath())
        );
        
$disk = Storage::disk('s3');

$contents = Storage::disk('local')->get('file.jpg')

获取文件

判断文件是否存在

$exists = Storage::disk('s3')->has('file.jpg');

获取文件大小、最后修改时间(UNIX 时间戳)

$size = Storage::size('file1.jpg');
$time = Storage::lastModified('file1.jpg');

获取文件内容

$contents = Storage::get('file.jpg');

保存文件

调用 put方法来保存文件。

小文件可以直接将“文件内容”传入

Storage::put('file.jpg', $contents);

如果文件较大，建议传入 PHP 的 resource，使用文件系统底层的 stream 来处理

Storage::put('file.jpg', $resource);

复制到新位置

Storage::copy('old/file1.jpg', 'new/file1.jpg');

在文件头部或尾部插入新内容

Storage::prepend('file.log', 'Prepended Text');
Storage::append('file.log', 'Appended Text');

删除一个或多个文件

Storage::delete('file.jpg');
Storage::delete(['file1.jpg', 'file2.jpg']);

目录

读取目录下的所有文件（一层），以及包含所有子目录下的文件（递归）

$files = Storage::files($directory);
$files = Storage::allFiles($directory);

同样适用于**“获取子目录”**

$directories = Storage::directories($directory);
$directories = Storage::allDirectories($directory);

创建目录(以及子目录)

Storage::makeDirectory($directory);

删除目录（及文件）

Storage::deleteDirectory($directory);

自定义文件系统

Flysystem 提供许多默认驱动，为了让你创建新的驱动（如 Dropbox），它还提供了其他保存系统的接口 Adapter。

为了建构一个自定义的磁盘系统，先创建一个 service provider, 例如 DropboxServiceProvider，然后在 boot 方法使用 Storage facade 的 extend 方法自定义你的驱动。第一个参数是驱动名称，第二个参数接受 $app 及 $config 变量的闭包。该闭包必须返回 League\Flysystem\Filesystem 实例。

$config 变量包含了定义在 config/filesystems.php 对指定磁盘的设置。

然后便能在 config/filesystem.php 配置文件中使用 dropbox 驱动。

use Dropbox\Client as DropboxClient;
use League\Flysystem\Dropbox\DropboxAdapter;
//...

class DropboxServiceProvider extends ServiceProvider
{
	public function boot()
    {
        Storage::extend('dropbox', function($app, $config) {
            $client = new DropboxClient(
                $config['accessToken'], $config['clientIdentifier']
            );

            return new Filesystem(new DropboxAdapter($client));
        });
    }

SFTP 远程传输

在 5.6.7 版本后自带了 sftp 功能，在此版本之前用这个

https://github.com/neoxia/laravel-sftp

composer require neoxia/laravel-sftp

然后向config/app.php的providers列表添加

Neoxia\Filesystem\SftpServiceProvider::class,

安装完毕后，把 key 文件（例如 sftp_key.ppk）放到一个目录下（最好不要在 public 目录，避免被用户下载），然后修改/config/filesystems.php配置文件, 添加内容

'sftp' => [
    'driver'     => 'sftp',
    'host'       => env('SFTP_HOST', 'demo.sftp.org'),
    'port'       => env('SFTP_PORT', '22'),
    'username'   => env('SFTP_USERNAME', 'the-sftp-user'),
    'password'   => env('SFTP_PASSWORD', ''),
    'privateKey' => env('SFTP_PRIVATE_KEY_PATH', storage_path('sftp_key.ppk')),
    'root'       => env('SFTP_ROOT', './'),
    'timeout'    => env('SFTP_TIMEOUT', '20'),
],

把需要通过 sftp 上传的文件，调用 Storage::disk('sftp') 的 put方法，发送到远程 sftp 目录

Storage::disk('sftp')->put('mouser_feed.json', file_get_contents(public_path() . '/files/mouser_feed.json'));

备注： ppk 文件生成
