我们在开发前端页面的时候，如果每一个按钮、样式、处理浏览器兼容性的代码都要自己从零开始去写，那就太浪费时间了。所以我们需要一个框架，帮我们实现一个页面的基础部分和解决一些繁琐的细节，只要在它的基础上进行个性化定制就可以了。

Bootstrap 就是这样一个简洁、直观、强悍的前端开发框架，只要学习并遵守它的标准，即使是没有学过网页设计的开发者，也能做出很专业、美观的页面，极大地提高了工作效率。像下面这个漂亮的网站就是基于Bootstrap来开发的。

Bootstrap的中文文档地址在这里 Bootstrap 中文文档，目前主流版本为 3.3.x。不过因为文档的内容结构比较零散，只是从头到尾把一个个组件拆开来讲一遍，缺乏趣味性和实战性（这也是现在很多教程的通病），因此在本教程中，我们会自己动手开发几个完整案例，在实践的过程中来学习和理解 Bootstrap这个前端开发框架的知识。

为了锻炼你阅读文档和主动学习的能力，我们只会把每个知识点最核心的知识告诉你，而它的扩展知识将以链接的形式附在后面。

第一部分：做一个企业网站

企业网站是十分常见的一种页面形式。一般包括一个展示企业形象的首页、几个介绍企业资料的文章页、一个“关于”页面。如下就是最终的首页效果。（当然了，这个界面还是比较粗糙的，不过我们这个教程重点是Bootstrap的学习，如果你要深入学习 CSS 样式方面的知识，请参看我们的相关教程）

引入 Bootstrap 框架相关文件

前面说到 Bootstrap是一个前端开发框架。其实说白了就是一个样式表文件（bootstrap.min.css）和一个javascript 文件（bootstrap.min.js），在页面里把它们引入进来后，就可以直接使用里面的 CSS 规则和各种组件了。

远程 CDN 引入方式

编辑 Home_index.html，把几个Bootstrap框架包含的文件添加进来，这里推荐使用 CDN 源 上的远程文件，可以节约本地的带宽。 另外，由于 Bootstrap还依赖jQuery库，所以也要一并把这个 jquery.min.js 文件引入。

<link rel="stylesheet" href="//cdn.bootcss.com/bootstrap/3.3.5/css/bootstrap.min.css">
<link href="../assets/style.css" rel="stylesheet">
<script src="//cdn.bootcss.com/jquery/1.11.3/jquery.min.js"></script>
<script src="//cdn.bootcss.com/bootstrap/3.3.5/js/bootstrap.min.js"></script>

注意这几个文件的前后顺序，如果不对，会导致页面无法正常运行。

  1. 先引入 bootstrap.min.css （Bootstrap的样式表文件）
  2. 然后引入自己写的 css 文件（style.css）
  3. 然后引入 jQuery（javascript 库）
  4. 最后引入 bootstrap.min.js 程序文件

本地文件引入方式

如果在没有联网的环境，或者用上面的方式引入文件后浏览器报错，可以把Bootstrap的所有文件下载到本地后再引用到页面中。打开下面这个地址，把压缩包下载后解压，全部放到 assets 目录的 bootstrap里面。

Bootstrap 做了些什么？

打开assets/bootstrap/目录下的样式表文件bootstrap.min.css，可以看到里面定义了大量的 CSS 规则，例如下面这段就定了一个类名为btn-primary的规则。

.btn-primary{color:#fff;background-color:#337ab7;border-color:#2e6da4}

现在测试一下，在 Home_index.html文件里面写一个按钮，添加 class 为 btn btn-primary。

<body>
	<button type="button" class="btn btn-primary">按钮</button>
</body>

刷新页面，你会看到一个蓝色的按钮。不需要自己写一行 CSS 代码，只要在页面里面给某个元素指定一个class，就可以直接显示出预定的样式—— 这就是使用Bootstrap前端框架的魔力。

轮播图组件分三部分

  • <ol class="carousel-indicators">是“指示器”，就是下方的那三个白色小点，标记当前播放到哪张图片了。
  • <div class="carousel-inner" role="listbox">里面是主体内容区域，包括几张图片和对应说明，分别用 <div class="item">包裹起来
  • 最后的两个<a class="left carousel-control"元素是手动操作图片左右切换的按钮。

使用自定义样式来完善 Bootstrap 默认样式

刷新页面，几张图片开始自动播放。但是我们会发现一个问题——如果三张图片大小不一致，这个轮播区域会忽大忽小地变化。

这种情况就是 Bootstrap 力不能及的范围了，因为它不知道我们将会使用什么样的素材，因此要自己写一些自定义样式的 CSS 代码来处理这些特殊的情况。

打开自定义样式表文件 assets/style.css，在里面添加如下的代码，把图片高度固定设置为 500px，并且设置最小宽度为 100% ，撑满页面。

.carousel-inner > .item > img {
  min-width: 100%;
  height: 500px;
}

好了，现在轮播图的图片滚动很正常了。

栅格布局

想想看，如果是你自己写 CSS ，准备怎样实现这个均等排列的效果？例如下面通过设置固定宽度/百分比来处理：

.item { float:left, width: 300px; /*或者 width: 33%*/ }

如果一行要显示4个、6个、或者更多呢？这样计算起来就太不灵活了（100/6 等于 16.6666666……）。

其实我们并不关心每一份的宽度是多少像素/百分比，我们只关心能不能自动地平均划分成多少份，Bootstrap的栅格系统布局就是为了实现自动计算每一份的宽度而诞生的。

首先看看这货来帮助理解什么是“栅格”

这是一个伸缩门，它的总长度可以拉长，可以缩短，但是总的间隔数量是不变的，并且所有间隔的宽度都一样。

这个伸缩的过程，像不像我们把浏览器拉宽、变窄的操作？Bootstrap的栅格系统规定了每个页面的宽度被平均划分为 12 等份，不管整个页面的宽度是 1000像素，还是500像素，都会自动计算每一份（1/12）的宽度是多少。

通过给栅格布局内部的元素指定 class 为 col-md-份数，来告诉它的宽度占据这12份里面的几份。

例如下面的代码中，有3 个 div 的 class为 col-md-4（先不管中间那个 -md-是什么，关注这个数字就好），算一算4 + 4 + 4 是不是正好等于 12？

<div class="container">
    <div class="row">
        <div class="col-md-4">bb</div>
        <div class="col-md-4">cc</div>
        <div class="col-md-4">ee</div>
    </div>
</div>

刷新页面，然后用 Chrome 浏览器的开发工具看看效果，发现整个页面的宽度正好被平均分成了 3 份(12/4 = 3)。栅格系统自动帮我们计算了这 3 份的宽度。

举一反三，如果要把页面平均分为 6 份，那每一份的class 就应该指定 col-md-2(12/6 = 2)；如果是 4 份，class 就是 col-md-3（12/4 = 3）

完整的栅格布局的说明在这里，一开始看不懂没有关系，自己多动手试试就明白了。一定要动手实践！

覆盖 Bootstrap 的默认样式

其实除了可以添加自定义样式外，还可以覆盖掉Bootstrap的默认样式，让它更符合我们的要求。

比如，把顶部的导航条改成黑色底，白色文字。

.navbar {

用户导入与进度条组件

系统刚开始运行的时候，我们的数据库里还没有数据，因此需要手动导入。由于需要导入的用户的数据量可能会比较大，为了避免漫长的等待让操作者误以为系统失去响应，因此在导入过程中，添加一个进度条组件来提示目前进行中的任务实时进度。

进度条上面显示了一个数字，表示任务完成的百分比，这个需要配合后台程序进行实时更新。

在页面里添加一个按钮以及一个进度条组件。进度条默认是隐藏的。点击按钮后，显示进度条，同时触发导入数据的动作，并根据任务进度更新进度条上的数字。（这里为了简化，用一段 javascript 代码来模拟后台写入数据的进度。）

<div class="col-md-9">
	<button class="btn btn-primary" id="btn-import">导入</button>
      <div id="progress-demo" class="hide">
          <p id="progress-status"></p>
          <div class="progress">
              <div class="progress-bar" role="progressbar" aria-valuenow="0" aria-valuemin="0" aria-valuemax="100" style="width: 0%;">
                  0%
              </div>
          </div>
      </div>
</div>


<script type="text/javascript">
   var i = 0;
   
   $("#btn-import").click(function(event) {
       $("#progress-demo").removeClass('hide');
       window.setInterval(function(){
           if(i > 100) {
               window.clearInterval();
               return false;
           }

           $(".progress-bar").attr("aria-valuenow", i);
           $(".progress-bar").css({'width': i + "%"});
           $(".progress-bar").html(i + "%");
           $("#progress-status").html('已完成' + i + "%");
           i += 10;
       }, 400);
   });
</script>

使用弹出框组件显示附加信息

页面中不重要的附加信息可以先不展示出来，在需要的时候点击才显示。这样可以节省很多屏幕空间，有利于信息的高效组织。例如下图中，素材对应每个公众号都会有一个 media_id，但这个数据平时并不会用到。所以像这样的情况就可以使用弹出框组件。

使用弹出框需要经过以下几个步骤（注意：官方的文档翻译不全，根本没有说清楚用法）：

  1. 找到被点击触发弹出框的元素，例如上面代码里的<a class="toggle-popover" href="#" role="button">
  2. 给这个元素添加属性data-toggle="popover"
  3. 给这个元素添加title和data-content属性，设置点击它后显示的弹出框的标题和内容
  4. 如果弹出框的内容要显示 HTML 代码，则添加属性data-html="true"(默认只支持显示纯文本)
  5. 最后一步最重要，在 javascript 代码里，给这些元素绑定弹出框事件触发器 $("a.toggle-popover").popover();
  6. 另外，还可以给被点击元素添加onclick="需要触发的点击事件"属性，在点击的时候除了显示弹出框，还会同时触发这个事件

素材分类显示——导航与标签页

如果把所有的图片、视频、音频全部放在一起显示，不仅看起来很混乱，而且也不便于分类管理。我们可以使用导航组件或者标签页组件将不同类型的数据放到不同的子页面里面，并且在不同的子页面之间切换。

注意导航组件和我们前面学习的导航条组件，虽然只有一字之差，却是完全不同的两个组件。

导航一般用于将数据放到几个子页面分类显示，但是各个子页面里不用类型的数据之间还是有关联，或者有共同的特征的（就像图片、视频、音频都是多媒体素材一样）。 而导航条仅仅用于整个网站的连接与全局功能，导航条内部的各个元素之间可能并没有什么关联。

代码如下：

<ul class="nav nav-tabs" role="tablist">
  <li role="presentation" class="dropdown active">
    <a class="dropdown-toggle" data-toggle="dropdown" href="#">
      多媒体素材 <span class="caret"></span>
    </a>
    <ul class="dropdown-menu" role="menu">
      <li><a href="">图片</a></li>
      <li><a href="">视频</a></li>
      <li><a href="">音频</a></li>
    </ul>
  </li>
  <li role="presentation"><a href="#">图文</a></li>
</ul>

上面的代码创建了一个导航组件，可以在几个子页面之间切换，一个是多媒体素材，一个是图文素材。其中“多媒体素材”标签页又带有下拉菜单，有图片、视频、音频这几种情况可以选择。

导航组件还可以设置样式，例如竖向排列等。关于导航组件的资料在这里：

Bootstrap-导航与标签页

另外，和导航极为相似的组件叫标签页，它们拥有类似的外观，区别在于

  1. 导航组件默认只显示当前激活的一部分数据，在点击标签的时候，浏览器跳转到另外一个页面，然后加载另外一部分数据，这种方式确保每次点击的时候，都会刷新数据；
  2. 而标签页组件一开始就把所有数据全部加载完毕，然后只显示当前激活的一部分数据，把其他的隐藏，点击标签切换后，显示被激活的另外一部分数据，同时把其他的数据隐藏，这种方式响应速度很快，但页面不刷新，所以数据更新不是很及时。

请根据自己的需要来选择

要实现一个模态框需要经过几个步骤：

  1. 创建一个模态框，这是一个class=modal的div，给它添加一个id，例如叫sync-materials-modal，然后添加role等属性，模态框在没有激活之前默认是隐藏的。
  2. 在需要点击激活模态框显示的元素上（button、a 等都可以）添加两个属性，一个是data-toggle="modal"；另外一个是data-target="模态框的id"，它的值是前面第一步创建的模态框的id，例如#sync-materials-modal
  3. 在模态框内部的<div class="modal-header">里添加标题
  4. 在模态框内部的<div class="modal-body">里添加要显示的主体内容
  5. 在模态框内部的<div class="modal-footer">里添加模态框底部的按钮，例如取消、确定等

搜索关键词——输入框组

数据量达到一定程度的时候，通过肉眼去查找某条数据就很困难了。所以我们来添加一个关键词搜索功能。一般情况下都是先在一个输入框里填写关键词，然后点击“搜索”按钮。

要通过Bootstrap来实现这个搜索功能很容易。前面我们学过表单里的输入框组件<input type="text">，也学过按钮组件，只要把它们拼合起来就可以了。

<div class="input-group">
   <span class="input-group-addon">输入关键词</span>
   <input type="text" class="form-control">
   <span class="input-group-btn">
       <button class="btn btn-primary" type="button">搜索</button>
   </span>
</div>

输入框组顾名思义，就是把输入框和其他组件拼接起来。

  1. 一个输入框组是由<div class="input-group">包裹起来的
  2. 内部的<input type="text" class="form-control">是必不可少的
  3. <span class="input-group-addon">一般用于显示“附加信息”
  4. <span class="input-group-btn">内部是一个按钮<button class="btn ">，它会和输入框连成一个整体
