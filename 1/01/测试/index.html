测试

slug: laravel-test tags:

  • laravel

背景

单元测试

单元测试是针对你代码中相对独立而且非常少的一部分代码来进行测试 （想想看，以前开发的时候，改一点代码，刷新一下界面；遇到表单还要重新填写一次内容。。。）

大多数单元测试可能都是针对某一个方法来进行的

功能测试

功能测试是针对你代码中大部分的代码来进行测试，包括几个对象的相互作用，甚至是一个完整的 HTTP 请求 JSON 实例

PHPUnit 学习

了解如何使用

Laraval 测试环境

Laravel 默认就支持用 PHPUnit 来做测试，并为你的应用程序配置好了 phpunit.xml 文件。还有一些 helper functions 也可以直观地测试应用程序

默认在 tests 目录下包含了两个子目录： Feature 和 Unit （功能测试 & 单元测试），里面分别有ExampleTest.php示例文件。

运行下面的命令就可以开始测试

phpunit

在运行测试时，Laravel 会根据 phpunit.xml 文件中设定好的环境变量，自动将环境变量设置为 testing，并将 Session 及缓存以 array 的形式存储，也就是说在测试时不会持久化任何 Session 或缓存数据（持久化保存这些数据必须是 file/redis/mysql 等方式）

可以修改 phpunit.xml 文件里的配置参数，但在下次运行测试之前，请确保使用 config:clear Artisan 命令来清除配置信息的缓存

创建一个测试用例

使用make:test 的 artisan 命令

// 在 Feature 目录下创建一个测试类(默认是功能测试)...
php artisan make:test UserTest

// 在 Unit 目录下创建一个测试类（指定 --unit 参数是单元测试）...
php artisan make:test UserTest --unit

编辑tests/Unit/UserTest.php文件

namespace Tests\Unit;

use Tests\TestCase;
use Illuminate\Foundation\Testing\RefreshDatabase;

class UserTest extends TestCase
{
    public function testExample()
    {
        $this->assertTrue(true);
    }
}

HTTP 测试

打开 Feature/ExampleTest.php文件，里面有一个基础的测试用例testBasicTest。get 方法会创建一个 GET 请求来请求你的应用，而 assertStatus 方法断言返回的响应是指定的 HTTP 状态码。

除了这个，Laravel 还有检查响应标头、内容、JSON 结构等各种断言

public function testBasicTest()
    {
        $response = $this->get('/');

        $response->assertStatus(200);
    }

Session / 认证

通用的 Session 数据

在测试一些功能时，需要提前加载一些 Session 数据。（还记得前面说过，所有的测试用例，Session 及缓存以 array 的形式存储，不会持久化存储）

public function testApplication()
    {
        $response = $this->withSession(['foo' => 'bar'])
                         ->get('/');
    }

模拟用户认证

一般使用 Session 都是用于用户认证的情况，有一个辅助函数actingAs简单地让某个指定用户认证为当前用户。（这里用一个技巧，用模型工厂来生成用户）

public function testApplication()
    {
        $user = factory(User::class)->create();

        $response = $this->actingAs($user)
                         ->withSession(['foo' => 'bar'])
                         ->get('/');
    }

传递 guard 作为第二参数，指定用户通过哪种 guard 来认证

$this->actingAs($user, 'api')

Testing JSON API

辅助函数来测试 JSON APIs 及其响应, json，get，post，put，patch 和 delete 方法可以用于发出各种 HTTP 动作的请求。你也可以轻松的传入数据或标头到这些方法上

public function testBasicExample()
    {
        $response = $this->json('POST', '/user', ['name' => 'Sally']);

        $response
            ->assertStatus(200)
            ->assertJson([
                'created' => true,
            ]);
    }

验证部分匹配

assertJson 方法会将响应转换为数组并且利用 PHPUnit::assertArraySubset 方法来验证传入的数组是否在应用返回的 JSON 中，也就是说，即使有其它的属性存在于该 JSON 响应中，但是只要指定的片段存在，此测试仍然会通过。 （本例子中就是，如果 $response 是 [‘created'=>true, ‘status'=>1]，测试就通过）

验证完全匹配

如果上面的 assertJson 改成 assertExactJson 就表示 $response 要完全匹配

测试文件上传

首先用 Illuminate\Http\UploadedFile 类提供的 fake 方法，生成用于测试的模拟文件或图像，再结合Storage facade 的 fake 方法。

namespace Tests\Feature;

use Tests\TestCase;
use Illuminate\Http\UploadedFile;
use Illuminate\Support\Facades\Storage;
use Illuminate\Foundation\Testing\WithoutMiddleware;
use Illuminate\Foundation\Testing\DatabaseMigrations;
use Illuminate\Foundation\Testing\DatabaseTransactions;

class ExampleTest extends TestCase
{
    public function testAvatarUpload()
    {
        Storage::fake('avatars');

        $response = $this->json('POST', '/avatar', [
            'avatar' => UploadedFile::fake()->image('avatar.jpg')
        ]);

        // 断言文件已存储...
        Storage::disk('avatars')->assertExists('avatar.jpg');

        // 断言文件不存在...
        Storage::disk('avatars')->assertMissing('missing.jpg');
    }
}

在生成模拟文件时，对 fake方法创建文件可以传入参数

UploadedFile::fake()->image('avatar.jpg', $width, $height)->size(100);

除了image方法创建图片，还可以使用create创建任意类型文件

UploadedFile::fake()->create('document.pdf', $sizeInKilobytes);

可用的断言方法（HTTP 测试）

Laravel 为你的 PHPUnit 测试提供了各种各样的自定义断言方法。json，get，post，put 和 delete 这些测试方法返回的响应都可以使用这些断言方法：

$response->assertSuccessful();	断言该响应具有成功的状态码。
$response->assertStatus($code);	断言该响应具有指定的状态码。
$response->assertRedirect($uri);	断言该响应被重定向至指定的 URI。
$response->assertHeader($headerName, $value = null);	断言该响应存在指定的标头。
$response->assertCookie($cookieName, $value = null);	断言该响应包含了指定的 Cookie。
$response->assertPlainCookie($cookieName, $value = null);	断言该响应包含了指定的 Cookie（未加密）。
$response->assertSessionHas($key, $value = null);	断言该 Session 包含指定的数据。
$response->assertSessionHasErrors(array $keys, $errorBag = 'default');	断言该 Session 包含指定的字段的错误信息。
$response->assertSessionMissing($key);	断言该 Session 不包含指定的键。
$response->assertJson(array $data);	断言该响应包含指定的 JSON 数据。
$response->assertJsonFragment(array $data);	断言该响应包含指定的 JSON 片段。
$response->assertJsonMissing(array $data);	断言该响应不包含指定的 JSON 片段。
$response->assertExactJson(array $data);	断言该响应包含完全匹配指定的 JSON 数据。
$response->assertJsonStructure(array $structure);	断言该响应存在指定的 JSON 结构。
$response->assertViewIs($value);	断言该视图响应的视图名称为指定的值。
$response->assertViewHas($key, $value = null);	断言该视图响应存在指定的数据。

浏览器测试与 CI 持续集成

暂略

数据库测试

基本了解

如果我们想验证 users 数据表中是否存在 email 值为 sally@example.com 的数据 (断言数据库中是否存在与指定条件互相匹配的数据)

$this->assertDatabaseHas('users', [
    'email' => 'sally@example.com'
]);

相似的 assertDatabaseMissing ,断言数据不在数据库

也可以随意使用 PHPUnit 内置的所有断言方法来扩充测试

每次测试后重置数据库，避免干扰后面的测试

使用迁移

一种方式就是在每次测试后都还原数据库，并在下次测试前运行迁移。使用DatabaseMigrations trait，它会自动帮你处理好这些操作

class ExampleTest extends TestCase
{
    //在测试类中使用`DatabaseMigrations` trait
    use DatabaseMigrations;

    public function testBasicExample()
    {
        $response = $this->get('/');

    }
}

使用事务

使用 DatabaseTransactions trait 来自动帮你处理好这些操作

class ExampleTest extends TestCase
{
    use DatabaseTransactions;

    public function testBasicExample()
    {
        $response = $this->get('/');

        // ...
    }
}

注意

此 trait 的事务只包含默认的数据库连接。 如果你的应用程序使用多个数据库连接，你需要在你的测试类中定义一个 $connectionsToTransact 属性，然后你就可以把你测试中需要用到的数据库连接名称以数组的形式放到这个属性中。

模型工厂

创建模型工厂

常常需要在运行测试之前写入一些测试数据到数据库中，除了手动的来设置每个字段的值，还可以使用 Eloquent 模型 的「工厂」来设置每个属性的默认值

模型工厂还能用在 Seeder 里面， 用来大量创建数据。

先查看下应用程序的 database/factories/UserFactory.php 文件。此文件包含一个现成的模型工厂定义。

$factory->define(App\User::class, function (Faker $faker) {
    static $password;

    return [
        'name' => $faker->name,
        'email' => $faker->unique()->safeEmail,
        'password' => $password ?: $password = bcrypt('secret'),
        'remember_token' => str_random(10),
    ];
});

闭包内为模型工厂的定义，你可以返回模型中所有属性的默认测试值。在该闭包内会接收到 Faker PHP 函数库的实例（资料 https://github.com/fzaninotto/Faker），它可以让你很方便的生成各种随机数据以进行测试

例如除了上面的$faker->name生成姓名等，还有 address， randomElement(数组元素)等等。自己可以查阅

你也可以自己为每个数据模型创建对应的模型工厂类，在 database/factories 目录中的文件都会被 Laravel 自动加载

运行 artisan 命令来创建一个新的模型工厂

php artisan make:factory PostFactory

然后修改 define()的第一个参数，改成对应的模型，参考 UserFactory修改内部代码, 例如

use Faker\Generator as Faker;

$factory->define(App\Entities\Partner::class, function (Faker $faker) {
    return [
        'role' => $faker->randomElement($array = array ('distributor','representatives')),
        'group' => $faker->randomElement($array = array ('Pulse World','Pulse R')),
        'aera' => $faker->randomElement($array = array ('Asia','Europe', 'North America', 'South America')),
        'country_or_state' => $faker->randomElement($array = array ($faker->country, $faker->state)),
        'organization' => $faker->company,
        'emails' => $faker->unique()->safeEmail,
        'phones' => $faker->phoneNumber,
        'address' => $faker->address,
        'other' => $faker->sentence($nbWords = 6, $variableNbWords = true),
    ];
});

使用模型工厂

创建模型

定义完模型工厂后，就可以通过全局的 factory 函数来生成实例，然后用make方法创建模型（但不会保存到数据库）。

$user = factory(App\User::class)->make();

// 在测试中使用模型...

或者创建一个集合，含有多个模型

$users = factory(App\User::class, 3)->make();

或者重写模型属性（重写模型中某些默认值）

传递一个包含数值的数组至 make 方法。只有指定的数值会被替换，其它剩余的数值则会按照模型工厂指定的默认值来设置

$user = factory(App\User::class)->make([
    'name' => 'Abigail',
]);

工厂状态

工厂状态可以让你任意组合你的模型工厂，仅需要做出适当差异化的修改，就可以达到让模型拥有多种不同的状态.

例如，你的 User 模型中可以修改某个默认属性值来达到标识一种 欠款 的状态。你可以使用 state 方法来进行这种状态转换。

对于简单的工厂状态，你可以直接传入要修改的属性数组. 修改之前的 Factory 类文件

use Faker\Generator as Faker;

$factory->define(App\Entities\Partner::class, function (Faker $faker) {
    return [
        //fields...
    ]
}

//在 define factory 的代码后面添加
$factory->state(App\User::class, 'delinquent', [
    'account_status' => 'delinquent',
]);

// 另外一个状态 invalid, 通过将字段 status 设置为 0
$factory->state(App\Entities\Partner::class, 'invalid', [
    'status' => 0,
]);

如果你的工厂状态需要计算或者需要使用 $faker 实例，你可以使用闭包方法来实现状态属性的修改：

$factory->state(App\User::class, 'address', function ($faker) {
    return [
        'address' => $faker->address,
    ];
});

实际上就是一个“状态”快捷方法来简化多个表示状态的 field 修改动作

这样，就可以在创建模型的时候 应用模型工厂状态，得到一个或多个具有某个“状态”的模型

//应用状态
$users = factory(App\User::class, 5)->states('delinquent')->make();

//如果你想模型加上多种不同的状态，你只须指定每个你想添加的状态名称即可：
$users = factory(App\User::class, 5)->states('premium', 'delinquent')->make();

持久化模型

前面的make方法创建的模型并不会保存到数据库。

使用create方法不仅会创建模型实例，同时会使用 Eloquent 的 save 方法来将它们保存至数据库

// 创建一个 App\User 实例
$user = factory(App\User::class)->create();

// 创建 3 个 App\User 实例
$users = factory(App\User::class, 3)->create();

还能在持久化模型时，传入数组来重写属性

$user = factory(App\User::class)->create([
    'name' => 'Abigail',
]);

模型关联（这个挺重要的，对于模型之间有关联时，创建模型工厂要注意）

除了创建普通的模型，还可以增加关联至我们所创建的模型

下面的例子中，当使用 create 方法创建多个模型时，它会返回一个 Eloquent 集合实例，让你能够使用集合所提供的便利函数。

像 each()就是集合可以使用的一个函数。

然后在每次循环中，Eloquent 集合实例中某个元素（$u），其关联的其他模型可以再调用模型方法。例如下面的$u->posts()->save()，表示每个测试用户添加一个和它关联的测试 Blog 模型实例（一篇博客）。

$users = factory(App\User::class, 3)
    ->create()
    ->each(function ($u) {
        $u->posts()->save(factory(App\Post::class)->make());
    });

关联和属性闭包

使用闭包参数来创建模型关联。

例如如果你想在创建一个 Post 的顺便创建一个 User 实例

$factory->define(App\Post::class, function ($faker) {
    return [
        'title' => $faker->title,
        'content' => $faker->paragraph,
        'user_id' => function () {
            return factory(App\User::class)->create()->id;
        }
    ];
});

这些闭包也可以获取到生成的模型工厂包含的属性数组

$factory->define(App\Post::class, function ($faker) {
    return [
        'title' => $faker->title,
        'content' => $faker->paragraph,
        'user_id' => function () {
            return factory(App\User::class)->create()->id;
        },
        'user_type' => function (array $post) {
            return App\User::find($post['user_id'])->type;
        }
    ];
});

可用的断言方法（数据库测试）

$this->assertDatabaseHas($table, array $data);	断言数据库里含有指定表。
$this->assertDatabaseMissing($table, array $data);	断言表里没有指定数据。
$this->assertSoftDeleted($table, array $data);	断言指定记录已经被软删除。

问题

运行测试后，所有的表都被删除了（天哪 好可怕。。。）

https://stackoverflow.com/questions/48510661/laravel-5-5-testing-with-databasemigrations-deletes-all-tables

根据这里的说明，5.5 后有RefreshDatabase替代了之前的DatabaseMigrations and DatabaseTransactions.

使用这个后，如果使用in-memory数据库，就会运行 php artisan migrate；如果没有使用内存数据库，就会do a fresh run of php artisan migrate

因此推荐在测试中使用 in-memory 数据库,修改 phpunit.xml

添加 DB_CONNECTION 和 DB_DATABASE 两行

<php>
    <env name="APP_ENV" value="testing"/>
    <env name="DB_CONNECTION" value="sqlite"/>
    <env name="DB_DATABASE" value=":memory:"/>
    <env name="CACHE_DRIVER" value="array"/>
    <env name="SESSION_DRIVER" value="array"/>
    <env name="QUEUE_DRIVER" value="sync"/>
</php>

由于在 memory 里运行，phpunit 运行速度也变得更快了
