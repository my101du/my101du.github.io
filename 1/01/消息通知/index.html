消息通知

slug: laravel-notification tags:

  • laravel

除了 发送邮件，Laravel 还支持通过多种频道发送通知，包括邮件、短信（通过 Nexmo）以及 Slack 。通知还能存到数据库，这样就能在网页界面上显示了。

创建

php artisan make:notification InvoicePaid

一条通知就是一个类，通常存在 app/Notifications 文件夹里。（就像 Email 保存在 app/Mail里面…）这个类包含 via 方法和几个消息构建方法（比如 toMail 或 toDatabase），它们会针对指定的渠道把通知转换过为对应的消息

发送

有两种方式

  • 使用 Notifiable Trait 的 notify 方法
  • Notification facade

Notifiable Trait 的 notify 方法

你可以在任意模型中使用 Illuminate\Notifications\Notifiable trait，例如，User 模型中,使用了这个 trait

use Illuminate\Notifications\Notifiable;
use Illuminate\Foundation\Auth\User as Authenticatable;

class User extends Authenticatable
{
    use Notifiable;
}

然后，就可以在用户模型实例上调用notify方法发通知，使用一个通知实例做参数（就是第一步创建的通知类的实例）

use App\Notifications\InvoicePaid;

$user->notify(new InvoicePaid($invoice));

使用 Notification Facade

主要用在当你给多个 可接收通知的实体发送通知的时候，比如给用户集合发通知

Notification::send($users, new InvoicePaid($invoice));

指定发送频道

每个通知类都有个 via 方法，它决定了通知在哪个频道上发送,例如 mail database broadcast nexmo(sms) slack 等，你也可以自己实现频道 (http://laravel-notification-channels.com/)

via 方法接收一个 $notifiable 实例，它是接收通知的类实例。

/**
 * 获取通知发送频道
 *
 * @param  mixed  $notifiable
 * @return array
 */
public function via($notifiable)
{
    return $notifiable->prefers_sms ? ['nexmo'] : ['mail', 'database'];
}

队列化通知

先熟悉“队列”。

发送通知可能会花很长时间，尤其是发送频道需要调用外部 API 的时候。 可以通过添加 ShouldQueue 接口和 Queueable trait 把通知加入队列

修改通知类

namespace App\Notifications;

use Illuminate\Bus\Queueable;
use Illuminate\Notifications\Notification;
use Illuminate\Contracts\Queue\ShouldQueue;

class InvoicePaid extends Notification implements ShouldQueue
{
    use Queueable;

    // ...
}

后面再调用$user->notify(new InvoicePaid($invoice));，Laravel 会检测 ShouldQueue 接口并自动将通知的发送放入队列中。

$user->notify(new InvoicePaid($invoice));

//就像处理队列任务一样，可以通过 delay 方法来链式操作你的通知实例
$user->notify((new InvoicePaid($invoice))->delay(Carbon::now()->addMinutes(10)));

按需通知

例如，将通知发送给某个不是以”用户”身份存储在你的应用中的人。例如在开发测试阶段，想直接全部发消息给管理员的邮箱，而不是发给数据库中的真实用户。

使用 Notification::route方法，你可以在发送通知之前指定 ad-hoc 通知路由信息：

Notification::route('mail', 'taylor@laravel.com')
            ->route('nexmo', '5555555555')
            ->send(new InvoicePaid($invoice));

邮件通知

在通知类里定义一个 toMail 方法，并返回一个 Illuminate\Notifications\Messages\MailMessage 实例

/**
 * 获取通知的邮件展示方式
 *
 * @param  mixed  $notifiable
 * @return \Illuminate\Notifications\Messages\MailMessage
 */
public function toMail($notifiable)
{
    $url = url('/invoice/'.$this->invoice->id);

    return (new MailMessage)
                ->line('One of your invoices has been paid!')
                ->action('View Invoice', $url)
                ->line('Thank you for using our application!');
}

MailMessage 提供的这些方法简化了对小的事务性的邮件进行格式化操作 最后得到 HTML 邮件模板并附上文本

如果想使用更复杂的邮件模板，使用 view 方法来指定一个应用于渲染通知电子邮件的自定义模板(resources/views/emails/name.blade.php)

public function toMail($notifiable)
{
    return (new MailMessage)->view(
        'emails.name', ['invoice' => $this->invoice]
    );
}
