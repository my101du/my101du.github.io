缓存

permalink: laravel-cache tags:

  • laravel

简介

选择缓存驱动

可以使用文件缓存（默认），数据库缓存（数据库比较慢啊，是管理大量session吗？避免 /tmp 下的 session 文件有问题）、Memcache，redis，等等。

比较小的数据就用 file 缓存即可(默认)，如果是大型的，一般用内存缓存（Memcache/APC）或内存数据库(redis)

file & array的区别（在使用 Entrust 的时候会遇到需要修改 cache driver 的问题）

数组缓存驱动（array）往往仅仅用于测试，好处是不会持久化，只会在一次PHP脚本执行的生命周期内有效。

文件缓存驱动（file）往往只用于本地开发测试，因为文件缓存将缓存存储到文件中，读取时从硬盘读取，性能自然不及基于内存的缓存系统如APC或Memcached以及Redis。

数据库缓存驱动（database）将缓存数据存储到数据库中，使用之前需要在数据库中新建一张表用于存放缓存项，该表表结构可定义如下：

contracts Illuminate\Contracts\Cache\Factory 提供了访问所有缓存驱动的机制 Illuminate\Contracts\Cache\Repository 缓存驱动实现

facade：Cache

配置

配置文件 config/cache.php

这里以 Memcache 为例，其他的自行查找

'memcached' => [
    [
        'host' => '127.0.0.1',
        'port' => 11211,
        'weight' => 100
    ],
],

如果使用 Redis 作为缓存驱动

Laravel 默认是没有安装 redis 包的，直接修改env文件中的 cache driver=redis，会报错

Class ‘Predis\Client’ not found

需要先安装

composer require predis/predis

然后如果 redis 默认端口无法运行，需修改/usr/local/redis/etc/redis.conf文件里的配置。

使用

取值

//是否有值
if (Cache::has('key')) {
    //
}

//取值，没有值返回 null
$value = Cache::get('key');

//没有值，返回设置的默认值
$value = Cache::get('key', 'defaultvalue');

// 还可以传入一个闭包作为返回值，以便延迟取出值
$value = Cache::get('key', function() {
    return DB::table(...)->get();
});

//使用 store 方法访问缓存仓库（在缓存配置文件的`stores`中配置）
$value = Cache::store('file')->get('foo');

写入/修改值

// 第三个参数是保存时间
Cache::store('redis')->put('bar', 'baz', $minutes);

// 对整数型的缓存值进行递增、递减
Cache::increment('key');
Cache::increment('key', $amount);
Cache::decrement('key');
Cache::decrement('key', $amount);

删除

$value = Cache::pull('key');

缓存标签

注意

不支持 file 或 database 缓存驱动

写入

通过传递一组标签名称的有序数组，以访问被标记的缓存

Cache::tags(['people', 'artists'])->put('John', $john, $minutes);
Cache::tags(['people', 'authors'])->put('Anne', $anne, $minutes);

获取

传递一样的标签串行表至 tags 方法：

$john = Cache::tags(['people', 'artists'])->get('John');
$anne = Cache::tags(['people', 'authors'])->get('Anne');

按单个或多个标签来清空一组缓存

第一段代码：清空拥有people或authors标签，或二者皆有 的缓存项，即前面的John和Anne都会被清除

第二段代码：只清空拥有authors标签的缓存，即只有Anne被清除

Cache::tags(['people', 'authors'])->flush();
Cache::tags('authors')->flush();
