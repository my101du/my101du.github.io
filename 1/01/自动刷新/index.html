Gulp.js-livereload 不用F5了，实时自动刷新页面来开发

正如上图所示，使用 gulp-livereload，可以实时保存刷新，那样就不用按F5和切换界面了，无形中省了好多时间，有没有！：D

  • Gulp.js：比 Grunt 更简单的自动化项目构建工具
  • gulp-livereload：顾名思义，这是 Gulp.js 中的其中一个插件，本文的主题就是它了。 第一步：全局安装gulp 和当前目录部署 gulp 和 gulp-livereload
npm install gulp g
npm install gulp gulplivereload

第二步：安装 chrome 插件，reload

第一步： 当前目录新建 gulpfile.js 文件，写入以下代码。

require’gulp’ livereload require’gulp-livereload'‘watch’function// 这里的watch，是自定义的，写成live或者别的也行 server livereload// app//*.*的意思是 app文件夹下的 任何文件夹 的 任何文件watch’app//.‘function serverchanged

第二步： 命令行下运行

gulp watch

此时，你会发现当你修改 app 文件夹下的任何文件，命令行都会有内容输出，说明能够监控到文件的变化，运作正常了。

第三步： 以服务器的方式打开页面，例如 localhost:3000，而不是 file:///F:/web/app/index.html接着点击刚刚安装的 chrome插件 ，中间变为实心黑色圆点表示开始运作了。

Gulp

npm install -g gulp
npm install --save-dev gulp gulp-connect

创建 gulpfile.js

var gulp = require('gulp'),
    connect = require('gulp-connect'); //livereload


// 需要检测的文件路径
var htmlSrc = '../protected/modules/prepay/views/**/*.php',
    jsSrc = '../static/prepay/js/*.js',
    cssSrc = '../static/prepay/style/*.css';

// 默认任务
gulp.task('default', [ 'css', 'js', 'html','watch', 'connect']);

//定义livereload任务
gulp.task('connect', function() {
    connect.server({
        livereload: true
    });
});

//定义html任务
gulp.task('html', function(){
    gulp.src(htmlSrc)
        .pipe(connect.reload());
});
//定义js任务
gulp.task('js', function(){
    gulp.src(jsSrc)
        .pipe(connect.reload());
});
//定义css任务
gulp.task('css', function(){
    gulp.src(cssSrc)
        .pipe(connect.reload());
});

/* 这里的watch，是自定义的，写成live或者别的都行 */
gulp.task('watch', function() {

    gulp.watch(htmlSrc, ['html']);
    gulp.watch(jsSrc, ['js']);
    gulp.watch(cssSrc, ['css']);

});

安装 chrome 插件 livereload。

然后运行 gulp命令把这个项目跑起来， 它会自动绑定一个端口，例如 3000，然后在 chrome 浏览器打开 gulp 提供 web 服务的页面地址，例如 localhost:3000，而不是 file:///F:/web/app/index.html这种“静态本地文件路径”

这时接着点击一下激活刚刚安装的 chrome插件 livereload 图标，会自动寻找到 gulp 开放的这个端口，并且与它关联起来。中间变为实心黑色圆点表示开始运作了。

修改目录下的文件，都会触发 chrome livereload 插件生效，然后刷新页面。

问题 修改 css 也会刷新页面，不能像 webpack 那样做到不刷新页面而只修改样式。 看下面的内容，以及另外的 webpack 部分。

Webpack 实现

webpack-dev-server

太复杂了，配置了半天，html 更新还是没变化

npm install --save-dev nodemon webpack-dev-server webpack-cli

webpack.config.js

module.exports = {
    entry:__dirname+'/src/main.js',
    output:{
        publicPath:'/dist',//必须加publicPath
        path:__dirname+'/dist',
        filename:'bundle.js'
    },
    devServer:{
        host:'localhost',
        port:'8080',
        publicPath: '/staic/',
        // open:true//自动拉起浏览器
        // hot:true,//热加载
        //hotOnly:true
    }
}

webpack-livereload-plugin

npm install --save-dev webpack-livereload-plugin

webpack.config.js

var LiveReloadPlugin = require('webpack-livereload-plugin');
 
module.exports = {
  plugins: [
    new LiveReloadPlugin(options)
  ]
}

在文件中

<script src="http://localhost:35729/livereload.js"></script>

启动

webpack --watch

全局安装 webpack-dev-server 然后监视一个目录

npm -g install webpack-dev-server

webpack-dev-server --content-base folder/

parcel

parcel ./src/app/index.html

修改 css 自动刷新（hot reload）

这个会把文件编译到 /dist/ 目录

如果是传统项目，需要保留 html css 文件原来的样子，还是用 gulp 只负责监听文件变化即可

或者 parcel watch ./src/app/index.html ?
