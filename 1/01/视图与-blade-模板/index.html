视图与 Blade 模板

slug: laravel-view-blade tags:

  • laravel

视图

普通的

可以在路由里，或者控制器里使用 view 方法来加载一个模板文件和数据。

return view('folder.page', ['key'=>'value'])

在所有视图中共享数据

使用ServiceProvider，创建新的或直接修改app/Providers/AppServiceProvier,

public function boot()
{
   // share data to all application views
   view()->share('shareData', 'test_share_data');
}

视图组件

这是一些闭包或类方法，在指定的视图被渲染前被调用。

创建一个目录App\Http\ViewComposers用来保存视图组件。内容

namespace App\Http\ViewComposers;

use Illuminate\Contracts\View\View;
use Illuminate\Users\Repository as UserRepository;

class ProfileComposer
{
    protected $users;

    public function __construct(UserRepository $users)
    {
        // 所有依赖都会自动地被服务容器解析...
        $this->users = $users;
    }

    /**
     * 将数据绑定到视图。视图被渲染前，这个方法会被调用，传入一个 Illuminate\Contracts\View\View 实例
     */
    public function compose(View $view)
    {
        $view->with('count', $this->users->count());
    }
}

然后要在一个 ServiceProvider （服务提供者）里注册视图组件，可以直接修改系统的，或者自己建一个App\Providers\ComposerServiceProvider

namespace App\Providers;
use Illuminate\Support\ServiceProvider;

class ComposerServiceProvider extends ServiceProvider
{
    /**
     * 在容器内注册所有绑定。
     */
    public function boot()
    {
        // 使用对象型态的视图组件...
        view()->composer(
            'profile', 'App\Http\ViewComposers\ProfileComposer'
        );

        // 使用闭包型态的视图组件...
        view()->composer('dashboard', function ($view) {

        });
    }

    /**
     * 注册服务提供者。
     */
    public function register()
    {
        //
    }
}

创建完了一个包含注册视图组件的新服务提供者后，要在config/app.php里把它加进providers数组

视图创建者，时机不同

view()->creator('profile', 'App\Http\ViewCreators\ProfileCreator');

Blade 模板

基本

使用

return view('parent.child', ['key'=>'value']);

文件resources/views/parent/child.blade.php

<!--会自动调用 PHP htmlentites 函数来防御 XSS 攻击-->
{{ $title }}

<!--如果不想处理, 注意只有一对花括号-->
{ !! $name !! }

<!--支持 PHP 函数-->
{{ time() }}

<!--执行一段 php 代码-->
@php
    //
@endphp

为了和 AngularJS 等js框架里的{% raw %}{{ }}{% endraw %}区分开来，可以用 @ {% raw %}{{ }}{% endraw %}来区别 blade 的模板变量

注释

{{ -- 此注释将不会出现在渲染后的 HTML -- }}

JavaScript 框架兼容

由于很多 JavaScript 框架都使用花括号来表明所提供的表达式，所以你可以使用 @ 符号来告知 Blade 渲染引擎你需要保留这个表达式原始形态

@ 符号最终会被 Blade 引擎剔除，并且 {{ name }} 表达式会被原样的保留下来.

如果你需要在页面中大片区块中展示 JavaScript 变量，你可以使用 @verbatim 指令来包裹 HTML 内容，这样你就不需要为每个需要解析的变量增加 @ 符号前缀了：

<h1>Laravel</h1>

Hello, @{{ name }}.


@verbatim
    <div class="container">
        Hello, {{ name }}.
    </div>
@endverbatim

模板继承

定义一个“母版”页面布局resources/views/layouts/master.blade.php

<head>
    <title>应用程序名称 - @yield('title')</title>
</head>
<body>

	@section('sidebar')
		main sidebar
	@show	

	<div class="container">
		@yield('content')
	</div>
</body>

@section定义一个内容区块（好比定义变量），而@yield用于显示指定区块（好比显示变量值），然后可以用 @section("name", "value")或@section('content') html内容 @endsection来给定义的内容区块赋值，这样在 @yield("name")里就能显示这个值了

然后可以让一个模板来继承这个布局resources/views/home/index.blade.php，下面的代码中，给 title，sidebar和content都赋值，还用@parent把主布局模板里的内容继承下来增加一起显示(如果不使用，则会覆盖掉布局中的这部分内容)

@extends('layouts.master')

@section('title', 'this is title')

@section('sidebar')
	@parent
	<p>this is the second sidebar</p>
@endsection

@section('content')
	<p>this is the content</p>
@endsection

组件 和 Slots

也能提供类似区块和布局的好处，但更容易理解。

例如有一个“警告”的组件。里面的 {{ $slot }} 变量将包含我们希望注入到组件的内容。

<!-- /resources/views/alert.blade.php -->

<div class="alert alert-danger">
    {{ $slot }}
</div>

然后使用 @component 指令来构造这个组件

@component('alert')
    <strong>哇！</strong> 出现了一些问题！
@endcomponent

如果一个组件有多个 slots，已命名的 slots 将显示「相对应」名称的变量的值:

<!-- /resources/views/alert.blade.php -->

<div class="alert alert-danger">
    <div class="alert-title">{{ $title }}</div>

    {{ $slot }}
</div>

可以使用 @slot 指令注入内容到已命名的 slot 中, 任何没有被 @slot 指令包裹住的内容将传递给组件中的 $slot 变量

@component('alert')
    @slot('title')
        Forbidden
    @endslot

    你没有权限访问这个资源！
@endcomponent

传递额外的数据给组件, 所有的数据都将以变量的形式传递给组件模版

@component('alert', ['foo' => 'bar'])
    ...
@endcomponent

IF 与 循环

//条件判断
@if ()
@elseif
@else
@endif

@unless(Auth::check()))
@endunless

// 对应 PHP 功能
@isset($records)
@endisset

@empty($records)
@endempty

// 身份验证快捷方式
@auth
@endauth

@guest
@endguest

// switch
@switch($i)
    @case(1)
        @break

    @case(2)
        @break

    @default
        Default case...
@endswitch

// 循环
@for()
@endfor

@foreach()
    // 可以某些条件下跳出/继续循环
    @if ($user->type == 1)
        @continue
    @endif

    <li>{{ $user->name }}</li>

    @if ($user->number == 5)
        @break
    @endif
@endforeach

@forelse ($users as $user)
@empty
    <p>没有用户</p>
@endforelse

@while (true)
    <p>我永远都在跑循环。</p>
@endwhile

循环变量

在循环内访问 $loop 变量。这个变量可以提供一些有用的信息，比如当前循环的索引，当前循环是不是首次迭代，又或者当前循环是不是最后一次迭代

@foreach ($users as $user)
    @if ($loop->first)
        这是第一个迭代。
    @endif

    @if ($loop->last)
        这是最后一个迭代。
    @endif

    <p>This is user {{ $user->id }}</p>
@endforeach

如果你是在一个嵌套的循环中，你可以通过使用 $loop 变量的 parent 属性来获取父循环中的 $loop 变量：

@foreach ($users as $user)
    @foreach ($user->posts as $post)
        @if ($loop->parent->first)
            This is first iteration of the parent loop.
        @endif
    @endforeach
@endforeach

$loop 变量也包含了其它各种有用的属性：

  • $loop->index 当前循环所迭代的索引，起始为 0。
  • $loop->iteration 当前迭代数，起始为 1。
  • $loop->remaining 循环中迭代剩余的数量。
  • $loop->count 被迭代项的总数量。
  • $loop->first 当前迭代是否是循环中的首次迭代。
  • $loop->last 当前迭代是否是循环中的最后一次迭代。
  • $loop->depth 当前循环的嵌套深度。
  • $loop->parent 当在嵌套的循环内时，可以访问到父循环中的 $loop 变量。

为集合渲染视图，最后一个参数可选

注意

通过 @each 呈现的视图不会从父视图继承变量。 如果子视图需要这些变量，则应该使用 @foreach 和 @include

第一个参数为每个元素要渲染的子视图，第二个参数是你要迭代的数组或集合，而第三个参数为迭代时被分配至子视图中的变量名称

@each('view.name', $jobs, 'job', 'view.empty')

待实验，是否类似?

foreach($jobs as $job){
	if($job){
		return view('view.name', ['job'=>$job]);
	}else{
		return view('view.empty');
	}
}

你也可以传递第四个参数到 @each 命令。当需要迭代的数组为空时，将会使用这个参数提供的视图来渲染。

@each('view.name', $jobs, 'job', 'view.empty')

引入子视图

注意

避免在 Blade 视图中使用 __DIR__ 及 __FILE__ 常量，因为他们会引用视图被缓存的位置

所有在父视图的可用变量在被引入的视图中都是可用的

<!--会继承父视图的数据，并额外传入数据-->
@include('view.name', ['some' => 'data'])

如果视图不存在，可以使用它来防止抛出错误（无法确定视图是否存在）

@includeIf('view.name', ['some' => 'data'])

如果你想根据给定的布尔条件 @include 一个视图，你可以使用 @includeWhen 指令：

@includeWhen($boolean, 'view.name', ['some' => 'data'])

堆栈

你在其它视图或布局中为已经命名的堆栈中压入数据，这在子视图中引入必备的 JavaScript 类库时尤其有用

可以根据需要多次压入堆栈

@push('scripts')
    <script src="/example.js"></script>
@endpush

@push('scripts')
    <script src="/example2.js"></script>
@endpush

通过 @stack 命令中键入堆栈的名字来渲染整个堆栈：

<head>
    <!-- Head Contents -->

    @stack('scripts')
</head>

注入服务容器中的服务

你可以使用 @inject 命令来从 Larvel service container 中取出服务。传递给 @inject 的第一个参数为置放该服务的变量名称，而第二个参数为你想要解析的服务的类或是接口的名称：

@inject('metrics', 'App\Services\MetricsService')

<div>
    每月收入：{{ $metrics->monthlyRevenue() }}
</div>

拓展 Blade()

Blade 甚至允许你使用 directive 方法来注册自己的命令。当 Blade 编译器遇到该命令时，它将会带参数调用提供的回调函数。

以下例子会创建一个把指定的 $var 格式化的 @datetime($var) 命令 (文件 AppServiceProvider)

//在 AppServiceProvider的boot方法里定义
public function boot()
    Blade::directive('datetime', function($expression) {
        return "<?php echo with{$expression}->format('m/d/Y H:i'); ?>";
    });
}

这样就可以在blade模板里用@datetime($var)来实现转换后为<?php echo with($var)->format('m/d/Y H:i'); ?>的php代码

<?php echo ($var)->format('m/d/Y H:i'); ?>

自定义 if 语句

Blade提供了一个 Blade::if 方法，它允许你使用闭包快速定义自定义条件指令 例如定义一个自定义条件来检查当前的应用程序环境。可以在 AppServiceProvider 的 boot 方法中这样做

public function boot()
{
    Blade::if('env', function ($environment) {
        return app()->environment($environment);
    });
}

然后就可以在模板中使用

@env('local')
    // The application is in the local environment...
@else
    // The application is not in the local environment...
@endenv
