如图，这是咱们国家的第一封电子邮件，于1987年发往德国。20多年后，我想再次怀旧一把，重新感受一下这历史时刻。于是找来了一台从二手市场淘来的14寸球面显示器，擦掉尘封多年的“双飞燕”机械键盘上的灰土，脱掉鞋子，穿上无尘袜，登陆路由器把网速限制在56kbps，安装了一台win95的虚拟机（好吧，1987年还没有这个），安装了一个非常老版本的firefox，然后提前准备了一个德国大学的Email地址。 然后尝试登陆 Gmail，这时候浏览器告诉我“您访问的页面无法显示”。 What？不是说越过长城走向世界吗？

本文是关于如何在个人电脑、手机、平板上使用客户端进行科学上网的教程，并不是教你如何搭建 VPN 或 Shadowsocks 环境提供科学上网服务，有关这方面的内容，请学习我们《Linux服务安装与配置》教程中的有关章节

$ sslocal -s server_ip -p server_port  -l 1080 -k password -t 600 -m aes-256-cfb

电脑

代理服务器在新加坡/香港等外地，google定位到 .sg .hk 等，导致在地址栏输入关键词后搜索很慢

注意一定要按顺序操作

  1. 首先把 www.google.com.sg/.hk 添加到 gfwlist 域名列表，确保等下能打开搜索结果（虽然很慢）
  2. 随便输入搜索关键词，在列出的结果底部找到“位置信息”
  3. 默认是空的，改成使用确切位置，在弹出的权限提示里选择“允许”，这样会自动定位你的具体位置。
  4. 输入 www.google.com，在右下角点击“使用 “google.com”
  5. 然后输入 www.google.com/ncr
  6. 然后关闭浏览器打开，再次搜索，就会自动使用 .com，而不会用 .hk 或 .sg 了。

但其中有几个坑要注意：

  1. 如果服务器是专有网络，/etc/shadowsocks.json 中的server ip 是私有ip,而非公网ip

  2. /etc/shadowsocks.json中开放的端口需要 运行命令开放，iptables -A INPUT -p tcp –dport 8388 -j ACCEPT

  3. 服务器实例的安全组规则需要增加 自定义 TCP 在 相关端口（8388） 的访问 ,（允许所有ip访问，设置为0.0.0.0/0）

最后用telnet your_public_ip 8388验证, 只有telnet能访问端口了，才能正常使用shadowsocks!

Cow 自动分流国内外

注意

（1）和直接使用 socks5 的国外域名白名单模式对比，如果国外vps速度慢，由于多个国外网址都走代理，可能会更慢 （2）对 vps 的流量消耗更大！（因为之前 block 的域名，现在都会从代理走了）

所以，我还是继续只用 ss 的socks5 PAC 模式，代理白名单内的网址

默认情况下， ss 在客户端运行时，用 PAC 列表来让里面的网址走 sock5 代理。

但是，有时候开一些网站，如果请求的资源没有在这个列表里（同时你不知道），那么速度就很慢

使用 PAC 可获得更好的性能，但若 PAC 中某网站从直连变成被封，浏览器会依然尝试直连。遇到这种情况可以暂时不使用 PAC 而总是走 HTTP 代理，让 COW 学习到新的被封网站。
